00 왜 3D, 왜 Three.js인가

웹은 원래 종이였습니다. 이제 무대가 됩니다.

웹페이지는 오랫동안 평평한 문서였습니다. 글과 이미지를 위에서 아래로 흘려보내는 종이에 가까웠죠. 하지만 오늘날의 브라우저는 여러분 컴퓨터의 GPU(그래픽 카드)를 직접 사용해 실시간 3D를 그릴 수 있습니다.

이 능력을 담당하는 브라우저의 기술이 WebGL입니다. 문제는 WebGL이 너무 저수준이라는 점입니다. 화면에 삼각형 하나를 띄우려 해도 정점 버퍼를 만들고, 셰이더(GPU에서 도는 작은 프로그램)를 직접 작성하고, 행렬을 손으로 계산해야 합니다. 입문자에게는 수백 줄의 진입 장벽이죠.

🧱
비유로 이해하기. WebGL이 벽돌·시멘트·철근이라면, Three.js는 그 위에 만들어진 조립식 가구와 전동 공구 세트입니다. 집을 지을 때 모래부터 반죽하는 대신, "여기에 벽 하나, 저기에 창문 하나"라고 말하면 되는 것이죠.

Three.js는 이 복잡한 WebGL을 감싸서, 우리가 평소 쓰는 직관적인 단어 — 장면(scene), 카메라(camera), 빛(light), 도형(geometry) — 로 3D를 다루게 해주는 자바스크립트 라이브러리입니다. 2010년부터 이어진 가장 성숙하고, 가장 많이 쓰이며, 예제와 자료가 압도적으로 많은 웹 3D 라이브러리입니다.

알아두기 WebGPU는 WebGL의 차세대 후속 기술로, 더 빠르고 현대적입니다. Three.js는 이미 WebGPU 렌더러를 지원하기 시작했으므로, 지금 Three.js를 배워두면 미래 기술로도 자연스럽게 이어집니다.
01 모든 3D 장면의 세 기둥

장면 · 카메라 · 렌더러

Three.js의 모든 작업은 결국 세 가지 핵심 요소로 귀결됩니다. 이 셋만 머릿속에 그릴 수 있으면 절반은 끝난 셈입니다.

🎬

좌표계: 3D 공간을 읽는 법

3D 공간의 모든 위치는 세 개의 축으로 표현됩니다. Three.js의 기본 약속은 다음과 같습니다.

  • X축 (빨강) — 좌우. 오른쪽이 +
  • Y축 (초록) — 위아래. 위쪽이 +
  • Z축 (파랑) — 앞뒤. 화면 바깥(나를 향하는 쪽)이 +

위 히어로 화면과 아래 데모들에 보이는 빨강·초록·파랑 선이 바로 이 X·Y·Z축입니다. 이 색 약속은 Three.js 어디에서나 똑같이 쓰이니 눈에 익혀 두면 좋습니다.

02 첫 장면 만들기 — Hello, Cube

회전하는 정육면체, 3D의 "Hello World"

이제 실제로 코드를 봅시다. 화면에 무언가를 띄우려면 한 가지 개념이 더 필요합니다. 바로 메시(Mesh)입니다.

핵심 공식 Mesh = Geometry + Material. Geometry(지오메트리)는 물체의 형태(뼈대·모양)이고, Material(머티리얼)은 그 표면의 재질(색·질감·광택)입니다. 둘을 합치면 화면에 보이는 하나의 물체(Mesh)가 됩니다.

hello-cube.js
import * as THREE from 'three';

// 1) 장면(Scene) — 3D 세계를 담는 컨테이너
const scene = new THREE.Scene();

// 2) 카메라(Camera) — 어디서 바라볼지
const camera = new THREE.PerspectiveCamera(
  75,                          // 시야각(FOV)
  window.innerWidth / window.innerHeight, // 화면 비율
  0.1, 1000                     // 보이는 거리 범위(near, far)
);
camera.position.z = 3;            // 뒤로 3칸 물러나 바라본다

// 3) 렌더러(Renderer) — 실제로 화면에 그려주는 엔진
const renderer = new THREE.WebGLRenderer({ antialias: true });
renderer.setSize(window.innerWidth, window.innerHeight);
document.body.appendChild(renderer.domElement);

// 4) 메시(Mesh) = 형태(Geometry) + 재질(Material)
const geometry = new THREE.BoxGeometry(1, 1, 1);
const material = new THREE.MeshNormalMaterial();
const cube = new THREE.Mesh(geometry, material);
scene.add(cube);          // 무대에 큐브를 올린다

// 5) 애니메이션 루프 — 매 프레임 다시 그린다 (보통 초당 60번)
function animate() {
  requestAnimationFrame(animate);
  cube.rotation.x += 0.01;
  cube.rotation.y += 0.01;
  renderer.render(scene, camera);  // 찰칵! 한 장 찍는다
}
animate();

이 코드의 결과가 바로 아래 데모입니다. 정육면체가 천천히 회전하죠. 버튼으로 회전을 멈추거나 다시 시작해 보세요.

애니메이션 속도

주목할 점은 requestAnimationFrame 안에서 회전값을 조금씩 바꾸고 다시 render를 호출한다는 것입니다. 영화 필름처럼 정지 화면을 1초에 약 60장 빠르게 갈아끼우면 우리 눈에는 부드러운 움직임으로 보입니다. 이것이 모든 실시간 3D의 원리입니다.

03 구성 요소 자세히 들여다보기

도형, 재질, 그리고 빛

"Mesh = Geometry + Material"을 기억하시죠? 이제 각각에 어떤 선택지가 있는지, 그리고 왜 빛이 필요한지 살펴봅니다. 아래 플레이그라운드에서 직접 바꿔 보며 읽으면 가장 빠르게 이해됩니다.

① Geometry — 형태

Three.js는 자주 쓰는 기본 도형을 미리 만들어 둡니다. 직접 정점을 찍지 않아도 한 줄로 불러올 수 있습니다.

  • BoxGeometry 정육면체 · SphereGeometry 구 · PlaneGeometry 평면(바닥·벽)
  • TorusGeometry 도넛 · ConeGeometry 원뿔 · CylinderGeometry 원기둥
  • TorusKnotGeometry 매듭 · DodecahedronGeometry 다면체 등 다양한 형태

② Material — 재질, 그리고 빛이 필요한 이유

같은 형태라도 재질에 따라 완전히 달라 보입니다. 가장 중요한 차이는 "빛에 반응하는가"입니다.

  • MeshBasicMaterial — 빛을 무시하고 지정한 색을 그대로 칠합니다. 조명이 없어도 보이지만 입체감이 없습니다.
  • MeshNormalMaterial — 표면 방향을 무지개색으로 보여 줍니다. 디버깅·연습용으로 좋아 빛 없이도 입체감이 납니다.
  • MeshStandardMaterial — 물리 기반(PBR) 재질. 금속성과 거칠기를 조절해 사실적인 표현이 가능합니다. 빛이 있어야 제대로 보입니다.
  • MeshPhongMaterial — 가볍고 광택(하이라이트) 표현에 좋은 전통적 재질입니다.
자주 겪는 함정 "화면이 새까매요!" 입문자가 가장 많이 겪는 문제입니다. 대부분 MeshStandardMaterial 같은 빛 반응 재질을 썼는데 장면에 조명을 추가하지 않아서입니다. 빛이 없으면 물체에 닿을 빛도 없으니 까맣게 보이죠. 아래 플레이그라운드에서 조명을 꺼 보면 바로 확인됩니다.

④ 위치·회전·크기 (Transform)

장면에 올린 모든 객체는 세 가지 속성으로 변형할 수 있습니다. 각각 X·Y·Z 세 축의 값을 가집니다.

transform.js
mesh.position.set(2, 0, -1);  // X로 2, Z로 -1 이동
mesh.rotation.y = Math.PI / 4;  // Y축 기준 45도 회전 (라디안 단위)
mesh.scale.set(1.5, 1.5, 1.5); // 1.5배 확대

회전 단위가 도(°)가 아니라 라디안이라는 점만 주의하세요. 180° = Math.PI, 90° = Math.PI / 2입니다.

04 빛, 움직임, 상호작용

장면에 생명을 불어넣기

조명의 종류

현실의 조명처럼, Three.js도 여러 종류의 빛을 제공합니다. 보통 여러 개를 섞어 씁니다.

  • AmbientLight — 사방에서 고르게 비추는 환경광. 그림자는 없지만 전체를 은은하게 밝힙니다.
  • DirectionalLight — 태양처럼 한 방향에서 평행하게 오는 빛. 그림자를 만들 수 있습니다.
  • PointLight — 전구처럼 한 점에서 모든 방향으로 퍼지는 빛.
  • SpotLight — 손전등처럼 원뿔 모양으로 비추는 빛.
Lighting
조명 켜고 끄기

방향광을 끄고 환경광만 남기면 입체감(그림자 면)이 사라지는 걸 볼 수 있습니다. 조명은 분위기와 입체감을 좌우하는 핵심입니다.

OrbitControls — 마우스로 둘러보기

3D 뷰어라면 대부분 마우스로 돌려보고 싶을 것입니다. 직접 구현할 필요 없이 Three.js의 부가 모듈 OrbitControls를 가져오면 끝납니다.

controls.js
import { OrbitControls } from 'three/addons/controls/OrbitControls.js';

const controls = new OrbitControls(camera, renderer.domElement);
controls.enableDamping = true;  // 부드러운 관성 효과

// 루프 안에서 매 프레임 갱신
function animate() {
  requestAnimationFrame(animate);
  controls.update();
  renderer.render(scene, camera);
}

아래 데모를 직접 드래그·확대·이동해 보세요. 바닥 격자(GridHelper)와 X·Y·Z 축(AxesHelper)도 함께 그려 두었습니다.

05 다른 3D 라이브러리와 비교

그래서 무엇을 골라야 할까

Three.js가 가장 인기 있지만 유일한 선택지는 아닙니다. 각 도구가 무엇에 강한지 알아두면 상황에 맞게 고를 수 있습니다.

도구유형난이도가장 잘 맞는 용도
Three.js 저수준 WebGL 래퍼 보통 웹사이트 인터랙티브 3D, 데이터 시각화, 제품 뷰어. 자유도와 자료가 가장 풍부.
Babylon.js 3D 게임 엔진 보통 물리·충돌·에디터 등 게임 기능이 기본 내장. 게임이나 복잡한 인터랙티브 앱에 강함.
React Three Fiber Three.js의 React 래퍼 보통 이미 React를 쓴다면 최적. Three.js를 컴포넌트처럼 선언적으로 작성. (내부는 Three.js)
A-Frame HTML 기반 WebXR 쉬움 HTML 태그만으로 VR/AR 장면 제작. 빠른 프로토타입, 교육용. (내부는 Three.js)
PlayCanvas 클라우드 게임 엔진 보통 브라우저 기반 비주얼 에디터와 협업 기능. 가벼운 웹 게임·광고 콘텐츠.
순수 WebGL / WebGPU 브라우저 원시 API 어려움 최대 성능과 완전한 제어가 필요한 특수한 경우. 셰이더를 직접 다룸.
Unity / Unreal (WebGL 빌드) 네이티브 게임 엔진 어려움 대규모 3D 게임을 웹으로 내보낼 때. 강력하지만 결과물 용량이 크고 무거움.
결정 가이드 웹사이트에 3D 요소를 넣고 싶다 → Three.js. React 프로젝트다 → React Three Fiber. 본격적인 웹 게임을 만든다 → Babylon.js 또는 PlayCanvas. VR/AR을 빠르게 시도한다 → A-Frame. 중요한 점: A-Frame과 React Three Fiber는 내부적으로 모두 Three.js를 사용합니다. 즉 Three.js를 배우면 이들 대부분으로 자연스럽게 확장됩니다 — 그래서 첫 3D 라이브러리로 가장 추천됩니다.

건축·디지털 트윈

BIM 모델 뷰어, 공장·설비의 실시간 디지털 트윈 대시보드. 센서 데이터를 3D 위에 시각화.

특히 마지막의 디지털 트윈은 데이터·클라우드 분야와 만나는 지점입니다. 백엔드에서 모은 실시간 지표를 Three.js 장면 위에 입히면, 추상적인 숫자가 직관적인 공간 정보로 바뀝니다.

07 학습 로드맵 & 성능 팁

다음에 무엇을 배울까

기초를 잡았다면, 다음 순서로 넓혀가는 것을 추천합니다.

  1. Scene · Camera · Renderer 손에 익히기빈 화면에서 시작해 직접 큐브를 띄우고 회전시켜 보세요. 외우지 말고 손가락이 기억하게.
  2. Geometry · Material · Light 조합 실험위 플레이그라운드처럼 다양한 조합을 만들어 보며 "이렇게 하면 이렇게 보이는구나"를 체득.
  3. OrbitControls와 애니메이션 루프마우스 조작과 시간 기반 움직임. 인터랙티브한 장면의 기본기.
  4. 텍스처와 외부 모델(glTF) 불러오기이미지를 표면에 입히고, 디자이너의 3D 모델을 장면에 올리는 법.
  5. 그림자, 환경맵, 후처리 효과사실감을 끌어올리는 단계. PBR 재질과 조명을 깊이 있게.
  6. 성능 최적화 & 셰이더 입문대규모 장면을 가볍게 굴리는 기법, 그리고 원하면 직접 셰이더(GLSL)에 도전.

꼭 기억할 성능 팁

  • Draw call을 줄이세요. 똑같은 물체를 수천 개 그릴 땐 InstancedMesh로 한 번에 처리합니다.
  • 안 쓰는 자원은 dispose 하세요. Geometry·Material·Texture는 GPU 메모리를 차지하므로 화면에서 치울 때 .dispose()로 정리합니다.
  • 화면 밖이면 멈추세요. 보이지 않는 장면은 렌더 루프를 일시정지해 배터리와 자원을 아낍니다. (이 페이지의 데모들도 화면에 보일 때만 그려집니다.)
  • 폴리곤과 텍스처는 적당히. 모바일에서는 고해상도 모델이 발열·끊김의 원인이 됩니다.
추천 자료 공식 문서 threejs.org/docs와, 실제 동작하는 코드가 가득한 threejs.org/examples가 가장 좋은 교재입니다. 예제 코드를 열어 한 줄씩 바꿔보는 것이 가장 빠른 학습법입니다.
08 이해도 점검 퀴즈

스스로 확인하기

읽은 내용을 가볍게 점검해 봅시다. 정답을 고르면 바로 채점됩니다.

Q1. 모든 Three.js 장면을 이루는 세 가지 핵심 요소는?
정답! 무대(Scene)·카메라(Camera)·촬영장비(Renderer) — 이 셋이 모든 장면의 뼈대입니다.
Q2. "Mesh = Geometry + ___" 빈칸에 들어갈 것은?
정답! Geometry는 형태, Material은 표면의 색·질감입니다. 둘을 합치면 보이는 물체(Mesh)가 됩니다.
Q3. MeshStandardMaterial을 썼는데 물체가 새까맣게 보입니다. 가장 흔한 원인은?
정답! Standard·Phong 같은 재질은 빛에 반응합니다. 비출 빛이 없으면 까맣게 보이죠.
Q4. A-Frame과 React Three Fiber의 공통점은?
정답! 그래서 Three.js를 배우면 이들로 확장하기가 훨씬 쉽습니다.
