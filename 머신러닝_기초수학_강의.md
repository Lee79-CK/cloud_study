## 들어가며: 왜 수학이 필요한가요?

### "코딩만 하면 안 되나요?"

머신러닝을 시작하려고 책을 펼치면 갑자기 이상한 기호들이 쏟아집니다.

$$
\nabla_\theta J(\theta) = \frac{1}{m}\sum_{i=1}^{m}(h_\theta(x^{(i)}) - y^{(i)})x^{(i)}
$$

여기서 많은 분들이 책을 덮습니다. 충분히 이해됩니다.

하지만 좋은 소식이 있습니다. 머신러닝에 필요한 수학은 **생각보다 적고, 생각보다 직관적**입니다. 우리에게 필요한 것은 어려운 증명을 풀어내는 능력이 아니라, **수식이 무슨 말을 하고 있는지 그림으로 떠올리는 능력**입니다.

### 이 강의가 드리는 약속

- 🚫 증명은 거의 하지 않습니다.
- ✅ 대신 **그림과 비유**로 설명합니다.
- ✅ 모든 개념을 **머신러닝의 어디에 쓰이는지**와 함께 설명합니다.
- ✅ MLOps/LLMOps 실무와의 **연결고리**를 짚어 드립니다.

### 왜 이 세 가지를 배우나요?

| 분야 | 머신러닝에서의 역할 | 한 줄 비유 |
|---|---|---|
| **선형대수** | 데이터와 모델을 **표현**하는 언어 | 데이터의 "문법" |
| **확률/통계** | 불확실성을 **다루는** 도구 | 데이터의 "신뢰도 측정기" |
| **미적분** | 모델을 **학습**시키는 엔진 | 모델의 "내비게이션" |

머신러닝 모델은 결국 이 세 가지가 맞물려 돌아가는 기계입니다.
- 데이터를 **벡터**로 표현하고 (선형대수)
- 모델 출력에 대한 **확률**을 다루며 (확률/통계)
- 손실을 **미분**해서 모델을 학습시킵니다 (미적분)

자, 이제 시작해 봅시다.

## Chapter 1. 벡터(Vector) — 방향이 있는 숫자

### 1.1 벡터가 뭔가요?

학교에서 배운 "숫자"는 이런 것이었습니다.

```
3, 7, -2.5, 100
```

이런 숫자를 **스칼라(scalar)** 라고 합니다. 그냥 "크기"만 있는 숫자죠.

**벡터(vector)** 는 여기서 한 발 더 나아갑니다. **여러 숫자를 한 묶음으로 다루는 것**이 벡터입니다.

```
[3, 7]        ← 2차원 벡터
[3, 7, -2]    ← 3차원 벡터
[1, 2, 3, 4]  ← 4차원 벡터
```

### 1.2 직관적 이해: 벡터는 "화살표"입니다

벡터를 가장 쉽게 이해하는 방법은 **화살표**로 그려보는 것입니다.

`[3, 2]` 라는 벡터가 있다고 해봅시다. 이건 이렇게 읽습니다:

> "원점에서 시작해서, **오른쪽으로 3칸, 위로 2칸** 간 점까지의 화살표"

```
       y
       ↑
    3  |
    2  |        ●  ← 도착점 (3, 2)
    1  |      /
       |    /
       |  /
    0  |/____________→ x
       0  1  2  3
```

벡터는 **두 가지 정보**를 담고 있습니다:
1. **방향**: 어느 쪽을 향하는가?
2. **크기(길이)**: 얼마나 멀리 가는가?

### 1.3 일상의 비유

벡터는 우리 주변 어디에나 있습니다.

- 🌬️ **바람**: "북동쪽으로 시속 20km" → 방향 + 크기 = 벡터
- 🚗 **자동차의 속도**: "서쪽으로 60km/h" → 벡터
- 🎮 **게임 캐릭터의 이동**: 키보드 방향키를 누르면 캐릭터가 움직이는 방향 = 벡터
- 📍 **지도 위의 위치**: 서울 시청에서 강남역까지 = 벡터 (남동쪽으로 약 12km)

### 1.4 벡터 연산 — 더하기와 곱하기

#### 벡터 덧셈: "이어 붙이기"

벡터 두 개를 더한다는 건 **화살표를 이어 붙이는 것**입니다.

```
[2, 1] + [1, 2] = [3, 3]
```

비유: "동쪽으로 2km, 북쪽으로 1km 가고 → 다시 동쪽으로 1km, 북쪽으로 2km 갔다"
= "총 동쪽으로 3km, 북쪽으로 3km 간 셈"

#### 스칼라 곱: "늘리거나 줄이기"

벡터에 그냥 숫자(스칼라)를 곱하면 길이가 바뀝니다.

```
2 × [3, 2] = [6, 4]    ← 두 배로 늘어남
0.5 × [3, 2] = [1.5, 1]  ← 절반으로 줄어듦
-1 × [3, 2] = [-3, -2]   ← 반대 방향으로 뒤집힘
```

비유: 같은 방향으로 두 배 멀리 가거나, 절반만 가거나, 반대로 가는 것.

### 1.5 핵심 개념: 내적(Dot Product) — 두 벡터의 "닮은 정도"

내적은 머신러닝에서 **가장 중요한 연산** 중 하나입니다.

#### 계산 방법
두 벡터를 **자리별로 곱한 다음, 다 더하는 것**입니다.

```
[1, 2, 3] · [4, 5, 6]
= (1×4) + (2×5) + (3×6)
= 4 + 10 + 18
= 32
```

#### 직관적 의미: "두 벡터가 얼마나 비슷한 방향인가?"

- 두 벡터가 **같은 방향** → 내적이 **큰 양수**
- 두 벡터가 **수직(90도)** → 내적이 **0**
- 두 벡터가 **반대 방향** → 내적이 **큰 음수**

```
같은 방향:    →  →     →  내적 큼 (양수)
수직:         →  ↑     →  내적 = 0
반대 방향:    →  ←     →  내적 작음 (음수)
```

### 1.6 머신러닝과의 연결: 단어 임베딩(Word Embedding)

여기서부터가 진짜 재미있습니다. **"왜 이걸 배우는가?"** 에 대한 답입니다.

#### 컴퓨터가 단어를 이해하는 법

컴퓨터는 "고양이", "강아지" 같은 단어를 그 자체로 이해할 수 없습니다. 그래서 **모든 단어를 벡터로 변환**합니다. 이걸 **임베딩(Embedding)** 이라고 합니다.

예를 들어:
```
"고양이" → [0.2, 0.8, -0.1, 0.5, ...]  (300차원 벡터)
"강아지" → [0.3, 0.7, -0.2, 0.6, ...]
"자동차" → [-0.5, 0.1, 0.9, -0.3, ...]
```

#### 단어의 유사도 = 벡터의 내적

신기한 점은, 잘 학습된 임베딩에서는 **비슷한 의미의 단어들은 비슷한 벡터**를 갖는다는 것입니다.

```
고양이 · 강아지 = 0.85   ← 둘 다 동물, 매우 유사
고양이 · 자동차 = 0.05   ← 거의 관련 없음
```

#### 🚀 LLMOps 실무 연결

**ChatGPT, Claude 같은 LLM**도 모든 텍스트를 벡터로 바꿔서 처리합니다.
- **벡터 데이터베이스(Vector DB)**: Pinecone, Weaviate, Chroma 등은 모두 "벡터들의 내적/거리"를 빠르게 계산하는 시스템입니다.
- **RAG (검색 증강 생성)**: 사용자 질문을 벡터로 바꾸고, 데이터베이스의 문서 벡터와 내적을 계산해서 "가장 비슷한 문서"를 찾습니다.
- **시맨틱 검색**: 키워드가 아닌 "의미"로 검색이 가능한 이유 = 단어를 벡터로 표현하고 내적을 계산하기 때문.

> 💡 **핵심**: 벡터의 내적을 이해하면, 현대 AI 검색 시스템의 핵심 원리를 이해한 것입니다.

### 1.7 벡터의 크기(Norm)와 거리

#### 벡터의 크기: 화살표의 길이

벡터 `[3, 4]`의 크기는?  → **피타고라스 정리**로 구합니다.

$$
\|v\| = \sqrt{3^2 + 4^2} = \sqrt{9 + 16} = \sqrt{25} = 5
$$

이걸 **L2 norm** 또는 **유클리드 거리(Euclidean distance)** 라고 합니다.

#### 두 점 사이의 거리

두 벡터 사이의 거리도 같은 방법으로 구합니다.
- 점 A: `[1, 2]`
- 점 B: `[4, 6]`
- 거리: $\sqrt{(4-1)^2 + (6-2)^2} = \sqrt{9 + 16} = 5$

#### 머신러닝에서의 활용
- **K-Nearest Neighbors (KNN)**: 가장 가까운 이웃 K개를 찾는 알고리즘 → 벡터 거리 사용
- **클러스터링(K-means)**: 데이터를 그룹으로 묶을 때 → 거리 계산
- **추천 시스템**: 비슷한 사용자/상품 찾기 → 거리 계산

### 1.8 챕터 요약

✅ 벡터는 **숫자들의 묶음**이자 **방향 있는 화살표**.
✅ 벡터끼리 더하면 화살표가 이어 붙고, 스칼라를 곱하면 길이가 변한다.
✅ **내적**은 두 벡터의 닮은 정도를 측정한다.
✅ 머신러닝은 **모든 데이터를 벡터로** 변환해서 다룬다.
✅ 단어 임베딩, 벡터 DB, RAG의 핵심 원리는 모두 **벡터의 내적**이다.

## Chapter 2. 행렬(Matrix) — 숫자들의 표

### 2.1 행렬이 뭔가요?

벡터가 숫자를 **한 줄**로 늘어놓은 것이라면, 행렬은 숫자를 **격자(표)** 로 늘어놓은 것입니다.

```
벡터:   [1, 2, 3]

행렬:   [1, 2, 3]
        [4, 5, 6]
        [7, 8, 9]
```

이 행렬은 **3행 3열**이라서 "3×3 행렬"이라고 부릅니다.

### 2.2 가장 익숙한 비유: 엑셀 표

행렬을 처음 보면 어렵게 느껴지지만, 사실 우리는 매일 행렬을 사용합니다. **엑셀 시트**가 바로 행렬입니다.

| | 키(cm) | 몸무게(kg) | 나이 |
|---|---|---|---|
| 사람1 | 170 | 65 | 30 |
| 사람2 | 165 | 55 | 25 |
| 사람3 | 180 | 80 | 35 |

이걸 행렬로 쓰면:

```
[170, 65, 30]
[165, 55, 25]
[180, 80, 35]
```

### 2.3 머신러닝에서 행렬은 어디에?

#### 🖼️ 이미지 = 행렬
흑백 사진은 픽셀들의 밝기 값으로 된 행렬입니다.

```
[0,   0,   255, 255, 0  ]
[0,   255, 255, 255, 255]
[255, 255, 255, 255, 255]
[0,   255, 255, 255, 0  ]
[0,   0,   255, 0,   0  ]
```
(0 = 검정, 255 = 흰색)

컬러 사진은 R, G, B 세 개의 행렬이 겹쳐진 것 = **텐서(Tensor)**.

#### 📊 데이터셋 = 행렬
머신러닝의 학습 데이터는 보통 행렬로 표현됩니다.
- **행(row)**: 하나의 데이터 샘플 (예: 한 명의 고객)
- **열(column)**: 특징(feature) (예: 나이, 소득, 구매 횟수)

#### 🧠 신경망의 가중치 = 행렬
신경망의 각 층은 **가중치 행렬**을 가지고 있습니다. 학습한다는 건 이 행렬의 숫자들을 조금씩 바꾸는 것입니다.

### 2.4 행렬 연산

#### 덧셈: 같은 자리끼리 더하기

```
[1, 2]   [5, 6]   [6,  8]
[3, 4] + [7, 8] = [10, 12]
```

같은 크기의 행렬끼리만 더할 수 있습니다.

#### 스칼라 곱: 모든 칸에 곱하기

```
       [1, 2]   [2, 4]
   2 × [3, 4] = [6, 8]
```

#### 행렬-벡터 곱: 가장 중요한 연산!

이건 머신러닝의 **심장**과도 같습니다.

```
[1, 2]   [10]   [1×10 + 2×20]   [50]
[3, 4] × [20] = [3×10 + 4×20] = [110]
```

**규칙**: 행렬의 각 행과 벡터의 내적을 계산하면 됩니다.

#### 행렬-행렬 곱

```
[1, 2]   [5, 6]   [1×5+2×7, 1×6+2×8]   [19, 22]
[3, 4] × [7, 8] = [3×5+4×7, 3×6+4×8] = [43, 50]
```

손으로 계산할 일은 거의 없습니다. NumPy가 알아서 해줍니다.
```python
import numpy as np
A @ B   # 행렬 곱
```

### 2.5 핵심 직관: 행렬은 "변환기"입니다

행렬을 가장 잘 이해하는 방법은 **"공간을 변형하는 기계"** 로 보는 것입니다.

벡터에 행렬을 곱하면, 그 벡터가 **늘어나거나, 회전하거나, 찌그러집니다**.

#### 예시 1: 회전 행렬

```
       [cos90°, -sin90°]   [0, -1]
   R = [sin90°,  cos90°] = [1,  0]
```

이 행렬을 어떤 벡터에 곱하면, 그 벡터가 **90도 회전**합니다.

```
        [0, -1]   [1]   [0]
   R × [1,  0] × [0] = [1]   ← (1,0)이 (0,1)로 회전!
```

#### 예시 2: 늘리기 행렬

```
   S = [2, 0]
       [0, 2]
```

이 행렬은 모든 벡터를 **2배 크게** 만듭니다.

#### 예시 3: 찌그러뜨리기 행렬

```
   T = [1, 0.5]
       [0, 1  ]
```

이 행렬은 도형을 **기울어진 평행사변형**으로 변형시킵니다.

### 2.6 머신러닝과의 연결

#### 🧠 신경망의 한 층 = 행렬 곱 + 활성화 함수

신경망의 한 층은 이런 식으로 작동합니다.

$$
y = f(Wx + b)
$$

- $x$: 입력 벡터
- $W$: **가중치 행렬** (이게 학습되는 부분!)
- $b$: 편향(bias) 벡터
- $f$: 활성화 함수 (ReLU, sigmoid 등)
- $y$: 출력 벡터

쉽게 말하면:
> "입력 벡터에 행렬을 곱하고, 약간의 보정값(bias)을 더한 뒤, 변환 함수를 통과시킨다"

이게 끝입니다. 신경망은 이 작업을 수십~수백 번 반복하는 것일 뿐입니다.

#### 🚀 LLMOps 실무 연결: Transformer와 행렬

GPT, Claude 같은 모델의 핵심인 **Transformer** 도 결국 거대한 행렬 곱셈의 연속입니다.
- **Attention 메커니즘**: Query, Key, Value 세 개의 행렬을 곱하는 연산
- **GPU가 빠른 이유**: GPU는 **행렬 곱셈을 병렬로** 매우 빠르게 처리하도록 설계됨
- **모델 크기**: "70B 파라미터 모델"이라는 건 가중치 행렬 안에 700억 개의 숫자가 있다는 뜻

> 💡 **MLOps 팁**: 모델 추론 비용의 대부분은 **행렬 곱셈**에 들어갑니다. 그래서 양자화(quantization), 행렬 분해(matrix factorization) 같은 최적화 기법이 중요합니다.

### 2.7 특수한 행렬들

#### 단위 행렬(Identity Matrix)

대각선만 1이고 나머지는 0인 행렬. **곱해도 아무것도 안 변하는** 행렬입니다.

```
        [1, 0, 0]
   I = [0, 1, 0]
        [0, 0, 1]
```

숫자 세계의 "1"과 같은 역할.  $I \times v = v$

#### 전치 행렬(Transpose)

행과 열을 바꾼 행렬. 행렬 $A$의 전치는 $A^T$로 표기.

```
       [1, 2, 3]              [1, 4]
   A = [4, 5, 6]   →   A^T = [2, 5]
                              [3, 6]
```

#### 역행렬(Inverse)

곱하면 단위 행렬이 되는 행렬. $A \times A^{-1} = I$.

숫자의 "역수"와 같은 개념. (3의 역수는 1/3, 곱하면 1)

머신러닝에서는 직접 역행렬을 구하는 일은 드물지만, 선형 회귀의 닫힌 해(closed-form solution)에서 등장합니다.

### 2.8 챕터 요약

✅ 행렬은 **숫자들의 표**, 벡터들의 묶음.
✅ 이미지, 데이터셋, 신경망 가중치 — 머신러닝의 거의 모든 데이터는 행렬로 표현된다.
✅ 행렬 곱셈은 **공간을 변형하는 연산**.
✅ 신경망은 본질적으로 **행렬 곱 + 비선형 함수의 반복**.
✅ GPU가 머신러닝에 강한 이유 = **행렬 연산을 병렬로** 처리하기 때문.

## Chapter 3. 고유값과 고유벡터 — 행렬의 DNA

### 3.1 들어가며: 가장 어렵게 느껴지지만 가장 강력한 개념

"고유값(Eigenvalue)", "고유벡터(Eigenvector)" — 이름부터 뭔가 무섭습니다.

하지만 직관만 잡으면 의외로 단순합니다. 천천히 가봅시다.

### 3.2 핵심 질문: "어떤 벡터는 행렬을 곱해도 방향이 안 변한다"

앞서 배웠듯, 행렬은 벡터를 **변형(회전, 늘리기, 찌그러뜨리기)** 하는 기계였습니다.

그런데 신기하게도, **어떤 특별한 벡터들은 행렬을 곱해도 방향이 변하지 않습니다.** 단지 길이만 늘어나거나 줄어들 뿐입니다.

이런 특별한 벡터를 **고유벡터(eigenvector)** 라고 하고, 길이가 몇 배로 변했는지를 **고유값(eigenvalue)** 이라고 합니다.

수식으로 쓰면:

$$
A v = \lambda v
$$

- $A$: 행렬
- $v$: 고유벡터 (방향이 안 변한 특별한 벡터)
- $\lambda$ (람다): 고유값 (길이 변화 비율)

해석: "$A$를 $v$에 곱했더니, $v$가 단지 $\lambda$ 배로 늘어났을 뿐이다."

### 3.3 비유로 이해하기: 회전하는 지구

지구가 자전한다고 생각해봅시다. 지구의 모든 점이 회전하고 있죠.

그런데 **두 점만은 움직이지 않습니다**: **북극과 남극**.

이 북극↔남극을 잇는 축이 바로 "회전 변환의 고유벡터"입니다. 회전을 시켜도 이 축의 방향은 그대로 유지되니까요.

### 3.4 시각적으로 보기

행렬 $A = \begin{bmatrix} 3 & 1 \\ 0 & 2 \end{bmatrix}$가 있다고 합시다.

이 행렬은 평면을 비스듬히 늘립니다. 대부분의 벡터는 곱하면 방향이 바뀝니다.

```
원래 벡터 →   행렬 A 곱하기  →   변형된 벡터
[1, 1]                        [4, 2]   ← 방향 바뀜
[2, 1]                        [7, 2]   ← 방향 바뀜
```

하지만 어떤 두 벡터는 방향이 안 변합니다:
- $v_1 = [1, 0]$ → $Av_1 = [3, 0]$ (3배로 늘어남, 방향 같음)  → 고유값 = 3
- $v_2 = [1, -1]$ → $Av_2 = [2, -2]$ (2배로 늘어남, 방향 같음) → 고유값 = 2

이 행렬의 **DNA**는 이 두 고유벡터와 고유값으로 표현됩니다.

### 3.5 고유값 분해(Eigenvalue Decomposition)

복잡한 행렬을 **고유값과 고유벡터들의 조합**으로 쪼개는 것을 **고유값 분해**라고 합니다.

$$
A = V \Lambda V^{-1}
$$

- $V$: 고유벡터들을 모은 행렬
- $\Lambda$: 고유값들을 대각선에 배치한 대각 행렬

쉽게 말해, "복잡한 행렬을 그 행렬의 본질적 특성(고유값/고유벡터)으로 분해하는 것"입니다.

> 비유: **사람의 DNA 분석**같은 것입니다. 복잡한 사람도 결국 DNA로 분해해서 보면 본질이 드러나죠.

### 3.6 머신러닝과의 연결: 차원 축소(PCA)

#### 문제 상황

머신러닝에서 자주 만나는 문제:
- 데이터의 **차원(특징의 개수)이 너무 많다**.
- 예: 이미지 한 장이 1000×1000 픽셀이면 100만 차원.
- 차원이 너무 많으면 → 학습이 느리고, 과적합 위험 증가, 시각화 불가능.

#### 해결책: 주성분 분석(PCA, Principal Component Analysis)

PCA는 **고유값 분해를 활용한 가장 유명한 알고리즘**입니다.

**아이디어**:
> "데이터에서 가장 중요한(분산이 큰) 방향들만 골라서, 그 방향들로만 데이터를 표현하자!"

**과정**:
1. 데이터의 **공분산 행렬**을 만든다.
2. 그 행렬의 **고유벡터와 고유값**을 구한다.
3. **고유값이 큰 순서**로 고유벡터들을 정렬한다. (큰 고유값 = 데이터에서 더 중요한 방향)
4. 상위 k개의 고유벡터만 골라서 데이터를 그 방향으로 **투영(projection)** 한다.

#### 시각화

100차원 데이터가 있다고 해봅시다. 이 데이터의 진짜 정보는 사실 2~3개 축에 몰려 있을 수 있습니다.

```
원본 데이터 (3D)         PCA 후 (2D)
   ●                    
  ● ●                    ●  ●
 ●   ●     →             ●●●●●  ← 가장 중요한 2개 축으로 투영
●     ●                  ●  ●
 ●   ●                   
```

이렇게 차원을 줄여도 데이터의 핵심 패턴은 보존됩니다.

### 3.7 실무 활용 예시

#### 1) 데이터 시각화
- 100차원 고객 데이터를 2D로 줄여서 산점도로 그린다.
- 비슷한 고객 그룹이 시각적으로 보인다.

#### 2) 노이즈 제거
- 작은 고유값에 해당하는 차원은 보통 노이즈.
- 그 차원들을 버리면 노이즈가 줄어든다.

#### 3) 이미지 압축
- 얼굴 사진들을 PCA로 압축 → "Eigenfaces".
- 100×100 = 10,000차원 얼굴을 100차원 정도로 압축 가능.

#### 4) 추천 시스템
- 사용자-아이템 행렬을 행렬 분해(Matrix Factorization)로 쪼갬.
- Netflix, YouTube 추천 시스템의 고전적 기법.

### 3.8 LLMOps/MLOps 실무 연결

#### 🚀 임베딩 차원 축소
LLM에서 만들어진 임베딩(예: 1536차원)을 그대로 저장/검색하면 비쌉니다.
- **PCA로 차원 축소** → 256차원으로 줄여서 저장 → 비용 절감, 속도 향상.
- 단, 정보 손실 vs 효율 사이의 트레이드오프 고려 필요.

#### 🚀 모델 압축 (LoRA의 직관)
**LoRA(Low-Rank Adaptation)** 는 LLM 미세조정의 표준 기법인데, 그 핵심 아이디어는:
> "거대한 가중치 행렬의 변화량은 사실 **저차원(low-rank)** 으로 표현 가능하다."

이는 고유값 분해와 직결된 개념입니다. 큰 고유값을 가진 몇 개의 방향만 학습하면 충분하다는 것이죠. **수십억 파라미터 모델을 수백만 파라미터만 학습해서 미세조정**할 수 있는 비결입니다.

#### 🚀 SVD (특이값 분해)
SVD는 고유값 분해의 일반화 버전입니다. 정사각형이 아닌 행렬에도 적용 가능.
- 추천 시스템 (협업 필터링)
- 토픽 모델링 (LSA, Latent Semantic Analysis)
- 모델 양자화 및 압축

### 3.9 챕터 요약

✅ **고유벡터**: 행렬을 곱해도 방향이 안 변하는 특별한 벡터.
✅ **고유값**: 그 벡터의 길이가 몇 배로 변했는지.
✅ 고유값 분해는 행렬의 **본질적 특성을 추출**하는 도구.
✅ **PCA(차원 축소)** 의 핵심 원리이며, 데이터 시각화/노이즈 제거/압축에 활용.
✅ LoRA, 임베딩 압축 등 **현대 LLMOps의 효율화 기법**의 수학적 기반.

## Chapter 4. 확률의 기초 — 불확실성을 숫자로

### 4.1 왜 머신러닝에 확률이 필요한가?

머신러닝 모델은 **절대 100% 확신하지 않습니다**. 예를 들어:
- 이메일이 스팸일 확률: **87%**
- 이 사진이 고양이일 확률: **94%**
- 내일 비 올 확률: **30%**

이렇게 **불확실성을 다루는 언어**가 바로 확률입니다.

### 4.2 확률이란 무엇인가?

확률은 **0과 1 사이의 숫자**입니다.
- **0**: 절대 일어나지 않음
- **1**: 무조건 일어남
- **0.5**: 반반 확률

#### 일상 예시
- 동전을 던져 앞면이 나올 확률 = **0.5** (50%)
- 주사위를 굴려 6이 나올 확률 = **1/6 ≈ 0.167** (16.7%)
- 내일 해가 뜰 확률 ≈ **1.0** (100%)

#### 수식

$$
P(\text{사건}) = \frac{\text{원하는 경우의 수}}{\text{전체 경우의 수}}
$$

주사위로 짝수가 나올 확률은?
- 원하는 경우(2, 4, 6) = 3
- 전체 경우(1~6) = 6
- 확률 = 3/6 = **0.5**

### 4.3 확률의 기본 규칙

#### 규칙 1: 모든 확률을 더하면 1

주사위 한 개를 굴렸을 때:
- P(1) + P(2) + P(3) + P(4) + P(5) + P(6) = 1/6 × 6 = **1**

#### 규칙 2: "또는(OR)" — 더하기 (배타적인 경우)

주사위에서 1 또는 2가 나올 확률 = 1/6 + 1/6 = **2/6**

#### 규칙 3: "그리고(AND)" — 곱하기 (독립인 경우)

동전을 두 번 던져 둘 다 앞면일 확률 = 1/2 × 1/2 = **1/4**

### 4.4 조건부 확률 — "만약 ~라면"

이건 머신러닝의 핵심 개념입니다.

**조건부 확률**: "사건 B가 일어났다는 정보가 있을 때, 사건 A가 일어날 확률"

기호: $P(A | B)$ — "B given A"가 아니라 "**A given B**"라고 읽음.

#### 예시

병원 데이터:
- 전체 환자 1000명 중
- 감기 환자 100명, 그 중 기침하는 사람 80명
- 비감기 환자 900명, 그 중 기침하는 사람 90명

질문: "어떤 사람이 기침을 한다. 이 사람이 감기일 확률은?"

이게 바로 **조건부 확률** $P(\text{감기} | \text{기침})$ 입니다.

- 기침하는 전체 사람 = 80 + 90 = 170명
- 그 중 감기인 사람 = 80명
- 따라서 P(감기 | 기침) = 80/170 ≈ **47%**

#### 공식

$$
P(A | B) = \frac{P(A \cap B)}{P(B)}
$$

"A와 B가 동시에 일어날 확률을 B가 일어날 확률로 나눈 것"

### 4.5 독립과 종속

#### 독립 (Independent)
두 사건이 서로 영향을 안 미치는 경우.

예: 동전 두 번 던지기. 첫 번째 결과가 두 번째에 영향 안 줌.

수학적으로: $P(A | B) = P(A)$

#### 종속 (Dependent)
한 사건이 다른 사건에 영향을 미치는 경우.

예: 카드 한 장 뽑고 다시 안 넣고 또 뽑기. 두 번째 카드는 첫 번째 결과에 영향 받음.

### 4.6 머신러닝과의 연결

#### 분류 모델의 출력
이미지 분류 모델이 사진을 보고 출력하는 것:
- P(고양이 | 이 이미지) = 0.92
- P(강아지 | 이 이미지) = 0.05
- P(토끼 | 이 이미지) = 0.03

→ 이건 모두 **조건부 확률**입니다. "이미지가 주어졌을 때 각 클래스일 확률".

#### LLM의 다음 토큰 예측
ChatGPT가 다음 단어를 생성할 때, 사실은 이런 계산을 하고 있습니다:

> "지금까지의 문맥이 주어졌을 때, 다음 단어가 X일 확률"
> $P(\text{다음 단어} | \text{이전 단어들})$

가장 확률이 높은 단어를 선택하거나, 확률에 따라 샘플링합니다.

### 4.7 챕터 요약

✅ 확률은 **0~1 사이의 숫자**로 불확실성을 표현.
✅ **조건부 확률** $P(A|B)$: B라는 정보가 있을 때 A가 일어날 확률.
✅ ML 분류 모델 출력 = 조건부 확률.
✅ LLM의 텍스트 생성 = 다음 토큰의 조건부 확률 계산.

## Chapter 5. 확률 분포 — 데이터의 모양

### 5.1 분포가 뭔가요?

**확률 분포**는 "어떤 값이 얼마나 자주 나타나는지"를 보여주는 그림입니다.

예를 들어, 한국 성인 남성 100만 명의 키를 측정했다고 합시다. 그 결과를 그래프로 그리면:

```
빈도
 ↑
 |          ●●●●●
 |        ●●     ●●
 |      ●●         ●●
 |    ●●             ●●
 |  ●●                 ●●
 |●●                     ●●
 +─────────────────────────→ 키(cm)
  150    165    175    190
```

가운데가 가장 많고, 양 끝으로 갈수록 적어지는 **종 모양**.

### 5.2 정규분포(Normal/Gaussian Distribution) — 가장 유명한 분포

위 그림이 바로 **정규분포** 입니다. 이름은 어렵지만, **자연계에서 가장 자주 보이는 분포**예요.

#### 어디에 나타나나?
- 사람의 키, 몸무게
- 시험 점수
- 측정 오차
- 주가 변동
- 기계의 부품 크기

**거의 모든 자연 현상**이 정규분포를 따릅니다. (중심극한정리 때문)

#### 두 가지 숫자로 결정됨

정규분포는 단 두 개의 숫자로 완전히 결정됩니다.

1. **평균(μ, 뮤)**: 분포의 **중심 위치** (산의 꼭대기 위치)
2. **표준편차(σ, 시그마)**: 분포의 **퍼진 정도** (산이 넓은가, 좁은가)

```
좁은 분포(σ 작음):    뾰족한 산
넓은 분포(σ 큼):      완만한 언덕
```

#### 68-95-99.7 법칙

정규분포의 유명한 성질:
- 평균에서 ±1σ 안에 데이터의 **68%**
- 평균에서 ±2σ 안에 데이터의 **95%**
- 평균에서 ±3σ 안에 데이터의 **99.7%**

> 💡 통계에서 "p < 0.05"라는 말이 나오는 이유 = 95% 신뢰구간 바깥의 5% 영역.

### 5.3 다른 중요한 분포들

#### 베르누이 분포(Bernoulli)
**이거냐 저거냐 — 둘 중 하나**의 결과.

예: 동전 던지기 (앞/뒤), 광고 클릭 여부 (클릭/안 함), 스팸 여부.

분포의 모양:
```
   p%        (1-p)%
    ↑          ↑
    █          █
    █          █
   성공       실패
```

#### 이항 분포(Binomial)
베르누이를 **n번 반복**했을 때 성공 횟수의 분포.

예: 동전을 100번 던졌을 때 앞면이 몇 번 나올지.

#### 균일 분포(Uniform)
모든 값이 똑같이 나올 확률.

예: 주사위 (1~6이 모두 1/6 확률).

```
빈도
 ↑
 |  ████████████
 |  ████████████   ← 평평함
 |  ████████████
 +───────────────→
   1  2  3  4  5  6
```

#### 푸아송 분포(Poisson)
"단위 시간/공간 안에 사건이 몇 번 일어날까?"

예: 콜센터에 1시간 동안 걸려오는 전화 횟수, 웹사이트의 분당 방문자 수.

### 5.4 머신러닝과의 연결

#### 1) 모델의 가정

많은 머신러닝 모델은 데이터가 특정 분포를 따른다고 **가정**합니다.
- 선형 회귀: 오차가 정규분포를 따른다고 가정.
- 나이브 베이즈: 특정 분포(가우시안, 베르누이 등)를 가정.

#### 2) 데이터 이해
- 데이터가 정규분포를 따르는지 보면, 어떤 모델을 써야 할지 감이 옵니다.
- 비정상적으로 한쪽으로 치우친(skewed) 분포 → 로그 변환 등이 필요할 수 있음.

#### 3) 이상치(Outlier) 탐지
- 정규분포에서 ±3σ를 벗어난 값 = 거의 일어날 확률 0.3% → 이상치로 판단.

### 5.5 LLM에서의 샘플링

#### 토큰 확률 분포

LLM은 다음 단어로 가능한 모든 후보에 대해 확률 분포를 만듭니다.

```
"안녕하세요, 저는" 다음에 올 단어:
- "오늘"   → 0.25
- "정말"   → 0.15
- "지금"   → 0.10
- "AI"    → 0.08
- ...
```

#### 샘플링 전략 (Temperature)

이 분포에서 어떻게 단어를 고르느냐가 LLM의 **창의성**을 결정합니다.

- **Greedy (T=0)**: 가장 확률 높은 단어만 선택 → 일관되지만 지루함.
- **High temperature (T=1.5)**: 분포를 평평하게 만들어 다양성↑ → 창의적이지만 헛소리할 수 있음.
- **Low temperature (T=0.3)**: 분포를 뾰족하게 만들어 안정적 → 사실적 답변에 좋음.

#### Top-k, Top-p 샘플링
- **Top-k**: 상위 k개의 단어 중에서만 샘플링.
- **Top-p (nucleus)**: 누적 확률이 p가 될 때까지의 단어들 중에서 샘플링.

> 💡 **LLMOps 실무 팁**: 코드 생성에는 낮은 temperature(0.0~0.3), 창의적 글쓰기에는 높은 temperature(0.7~1.0)를 쓰는 게 일반적.

### 5.6 챕터 요약

✅ **확률 분포**는 값들이 얼마나 자주 나타나는지 보여주는 그림.
✅ **정규분포**: 평균과 표준편차로 결정되는 종 모양 곡선. 자연계에 가장 흔함.
✅ 머신러닝 모델은 종종 데이터의 분포를 가정함.
✅ **LLM의 텍스트 생성** = 분포에서 토큰을 샘플링하는 것.
✅ Temperature, Top-k, Top-p 같은 LLM 파라미터는 **샘플링 분포를 조절**하는 역할.

## Chapter 6. 기댓값과 분산 — 평균과 흩어짐

### 6.1 기댓값(Expected Value) — "평균적으로 얼마인가?"

**기댓값**은 한 마디로 **평균값**입니다. 단, "확률이 다 다를 때의 가중 평균"입니다.

#### 일상 예시: 복권

복권을 한 장 샀을 때 평균적으로 얼마를 벌 수 있을까?

| 경우 | 확률 | 상금 |
|---|---|---|
| 1등 당첨 | 0.0001 | 100,000,000원 |
| 꽝 | 0.9999 | 0원 |

기댓값 = (0.0001 × 1억) + (0.9999 × 0)
= **10,000원**

복권 한 장 가격이 5,000원이면 → 살 만한 가치 있음(?).
가격이 15,000원이면 → 사면 손해.

#### 공식

$$
E[X] = \sum_i p_i \cdot x_i
$$

"각 결과($x_i$)에 그 확률($p_i$)을 곱한 다음, 다 더하기."

### 6.2 분산(Variance) — "얼마나 흩어져 있나?"

**분산**은 데이터가 평균에서 얼마나 떨어져 있는지를 나타냅니다.

#### 비유

두 반의 시험 점수:
- A반: 60, 65, 70, 75, 80 → 평균 70, **고른 점수**
- B반: 30, 50, 70, 90, 110 → 평균 70, **편차 큼**

평균은 같지만 **흩어진 정도**가 다릅니다. 이걸 측정하는 것이 분산.

#### 공식

$$
Var(X) = E[(X - E[X])^2]
$$

"각 값과 평균의 차이를 제곱한 것의 평균"

차이를 그냥 더하면 양수와 음수가 상쇄되니, **제곱**해서 양수로 만든 후 평균.

#### 표준편차(Standard Deviation)

분산은 단위가 제곱이라 직관적이지 않음. **분산의 제곱근**이 표준편차.

- 키 데이터의 분산: 25 cm²  ← 이상함
- 키 데이터의 표준편차: 5 cm  ← 직관적!

### 6.3 머신러닝과의 연결

#### 1) 손실 함수 = 기댓값

머신러닝의 **손실 함수(Loss Function)** 는 본질적으로 "오차의 기댓값"입니다.

평균제곱오차(MSE):
$$
\text{MSE} = \frac{1}{N}\sum_{i=1}^N (y_i - \hat{y}_i)^2
$$

= "실제값과 예측값의 차이의 제곱"의 평균
= "오차 제곱의 기댓값"

→ **분산과 똑같은 형태**!

#### 2) Bias-Variance Tradeoff (편향-분산 균형)

머신러닝 모델의 오차는 두 가지로 쪼개집니다:

- **편향(Bias)**: 모델이 평균적으로 얼마나 틀리는가? (정확도)
- **분산(Variance)**: 데이터가 바뀌면 모델이 얼마나 출렁이는가? (안정성)

```
    ┌──────────────────────────────┐
    │  과소적합   ↔   과대적합     │
    │  (Underfit)    (Overfit)    │
    │  편향 큼       분산 큼       │
    │  분산 작음     편향 작음     │
    └──────────────────────────────┘
```

#### 3) 강화학습의 Value Function

강화학습에서 "어떤 상태의 가치"는 **미래 보상의 기댓값**으로 정의됩니다.

$$
V(s) = E[\text{미래 받을 보상의 합} | \text{현재 상태 } s]
$$

#### 4) MLOps 모니터링

배포된 모델의 성능을 모니터링할 때 평균뿐 아니라 **분산도 추적**해야 합니다.
- 평균 응답 시간이 100ms여도, 분산이 크면 사용자 경험이 들쭉날쭉.
- p50, p95, p99 latency를 모니터링하는 이유.

### 6.4 챕터 요약

✅ **기댓값** = 가중 평균 = "평균적으로 얼마"
✅ **분산** = "평균에서 얼마나 흩어져 있나"
✅ **표준편차** = 분산의 제곱근, 직관적인 단위
✅ ML 손실 함수, Bias-Variance Tradeoff, 강화학습의 Value Function 모두 기댓값 개념.

## Chapter 7. 베이즈 정리 — 새로운 정보로 믿음 갱신하기

### 7.1 베이즈 정리는 왜 중요한가?

**베이즈 정리(Bayes' Theorem)** 는 18세기 토머스 베이즈가 만든 공식입니다. 한 줄로 요약하면:

> "새로운 증거가 주어졌을 때, 기존의 믿음을 어떻게 갱신할 것인가?"

이건 인간이 추론하는 방식 그 자체이며, **현대 AI의 가장 중요한 사고방식** 중 하나입니다.

### 7.2 가장 유명한 예시: 의료 진단

#### 시나리오
- 어떤 희귀병이 있다. 전체 인구의 **1%** 만 걸림.
- 진단 검사가 있다. 정확도 **99%** (병이 있으면 99% 확률로 양성, 병이 없으면 99% 확률로 음성).
- 당신이 검사를 받았는데 **양성**이 나왔다.

**질문**: 당신이 정말로 병에 걸렸을 확률은?

대부분의 사람들은 **"99%"** 라고 대답합니다.

하지만 정답은 **약 50%** 입니다. 충격이죠?

#### 왜 그럴까?

10,000명을 검사한다고 가정해봅시다.

- **실제 환자**: 100명 (1%)
  - 양성으로 정확히 진단: 99명
  - 음성으로 잘못 진단: 1명
- **건강한 사람**: 9,900명 (99%)
  - 음성으로 정확히 진단: 9,801명
  - 양성으로 **잘못** 진단: **99명** ← 위양성!

양성으로 나온 사람은 총 **99 + 99 = 198명**.
그 중 진짜 환자는 **99명**.

따라서 양성 판정을 받았을 때 진짜 환자일 확률 = 99 / 198 = **50%**.

#### 핵심 통찰

희귀한 병의 경우, **검사가 매우 정확해도** "양성 = 환자"가 아닐 수 있습니다. **사전 확률(병의 희귀성)** 이 매우 낮기 때문입니다.

### 7.3 베이즈 정리 공식

$$
P(A | B) = \frac{P(B | A) \cdot P(A)}{P(B)}
$$

용어들을 풀어 보겠습니다.

| 기호 | 이름 | 의미 |
|---|---|---|
| $P(A)$ | **사전 확률(Prior)** | 검사 전, 병에 걸렸을 확률 (1%) |
| $P(B \| A)$ | **우도(Likelihood)** | 병이 있을 때 양성이 나올 확률 (99%) |
| $P(B)$ | **증거(Evidence)** | 양성이 나올 전체 확률 |
| $P(A \| B)$ | **사후 확률(Posterior)** | 양성일 때 병일 확률 — 이게 우리가 알고 싶은 것! |

#### 직관적 해석

> "사후 확률 = (우도 × 사전 확률) / 증거"

"새로운 증거(B)를 봤다. 이걸 바탕으로 기존 믿음(prior)을 어떻게 업데이트할까?"

### 7.4 일상에서의 베이즈적 사고

베이즈 정리는 사실 **상식** 입니다. 우리는 이미 매일 베이즈적으로 추론합니다.

#### 예시 1: 발자국 소리

밤에 집에 혼자 있는데 발자국 소리가 들립니다.
- **사전 확률**: 우리 집에 도둑이 들 확률 → 매우 낮음 (0.001).
- **우도**: 도둑이 있다면 발자국 소리가 날 확률 → 높음 (0.9).
- **증거**: 발자국 소리는 가족, 바람, 고양이 등 이유 다양.

→ 발자국 소리가 들린다고 해서 도둑일 확률은 그리 높지 않다고 결론.

#### 예시 2: 친구의 사과

평소 약속을 잘 지키는 친구가 처음으로 약속에 늦었습니다.
- 사전 확률: 이 친구는 보통 시간을 잘 지킴.
- 새로운 증거: 한 번 늦었음.
- 사후 확률: 여전히 신뢰할 만 + 약간의 우려.

평소 자주 늦는 친구라면? 한 번 더 늦었을 때 신뢰가 크게 떨어지죠.

→ 베이즈는 **새로운 정보가 기존 믿음을 얼마나 바꿔야 하는지** 알려줍니다.

### 7.5 머신러닝과의 연결

#### 1) 나이브 베이즈 분류기 (Naive Bayes Classifier)

**스팸 메일 필터**의 고전적 알고리즘. 메일에 있는 단어들을 보고 스팸인지 판단.

$$
P(\text{스팸} | \text{단어들}) = \frac{P(\text{단어들} | \text{스팸}) \cdot P(\text{스팸})}{P(\text{단어들})}
$$

- "무료", "당첨", "광고" 같은 단어가 있으면 → 스팸일 확률 ↑
- 평범한 단어만 있으면 → 정상일 확률 ↑

#### 2) 베이지안 머신러닝

기존 머신러닝: "최적의 파라미터 하나"를 찾음.

베이지안 머신러닝: "파라미터의 **확률 분포**"를 다룸.
- 단순한 점 추정이 아닌, **불확실성까지 표현**.
- 의료, 자율주행 등 **신뢰도가 중요한 분야**에서 활용.

#### 3) A/B 테스트

새로운 기능 A vs B 중 어떤 것이 좋을지 테스트할 때:
- **베이지안 A/B 테스트**: 사전 확률 + 실험 데이터 → "A가 B보다 좋을 확률 87%"
- 전통적 통계보다 직관적이고 실무 의사결정에 유리.

#### 4) 추천 시스템

"사용자가 X 영화를 봤다. 그렇다면 Y를 좋아할 확률은?"
- 베이즈 추론으로 사용자 선호를 갱신.

### 7.6 LLMOps/MLOps에서의 베이즈

#### 🚀 LLM 평가의 함정

GPT-4가 어떤 작업을 **95% 정확도**로 수행한다고 합시다. 이게 좋은 걸까요?

- 그 작업이 **흔한 경우**라면(예: 일반적 질문응답) → 95%면 충분.
- 그 작업이 **희귀한 의료/법률 케이스**라면 → 95%여도 위험할 수 있음 (베이즈 정리!).

#### 🚀 모델 모니터링과 알람

배포된 모델에 이상 감지 알람을 설정할 때:
- 평소 에러율 0.1% → 갑자기 1%로 상승.
- 이게 진짜 문제인가, 일시적 노이즈인가?
- 베이즈적 접근: 사전 분포를 가정하고, 새 데이터로 업데이트해서 "진짜 문제일 확률"을 계산.

#### 🚀 RAG 시스템의 신뢰도

RAG에서 검색된 문서가 정말 답변에 적절한지:
- **사전 확률**: 일반적으로 검색 결과의 정확도.
- **우도**: 이 질문 유형에서 정확도가 어땠는지.
- → 답변의 신뢰도를 동적으로 평가.

### 7.7 챕터 요약

✅ **베이즈 정리**: 새로운 증거로 기존 믿음을 갱신하는 공식.
✅ 사전 확률 × 우도 / 증거 = 사후 확률.
✅ **희귀한 사건**에서는 검사가 정확해도 오해하기 쉽다 (사전 확률의 중요성).
✅ 스팸 필터, 베이지안 ML, A/B 테스트, 추천 시스템의 핵심.
✅ 모델 모니터링과 신뢰도 평가에 응용 가능.

## Chapter 8. 미분 기초 — 변화의 속도

### 8.1 미분이 왜 필요한가?

머신러닝에서 모델을 "학습한다"는 것은 결국:

> **"손실(loss)을 가장 작게 만드는 파라미터를 찾는 것"**

산을 내려가서 가장 낮은 골짜기를 찾는 것과 같습니다. 그런데 어느 방향으로 내려가야 할지 알려면? **그 지점의 기울기(미분)** 를 알아야 합니다.

미분은 **"기울기를 구하는 도구"** 입니다.

### 8.2 미분이란? — 한 마디로 "기울기"

#### 직선의 기울기

직선 $y = 2x + 1$의 기울기는?
- $x$가 1 증가하면 $y$는 2 증가.
- 기울기 = $\Delta y / \Delta x$ = **2**

#### 곡선의 기울기?

직선이 아니라 곡선이라면 기울기가 위치마다 다릅니다.

```
        곡선 y = x²
     y
     ↑
   9 |          ●  ← 여기 기울기는?
   4 |     ●
   1 | ●
   0 |____________→ x
       1  2  3
```

**미분**은 "곡선 위의 한 점에서의 기울기"를 구하는 방법입니다.

### 8.3 비유: 자동차 속도계

자동차로 1시간에 60km를 달렸습니다.
- 평균 속도: 60 km/h.

하지만 자동차가 매 순간 정확히 60 km/h로 달린 건 아닙니다. 빨리도 갔다, 신호에서 멈췄다 합니다.

**속도계**는 "**바로 지금 이 순간**의 속도"를 보여줍니다. 이게 바로 **미분**입니다.

> 미분 = "거리(위치)를 시간으로 미분한 것 = 순간 속도"

### 8.4 도함수(Derivative) 표기법

함수 $f(x)$의 미분을 다음과 같이 씁니다:

$$
f'(x), \quad \frac{df}{dx}, \quad \frac{d}{dx}f(x)
$$

세 가지 다 같은 뜻입니다.

### 8.5 자주 쓰이는 미분 공식 (간단)

| 함수 | 미분 |
|---|---|
| $x^2$ | $2x$ |
| $x^3$ | $3x^2$ |
| $x^n$ | $nx^{n-1}$ |
| 상수 $c$ | $0$ |
| $e^x$ | $e^x$ |
| $\ln(x)$ | $1/x$ |

#### 직관적 해석

- $f(x) = x^2$: $x = 3$에서 기울기는 $f'(3) = 2 \times 3 = 6$.
- 즉, 이 점에서 함수는 가파르게 증가하고 있다.

### 8.6 미분의 중요한 의미: "어디로 가야 함수가 커지나/작아지나?"

- $f'(x) > 0$ (양수): 그 지점에서 함수가 **증가** 중. 즉, $x$를 늘리면 함수값이 더 커짐.
- $f'(x) < 0$ (음수): 그 지점에서 함수가 **감소** 중. 즉, $x$를 늘리면 함수값이 작아짐.
- $f'(x) = 0$: 그 지점은 **극값**(최대/최소/안장점).

이 사실이 바로 머신러닝 학습의 핵심입니다.

### 8.7 머신러닝과의 연결

#### 손실 함수 최소화

손실 함수 $L(\theta)$가 있다고 합시다 ($\theta$는 모델 파라미터).

목표: $L$을 최소화하는 $\theta$ 찾기.

방법:
1. 현재 위치에서 $L$의 미분 $\frac{dL}{d\theta}$ 계산.
2. 미분이 양수면 → $\theta$를 줄임.
3. 미분이 음수면 → $\theta$를 키움.
4. 미분이 0에 가까워질 때까지 반복.

이게 바로 **경사하강법(Gradient Descent)** 입니다.

### 8.8 챕터 요약

✅ **미분** = 함수의 기울기, 변화율.
✅ 자동차 속도계 비유: "지금 이 순간"의 변화 속도.
✅ 미분이 양수면 함수가 증가, 음수면 감소.
✅ 머신러닝 학습 = 손실 함수의 미분을 0으로 만드는 파라미터 찾기.

## Chapter 9. 편미분과 그래디언트 — 다차원의 미분

### 9.1 변수가 여러 개라면?

지금까지는 변수가 하나인 함수($f(x)$)를 봤습니다. 하지만 머신러닝 모델은 **수백만~수십억 개의 파라미터**를 가집니다.

이런 다변수 함수를 어떻게 미분할까요?

### 9.2 편미분(Partial Derivative) — "한 변수만 보고 미분"

함수 $f(x, y) = x^2 + 3y^2$가 있다고 합시다.

**편미분**: 다른 변수는 상수로 두고, 한 변수에 대해서만 미분.

- $x$에 대한 편미분 ($y$는 상수 취급): $\frac{\partial f}{\partial x} = 2x$
- $y$에 대한 편미분 ($x$는 상수 취급): $\frac{\partial f}{\partial y} = 6y$

기호 $\partial$ ("델" 또는 "라운드 디")는 편미분을 나타냅니다.

#### 비유: 산의 경사

당신이 산 위에 있다고 합시다. 산은 위치(x, y)에 따라 높이가 다른 함수죠.
- $\frac{\partial 높이}{\partial x}$ = 동쪽으로 한 발 갔을 때 높이 변화 (남북 방향은 무시).
- $\frac{\partial 높이}{\partial y}$ = 북쪽으로 한 발 갔을 때 높이 변화 (동서 방향은 무시).

### 9.3 그래디언트(Gradient) — "모든 편미분의 묶음"

**그래디언트**는 모든 편미분을 모아 놓은 **벡터**입니다.

$$
\nabla f = \left[ \frac{\partial f}{\partial x}, \frac{\partial f}{\partial y}, \frac{\partial f}{\partial z}, ... \right]
$$

기호: $\nabla f$ ("나블라 f" 또는 "델 f")

위 함수 $f(x, y) = x^2 + 3y^2$의 그래디언트:
$$
\nabla f = [2x, 6y]
$$

### 9.4 그래디언트의 핵심 성질: "가장 가파른 오르막 방향"

이게 가장 중요한 직관입니다.

> **그래디언트는 함수가 가장 빠르게 증가하는 방향을 가리킨다.**

#### 산 비유

당신이 산 어딘가에 있습니다. 그 지점의 그래디언트는 **"가장 가파른 오르막 방향"** 을 가리킵니다.

- 그래디언트의 **방향**: 가장 빠르게 위로 올라가는 길.
- 그래디언트의 **크기**: 그 방향으로 가면 얼마나 가파른가.

```
                    ⬆ 그래디언트 방향
       ●     ●  ●
        ●  ●  ●     산
         ●●●        
        ●  ●  
       ●     ●
                ↘
               당신의 위치
```

#### 반대로 생각하면?

**가장 빠르게 내려가는 방향 = 그래디언트의 반대 방향**

이게 바로 머신러닝의 핵심 기법입니다.

### 9.5 그래디언트의 시각화

2변수 함수의 그래디언트 시각화:

```
함수 등고선 (위에서 본 모습):

   y ↑
     |       ╱╱╱╱╱╱╱╱╱╱╱
     |      ╱  높이 100  ╲
     |     ╱  높이 50    ╲
     |    ╱  ◯ 최저점    ╲
     |   ╱   높이 50      ╲
     |  ╱   높이 100        ╲
     |________________________→ x
```

각 점에서 그래디언트는 **밖으로(높이가 증가하는 쪽)** 향합니다.
가장 깊은 골짜기로 가려면? 그래디언트의 **반대 방향**으로 가면 됩니다.

### 9.6 머신러닝과의 연결

#### 손실 함수의 그래디언트

신경망 모델이 있다고 합시다.
- 파라미터: $\theta = [\theta_1, \theta_2, ..., \theta_n]$ (수백만 개)
- 손실 함수: $L(\theta)$

$L$의 그래디언트:
$$
\nabla_\theta L = \left[ \frac{\partial L}{\partial \theta_1}, \frac{\partial L}{\partial \theta_2}, ..., \frac{\partial L}{\partial \theta_n} \right]
$$

이건 "각 파라미터를 조금씩 바꿨을 때 손실이 어떻게 변할지"를 나타내는 벡터입니다.

학습 규칙:
$$
\theta \leftarrow \theta - \alpha \cdot \nabla_\theta L
$$

"파라미터를 그래디언트의 **반대 방향**으로 조금씩 이동시켜라."
- $\alpha$: 학습률(learning rate). 한 번에 얼마나 이동할지.

이게 바로 **경사하강법(Gradient Descent)** 입니다.

### 9.7 챕터 요약

✅ **편미분**: 한 변수에 대해서만 미분, 나머지는 상수 취급.
✅ **그래디언트** = 모든 편미분을 모은 벡터.
✅ 그래디언트는 "가장 빠르게 함수가 증가하는 방향"을 가리킴.
✅ 그래디언트의 **반대 방향**으로 가면 함수가 빠르게 감소 → 경사하강법.

## Chapter 10. 경사하강법과 역전파 — AI는 어떻게 학습하나

### 10.1 경사하강법(Gradient Descent)

이제 모든 퍼즐 조각이 모였습니다.

#### 알고리즘 (한 줄 요약)

```
손실의 그래디언트의 반대 방향으로 파라미터를 조금씩 움직인다 — 손실이 더 줄지 않을 때까지.
```

#### 의사 코드

```python
theta = 초기값  # 예: 무작위
learning_rate = 0.01

for step in range(10000):
    gradient = compute_gradient(loss, theta)
    theta = theta - learning_rate * gradient
    
    if gradient ≈ 0:
        break  # 골짜기 도착!
```

#### 산을 내려가는 비유

안개 속에서 산을 내려가야 한다고 합시다.
- 멀리 볼 수 없으니, **발 밑의 경사**만 보고 가장 가파른 내리막으로 한 걸음 내려갑니다.
- 다시 멈춰서 발 밑 경사 확인 → 또 한 걸음.
- 평평한 곳에 도달하면 멈춥니다 → 골짜기에 도착!

```
           ╱╲
          ╱  ╲   ← 시작
         ╱    ╲ 
        ╱      ╲ ← 한 걸음
       ╱        ╲
      ╱          ╲ ← 또 한 걸음
     ╱            ╲
    ╱     ◯골짜기   ← 도착!
```

### 10.2 학습률(Learning Rate) — 한 걸음의 크기

`learning_rate`(α)는 **한 번에 얼마나 움직일지** 결정합니다.

#### 너무 작으면?
- 학습이 매우 느림.
- 영원히 도착 못 할 수도.

```
●→●→●→●→●→●→●→...   학습 너무 느림
```

#### 너무 크면?
- 골짜기를 **건너뛸 수** 있음.
- 발산해서 학습 실패.

```
●           ●
  →       ←
    ●   ●           위아래로 튀고 있음 - 발산!
        ●  ●
              →
              ●
```

#### 적절하면?
- 빠르고 안정적으로 수렴.

#### 실무에서
- **학습률 스케줄링**: 처음에는 크게, 나중에는 작게.
- **Adam, AdamW 등의 옵티마이저**: 학습률을 자동 조정.

### 10.3 변형들

#### Batch Gradient Descent
- 모든 데이터를 다 보고 한 번 업데이트.
- 정확하지만 느림.

#### Stochastic Gradient Descent (SGD)
- 한 데이터마다 업데이트.
- 빠르지만 불안정.

#### Mini-batch SGD (실제 사용)
- 32~512개씩 묶어서 업데이트.
- 정확도와 속도의 좋은 균형.

> 💡 **MLOps 실무**: GPU 메모리, 배치 크기, 학습률은 서로 영향을 미칩니다. 배치 크기를 2배로 하면 학습률도 √2배 정도 키우는 것이 일반적.

### 10.4 연쇄법칙(Chain Rule) — 신경망의 핵심

이제 가장 중요한 개념입니다.

#### 합성 함수의 미분

함수 $y = f(g(x))$가 있다고 합시다. (안에 또 다른 함수가 있음)

연쇄법칙:
$$
\frac{dy}{dx} = \frac{dy}{dg} \cdot \frac{dg}{dx}
$$

말로 풀면:
> "전체의 변화율 = 바깥 함수의 변화율 × 안쪽 함수의 변화율"

#### 비유: 톱니바퀴 시스템

세 개의 톱니바퀴가 맞물려 있습니다: A → B → C
- A가 1바퀴 돌면 B가 2바퀴 돌고, B가 1바퀴 돌면 C가 3바퀴 돈다.
- **A가 1바퀴 돌면 C는 몇 바퀴 돌까?**
- 답: 2 × 3 = **6바퀴**.

이게 연쇄법칙입니다.

### 10.5 역전파(Backpropagation)

신경망은 여러 층의 합성 함수입니다.

```
입력 → [층1] → [층2] → [층3] → [층4] → 출력 → 손실
```

각 층에는 가중치 행렬(파라미터)이 있고, 우리는 **모든 층의 파라미터에 대한 손실의 그래디언트**가 필요합니다.

#### 역전파의 아이디어

> **"손실에서 시작해서, 연쇄법칙을 거꾸로 적용하며 그래디언트를 전달한다"**

```
입력 → [층1] → [층2] → [층3] → [층4] → 손실
        ↑       ↑       ↑       ↑
        그래디언트   그래디언트   그래디언트   그래디언트
        ←───────←───────←───────←  (역방향으로 전파!)
```

#### 직관적 설명

각 층은 자기 다음 층에서 받은 "이 정도로 틀렸어!"라는 정보를 받아, 자기 가중치에 대한 기여를 계산하고, 다시 이전 층에 전달합니다.

마치 회사에서 사장(손실)이 "매출이 안 좋다"고 했을 때:
- 영업팀장(층4): "내 잘못 30%, 마케팅에 부탁한다"
- 마케팅팀장(층3): "내 잘못 20%, 디자인에 부탁한다"
- ... (계속 책임 분배가 거꾸로 전파)

각 팀(층)은 자기 책임을 알게 되어, 그만큼 행동을 수정(가중치 업데이트)합니다.

### 10.6 전체 학습 사이클

신경망 학습의 한 스텝:

```
1. Forward Pass (순전파)
   입력 → 모델 → 예측
   
2. Loss 계산
   손실 = (예측 - 실제값)²
   
3. Backward Pass (역전파)
   각 파라미터에 대한 그래디언트 계산 (연쇄법칙)
   
4. Update (업데이트)
   파라미터 ← 파라미터 - 학습률 × 그래디언트
   
5. 반복
   epoch 끝까지, 손실이 충분히 작아질 때까지
```

이게 **모든 딥러닝의 기본 원리**입니다. GPT-4, Claude, Stable Diffusion 모두 이 방식으로 학습합니다.

### 10.7 LLMOps/MLOps 실무 연결

#### 🚀 학습 비용
- GPT-4 같은 모델은 수개월에 걸쳐 수만 개 GPU에서 역전파를 수행.
- 한 번의 순전파+역전파 = 수많은 행렬 곱셈 + 미분 = GPU 핵심 부하.

#### 🚀 그래디언트 관련 문제

**Vanishing Gradient (그래디언트 소실)**
- 깊은 신경망에서 역전파할 때 그래디언트가 점점 작아져 0에 가까워짐.
- 초기 층들이 학습이 안 됨.
- 해결: ReLU 활성화 함수, 잔차 연결(ResNet), 배치 정규화 등.

**Exploding Gradient (그래디언트 폭발)**
- 그래디언트가 너무 커져서 발산.
- 해결: Gradient Clipping (그래디언트 크기에 상한선 설정).

#### 🚀 LLM 학습 기법

**LoRA (Low-Rank Adaptation)**
- 거대한 가중치 행렬 전부를 학습하는 대신, 저차원 행렬만 학습.
- 그래디언트 계산량이 훨씬 적음 → 메모리/시간 절약.

**Gradient Accumulation**
- GPU 메모리가 부족할 때, 작은 배치들의 그래디언트를 누적해서 큰 배치 효과.

**Mixed Precision Training**
- FP16(반정밀도)로 그래디언트 계산 → 빠르고 메모리 절약.
- FP32와 적절히 섞어서 정확도 유지.

#### 🚀 분산 학습

**Data Parallelism**
- 여러 GPU에 데이터를 분산.
- 각 GPU가 그래디언트 계산 → 합쳐서 평균 → 모든 GPU 동기화.
- AllReduce 연산이 핵심.

**Model Parallelism / Pipeline Parallelism**
- 모델 자체를 여러 GPU에 쪼개서 배치.
- 거대 모델(GPT-4 등) 학습에 필수.

### 10.8 챕터 요약

✅ **경사하강법**: 손실의 그래디언트 반대 방향으로 파라미터 이동.
✅ **학습률**: 한 걸음의 크기. 너무 크면 발산, 너무 작으면 느림.
✅ **연쇄법칙**: 합성 함수의 미분. 톱니바퀴 비유.
✅ **역전파**: 손실에서 시작해 거꾸로 그래디언트를 전파.
✅ **딥러닝 학습 사이클**: 순전파 → 손실 계산 → 역전파 → 업데이트 → 반복.
✅ Vanishing/Exploding Gradient, LoRA, 분산 학습 등 실무 이슈는 모두 이 원리에 기반.

### 선형대수 용어
| 용어 | 영어 | 한 줄 설명 |
|---|---|---|
| 스칼라 | Scalar | 그냥 숫자 하나 |

| 행렬 | Matrix | 숫자들의 표 |
| 텐서 | Tensor | 행렬의 다차원 일반화 (3차원 이상) |

| 노름 | Norm | 벡터의 크기 |
| 전치 | Transpose | 행과 열을 바꾼 행렬 |
| 역행렬 | Inverse Matrix | 곱하면 단위 행렬 되는 행렬 |

| 고유벡터 | Eigenvector | 변환 후에도 방향 안 변하는 벡터 |

### 확률/통계 용어
| 용어 | 영어 | 한 줄 설명 |
|---|---|---|
| 확률 | Probability | 0~1 사이로 표현되는 가능성 |
| 조건부 확률 | Conditional Probability | "B일 때 A일 확률" |
| 분포 | Distribution | 값들이 얼마나 자주 나오는지 |
| 정규분포 | Normal/Gaussian | 종 모양 분포 |
| 평균 | Mean | 데이터의 중심값 |
| 분산 | Variance | 평균에서 흩어진 정도 |
| 표준편차 | Standard Deviation | 분산의 제곱근 |
| 기댓값 | Expected Value | 가중 평균 |
| 사전 확률 | Prior | 데이터 보기 전의 믿음 |
| 사후 확률 | Posterior | 데이터 본 후의 갱신된 믿음 |
| 우도 | Likelihood | 가설이 맞을 때 데이터가 나올 확률 |
| 베이즈 정리 | Bayes' Theorem | 믿음 갱신 공식 |

### 미적분 용어
| 용어 | 영어 | 한 줄 설명 |

| 그래디언트 | Gradient | 모든 편미분을 모은 벡터 |

| 학습률 | Learning Rate | 한 걸음 크기 |
| 역전파 | Backpropagation | 거꾸로 그래디언트 전파 |
| 손실 함수 | Loss Function | 모델이 얼마나 틀렸는지 측정 |

## E. MLOps/LLMOps 엔지니어를 위한 추가 조언

수학을 깊이 알 필요 없는 MLOps 영역도 많지만, 다음 상황에서는 수학 이해가 결정적입니다.

### 🎯 모델 디버깅
- 학습이 안 될 때: 그래디언트 소실/폭발인지 확인 (미분 이해 필요).
- Loss curve가 이상할 때: 분포와 통계 직관 필요.

### 🎯 모델 최적화
- **양자화 (Quantization)**: FP32 → INT8 변환의 수치적 의미.
- **Pruning**: 어떤 가중치를 제거할지 (행렬의 중요도 = 고유값).
- **Distillation**: 작은 모델로 큰 모델 모방 (확률 분포 매칭).

### 🎯 평가와 모니터링
- **Drift Detection**: 데이터 분포 변화 감지 (KL 발산, Wasserstein 거리).
- **A/B 테스트**: 통계적 유의성, 베이지안 접근.
- **신뢰도 평가**: 모델 confidence가 의미 있는지 (calibration).

### 🎯 비용/성능 트레이드오프
- **임베딩 차원 vs 검색 정확도**: PCA로 압축하면 얼마나 손해?
- **배치 크기 vs 학습률**: 수학적 관계.
- **샘플링 전략 (Temperature, Top-p)**: 분포 조작의 의미.
