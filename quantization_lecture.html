/ 01 · WHY QUANTIZE?

왜 우리는 모델을 압축해야 하는가

현대 LLM은 너무 크다. 그리고 GPU는 비싸고, 메모리는 부족하고, 사용자는 빠른 응답을 원한다. 양자화는 이 세 가지 제약을 한꺼번에 푸는 가장 가성비 높은 도구다.

/ 01 · MEMORY

메모리 절벽

FP16의 70B 모델은 140GB. H100 80GB 한 장으로는 절대 못 올린다. 양자화는 이 벽을 절반, 또는 1/4로 낮춘다.

140 → 35 GB
/ 02 · LATENCY

추론 속도

LLM 추론은 대부분 메모리 대역폭 병목이다. 가중치를 작게 만들면 GPU가 읽어들이는 속도가 빨라져 처리량이 늘어난다.

2~4× 더 빠르게
/ 03 · COST

운영 비용

같은 모델을 더 작은 GPU에 올리면, 시간당 인스턴스 비용이 직접적으로 줄어든다. 엣지/온디바이스 배포의 핵심 전제 조건.

−60% infra cost
/ 04 · ACCESSIBILITY

접근성

맥북, 게이밍 GPU, 심지어 라즈베리파이에서도 LLM이 돈다. 오픈소스 생태계의 폭발은 양자화가 만든 가능성이다.

Edge-ready

/ 03 · WHEN TO QUANTIZE

학습 후냐, 학습 중이냐

양자화는 적용 시점에 따라 두 가지로 나뉜다. 대부분의 오픈 LLM 사용자는 PTQ만 알면 충분하지만, 두 차이를 알아두면 트레이드오프가 명확해진다.

/ PTQ · Post-Training Quantization

학습 후 양자화

이미 학습된 모델의 가중치를 그대로 양자화한다. 추가 학습 없이 GPU 한두 장으로 몇 시간 내 끝난다. Hugging Face에 올라오는 대부분의 양자화 모델이 이 방식이다.

  • 속도: 분~시간 단위로 빠름
  • 비용: 매우 낮음 (단일 GPU OK)
  • 품질: INT8까지는 거의 무손실, INT4부터 살짝 손실
  • 대표: GPTQ, AWQ, bitsandbytes, llama.cpp

/ QAT · Quantization-Aware Training

학습 중 양자화 시뮬레이션

학습 과정에서 양자화로 인한 오차를 미리 시뮬레이션하며 학습한다. 모델이 양자화에 "익숙해진" 상태로 수렴하므로 INT4·INT2 같은 극단적 압축에서 PTQ보다 강하다.

  • 속도: 일~주 단위 학습 필요
  • 비용: 매우 높음 (멀티 GPU)
  • 품질: 저비트(2~3bit) 영역에서 PTQ보다 우위
  • 대표: LLM-QAT, BitNet
/ 04 · CORE ALGORITHMS

양자화 알고리즘의 안쪽

"FP16을 INT4로 바꿔라" 자체는 단순한 산수다. 진짜 문제는 품질을 어떻게 지킬 것인가. 각 카드를 열어 핵심 아이디어를 확인해 보자.

GPTQ
PTQ · 2022
레이어 단위로, 오차를 다음 가중치로 흘려보내며 보정
+

핵심 아이디어. 가중치 행렬을 한 번에 양자화하지 않고, 한 컬럼씩 양자화하면서 발생한 오차를 Hessian(2차 미분 정보)을 이용해 아직 양자화되지 않은 컬럼들에 보정으로 흘려보낸다. 결과적으로 단순 반올림보다 훨씬 작은 출력 오차를 얻는다.

W_quantized = round(W) → 오차 ε → 다음 컬럼 W' ← W' + correction(ε, H⁻¹)

특징. 빠르고(한 모델 1~2시간), 4bit에서 잘 동작. 다만 calibration 데이터셋이 필요하며, 데이터 분포가 안 맞으면 품질이 흔들린다.

대표 도구. AutoGPTQ, ExLlamaV2

AWQ
PTQ · 2023 · MIT
"중요한 1%의 가중치"만 보호하면 품질을 지킨다
+

핵심 아이디어. AWQ(Activation-aware Weight Quantization)는 모든 가중치가 동등하지 않다는 관찰에서 출발한다. 활성화(activation)의 크기가 큰 채널과 곱해지는 가중치 채널은 모델 성능에 큰 영향을 준다.

핵심 트릭: 중요한 채널의 가중치에 scale을 곱해서 키워두고(양자화 오차의 상대적 비중을 줄임), 활성화 쪽에서 동일한 값으로 나누어 수학적 등가성을 유지한다.

Y = (W · s) · (X / s) = W · X   //   s는 채널별 scale (activation 통계 기반)

장점. calibration 데이터 의존도가 GPTQ보다 낮고, 일반화 성능이 안정적이다. 4bit에서 PPL(perplexity) 손실이 0.1~0.3 수준.

대표 도구. AutoAWQ, vLLM, TGI가 네이티브 지원

SmoothQuant
PTQ · W8A8
활성화의 outlier를 가중치 쪽으로 옮겨 양쪽을 동시에 양자화
+

문제 인식. LLM의 활성화 텐서에는 가끔 매우 큰 outlier가 나타난다. 이 때문에 활성화(A)를 INT8로 양자화하면 다른 값들이 모두 0에 가까워져 정보가 손실된다. 반면 가중치(W)는 분포가 균일해 양자화하기 쉽다.

해법. 수학적 등가 변환을 이용해 activation의 "양자화 어려움"을 weight 쪽으로 일부 이전(smooth)시켜 양쪽 모두 양자화 가능하게 만든다. 결과는 W8A8 (가중치·활성화 모두 INT8) inference.

왜 중요한가. AWQ/GPTQ는 weight만 양자화(W4A16)하는 반면, SmoothQuant는 activation도 양자화해 추론 연산 자체를 INT8 텐서코어에서 돌릴 수 있다 — 진짜 속도 이득의 핵심.

bitsandbytes
PTQ · NF4 / FP4 / INT8
HF Transformers와 가장 잘 통합되는 "그냥 쓰는" 양자화
+

특징. Tim Dettmers의 라이브러리. load_in_8bit=True 또는 load_in_4bit=True 한 줄로 끝난다. LLM.int8()(outlier-aware INT8), NF4(Normal Float 4)를 제공.

NF4 = Normal Float 4. 가중치 분포가 대략 정규분포라는 점을 이용해, 4bit이 표현하는 16개 값을 정규분포의 16분위수에 맞춘다. 균등 분할 INT4보다 정보 손실이 작다.

Double Quantization. 양자화에 필요한 scale 값 자체도 한번 더 양자화해 모델당 0.4bit/param을 추가로 절감 — QLoRA의 핵심 트릭.

주된 용도. 빠른 프로토타이핑, QLoRA 기반 파인튜닝. 추론 속도는 AWQ/GPTQ보다 살짝 느린 편.

FP8
Hopper / Ada · H100·H200·B200
NVIDIA가 학습·추론 모두 밀고 있는 차세대 표준
+

두 가지 변종. E4M3은 정밀도(mantissa)가 더 많아 forward용으로, E5M2는 동적 범위(exponent)가 더 넓어 backward(gradient)용으로 사용한다.

장점. H100/H200의 FP8 텐서코어는 FP16 대비 2배의 처리량을 낸다. 학습에도 쓸 수 있어 INT8 대비 큰 변화 없이 도입 가능.

실전. vLLM, TensorRT-LLM, NeMo가 FP8 추론을 본격 지원. NVIDIA Transformer Engine이 학습 측 표준 도구.

참고. 4bit(FP4)는 Blackwell(B100/B200) 세대에서 본격적인 텐서코어 가속이 들어오면서 추론 표준 후보로 부상 중.

GGUF · k-quants
Format · llama.cpp
CPU/Apple Silicon에서 LLM을 돌리는 사실상의 표준
+

정체. GGUF는 알고리즘이라기보다 파일 포맷 + 양자화 방식 모음이다. llama.cpp 생태계의 표준이며, Mac/Linux/Windows의 CPU/Metal/CUDA에서 동일한 파일로 동작한다.

k-quants. 블록 단위로 가중치를 묶어 양자화하되, 중요한 블록은 더 많은 비트로 표현한다. Q2_K부터 Q8_0까지 단계별 옵션 제공.

Q4_K_M ≈ 4.84 bits/weight · 품질/크기 균형 좋아 가장 인기

왜 인기인가. CPU 추론에 최적화된 SIMD 커널, mmap을 통한 즉시 로딩, 메타데이터 내장. Ollama, LM Studio, Jan 같은 데스크톱 도구가 전부 GGUF 기반.

/ 06 · BEYOND QUANTIZATION

양자화 vs 다른 최적화 기법들

양자화는 LLM 최적화의 한 축일 뿐이다. 실제 프로덕션에서는 여러 기법을 조합한다. 각 기법이 무엇을 줄이는지, 양자화와 어떻게 다른지 비교해 보자.

압축 · 가중치

Quantization

가중치/활성화의 비트 수를 줄임. 모델 구조는 그대로 유지하고 표현만 압축.

📉 메모리 ↓⚡ 속도 ↑🎯 품질 ~유지
압축 · 가중치

Pruning · 가지치기

중요도가 낮은 가중치를 0으로 만들어 제거. structured(채널 단위)와 unstructured(개별 가중치)로 나뉨.

📉 메모리 ↓⚡ 속도 ↑*⚠ 재학습 필요
압축 · 모델 자체

Knowledge Distillation

큰 teacher 모델의 출력을 작은 student가 흉내내며 학습. 더 작은 새 모델을 만드는 접근.

📦 모델 ↓🧠 학습 비용 ↑🎓 도메인 특화 강함
학습 효율

LoRA · QLoRA

전체가 아니라 저랭크 어댑터만 학습. QLoRA는 베이스를 4bit로 양자화 + LoRA 결합 = 메모리 미친 절감.

💾 학습 메모리 ↓🔧 fine-tune 전용🤝 양자화와 결합
추론 엔진

FlashAttention

Attention 연산을 SRAM-aware하게 재작성. 메모리 I/O를 줄여 같은 모델을 더 빠르게.

⚡ 속도 ↑📉 활성화 메모리 ↓🎯 무손실
추론 엔진

PagedAttention · KV Cache 최적화

KV cache를 OS 가상 메모리처럼 페이지 단위로 관리. vLLM의 핵심으로 처리량을 수 배 끌어올림.

📈 throughput ↑🎯 무손실🤝 양자화와 결합
추론 알고리즘

Speculative Decoding

작은 draft 모델이 토큰 후보를 빠르게 만들면, 큰 모델이 한 번에 검증. 평균 latency 2~3배 단축.

⚡ latency ↓🎯 무손실🤝 양자화와 결합
모델 구조

MoE · Mixture of Experts

전체 파라미터 중 일부 expert만 활성화. 큰 모델의 "용량"은 유지하며 "연산"은 작게 가는 구조 선택.

💪 capacity ↑⚙ 연산 ↓🚧 구조적 변경
한 줄 요약. 양자화는 "표현 비트 수"를, Pruning은 "가중치 개수"를, Distillation은 "모델 자체"를, LoRA는 "학습할 파라미터 수"를, FlashAttention/PagedAttention은 "메모리 접근 패턴"을, MoE는 "활성 파라미터 비율"을 줄인다. 실전에서는 보통 양자화(AWQ/GGUF) + 추론 엔진 최적화(vLLM/llama.cpp) + 필요시 Speculative 조합으로 간다.

/ 08 · TOOLING ECOSYSTEM

알아두면 좋은 주요 컴포넌트

양자화 생태계는 "모델을 만드는 도구"와 "양자화된 모델을 돌리는 엔진"으로 크게 나뉜다.

간단한 자기 점검

강의 내용을 잘 따라왔다면 어렵지 않을 것이다.

맥북 M3 Pro에서 Llama-3 8B를 직접 돌리려고 한다. 가장 합리적인 선택은?
정답. GGUF는 Apple Silicon의 Metal을 통해 가속되고, 단일 파일·즉시 로딩으로 데스크톱에 최적이다.
"AWQ의 핵심 아이디어"를 가장 잘 설명한 것은?
정답. AWQ = Activation-aware Weight Quantization. 활성화 크기를 단서로 중요한 가중치 채널을 식별·보호한다.
QLoRA에서 베이스 모델에 사용되는 양자화는?
정답. NF4는 가중치 분포가 정규분포에 가깝다는 점을 이용한 4bit 부동소수 표현이다.
