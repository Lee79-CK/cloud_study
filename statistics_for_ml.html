CHAPTER 01 · PROLOGUE

왜 ML 엔지니어가 통계학을 알아야 하는가

— On the necessity of statistical thinking in the age of machine learning

머신러닝은 본질적으로 통계적 추론(statistical inference)입니다. 우리가 "모델을 학습시킨다"라고 말할 때, 실제로 하는 일은 데이터로부터 확률 분포를 추정하는 것입니다. 손실 함수(loss function)는 우연이 아니라 통계학의 우도(likelihood)에서 유도되었고, 모델 평가는 표본을 통한 추론이며, 과적합은 표본 편향의 한 형태입니다.

그런데 많은 엔지니어가 model.fit()은 익숙해도 그 뒤에서 무슨 일이 일어나는지 모릅니다. 통계학을 모르면 모델이 잘 작동할 때는 좋지만, 문제가 생겼을 때 디버깅할 언어가 없습니다. "왜 이 모델은 일반화가 안 되지?", "이 메트릭 차이가 유의미한가?", "내 데이터의 분포가 이상한가?" — 이 모든 질문은 통계학의 영역입니다.

▮ Why now
LLM 시대에 들어서면서 통계학의 중요성은 오히려 더 커졌습니다. 다음 토큰을 예측한다는 것은 결국 조건부 확률 분포 $P(x_t \mid x_{1:t-1})$를 학습하는 일이고, RLHF의 보상 모델은 통계적 추정기이며, 평가 벤치마크의 신뢰성도 표본 통계로 판단합니다.

이 강의가 다루는 것

  • 개념의 직관 — 공식을 외우기 전에, 왜 그런 공식이 만들어졌는지를 먼저 이해합니다.
  • 다른 수학과의 비교 — 미적분, 선형대수, 확률론과 어떻게 다른지 명확히 합니다.
  • ML 연결고리 — 모든 개념을 실제 ML 알고리즘 또는 LLMOps 실무에 연결합니다.
  • 인터랙티브 시각화 — 슬라이더를 움직이며 직접 분포가 변하는 것을 체험합니다.
이 강의를 마치면, 논문의 수식을 볼 때 막막함이 아닌 친밀함을 느끼게 됩니다.
CHAPTER 02 · COMPARISON

통계학 vs 다른 수학

— Where statistics sits in the mathematical landscape

통계학을 처음 만나는 엔지니어는 종종 이렇게 묻습니다. "확률론과 통계학은 같은 거 아닌가?", "선형대수와 통계학은 어떻게 다른가?" 결론부터 말하면, 이들은 같은 수학적 우주의 다른 행성들입니다. 서로 가깝지만 풍경이 다릅니다.

학문 중심 질문 다루는 대상 ML에서의 역할
통계학
Statistics		 "이 데이터로부터 어떤 결론을 내릴 수 있는가?" 표본(sample)과 그로부터의 추론 모델 학습, 평가, A/B 테스트
확률론
Probability		 "분포가 주어졌을 때, 사건이 일어날 가능성은?" 분포 자체와 그 성질 생성 모델, 베이즈 추론, 강화학습
미적분학
Calculus		 "이 함수의 변화율은? 최솟값은?" 연속 함수와 그 도함수 경사하강법, 역전파(backprop)
선형대수
Linear Algebra		 "이 데이터를 어떤 공간에서 어떻게 변환할까?" 벡터, 행렬, 선형 변환 임베딩, attention, 차원 축소
이산수학
Discrete Math		 "유한한 구조에서 무엇이 가능한가?" 그래프, 조합, 논리 그래프 신경망, 토큰화, 알고리즘

확률론 vs 통계학 — 가장 헷갈리는 쌍

이 둘은 방향이 반대인 한 동전의 양면입니다. 비유로 풀어봅시다.

◐ 확률론 (Probability)
"동전이 공정하다는 것을 안다. 그러면 10번 던져서 앞면이 7번 나올 확률은?"
→ 모형 → 데이터의 방향. 분포가 먼저 주어지고, 데이터에 대한 예측을 한다.
◑ 통계학 (Statistics)
"동전을 10번 던졌더니 앞면이 7번 나왔다. 이 동전은 공정한가?"
→ 데이터 → 모형의 방향. 데이터가 먼저 주어지고, 분포(모형)를 추론한다.

머신러닝은 이 두 방향을 모두 씁니다. 학습(training)은 통계학에 가깝고 (데이터로부터 모델 파라미터 추정), 추론(inference)은 확률론에 가깝습니다 (학습된 모델로부터 새 데이터의 확률 계산). 그래서 ML 엔지니어는 두 언어를 모두 알아야 합니다.

왜 미적분과 선형대수만으로는 부족한가

딥러닝 입문서들이 종종 미적분과 선형대수에만 집중하는 이유는, 이들이 최적화의 도구이기 때문입니다. 하지만 무엇을 최적화할 것인가는 통계학이 결정합니다.

  • 왜 MSE(Mean Squared Error)를 손실로 쓰는가? → 데이터가 가우시안 노이즈를 따른다고 가정했을 때의 MLE이기 때문.
  • 왜 Cross-Entropy를 분류에 쓰는가? → 라벨이 다항 분포를 따른다는 가정의 MLE.
  • 왜 정규화(regularization)가 효과적인가? → 사전 분포(prior)를 부여하는 MAP 추정의 효과.

즉, 미적분이 "어떻게 내려갈까"라면, 통계학은 "어디로 내려가야 하는가"를 알려줍니다.

CHAPTER 03 · DESCRIPTION

기술 통계 — 데이터를 한눈에

— Descriptive statistics: summarizing the shape of data

1억 개의 데이터 포인트가 있다면 그것을 다 살펴볼 수는 없습니다. 기술 통계(descriptive statistics)는 데이터를 몇 개의 숫자로 요약하는 기술입니다. 평균이 얼마인지, 얼마나 흩어져 있는지, 모양은 어떤지 — 이 세 가지가 핵심입니다.

중심 경향 — 데이터의 "가운데"

평균 (Mean, Average)
$$\bar{x} = \frac{1}{n}\sum_{i=1}^{n} x_i$$
모든 값을 더해서 개수로 나눈다. 가장 직관적이지만 이상치(outlier)에 매우 취약합니다.

예를 들어 직원 9명의 연봉이 모두 5천만 원인데, 사장님 1명이 50억을 가져간다면? 평균은 약 5.5억이 되지만, 이는 "평균적인 직원의 연봉"을 전혀 설명하지 못합니다. 이때 필요한 것이 중앙값(median)입니다 — 정렬했을 때 한가운데 있는 값. 위 예에서 중앙값은 5천만 원으로 현실을 더 잘 반영합니다.

▮ ML 적용
LLM이 생성한 응답 품질을 평가할 때, 평균 점수만 보면 몇 개의 매우 나쁜 응답이 가려질 수 있습니다. 중앙값, 사분위수(quartile), 최악 케이스(P99)를 함께 봐야 합니다. 실무에서 "P95 레이턴시"를 모니터링하는 이유도 동일합니다.

산포 — 데이터의 "흩어짐"

평균만으로는 데이터의 모양을 알 수 없습니다. 두 데이터셋의 평균이 같아도 분포는 완전히 다를 수 있습니다. 그래서 분산(variance)과 표준편차(standard deviation)가 등장합니다.

분산 (Variance)
$$\sigma^2 = \frac{1}{n}\sum_{i=1}^{n}(x_i - \bar{x})^2$$
각 데이터와 평균의 차이를 제곱하여 평균. 제곱하는 이유는 ① 부호 상쇄 방지, ② 큰 차이에 더 큰 페널티.
표준편차 (Standard Deviation)
$$\sigma = \sqrt{\sigma^2}$$
분산의 제곱근. 원본 데이터와 같은 단위를 갖기 때문에 해석하기 쉽습니다. "평균에서 평균적으로 얼마나 떨어져 있는가."

분산은 제곱된 단위(예: 원²)라서 직관이 어렵습니다. 표준편차는 다시 원래 단위로 돌아오므로 일상적으로 더 자주 쓰입니다.

왜 ML에서 "정규화(normalization)"가 필요한가

피처(feature)들의 스케일이 다르면 — 한쪽은 0~1, 다른 쪽은 0~10,000 — 모델 학습이 불안정해집니다. 표준화(standardization)는 각 피처를 평균 0, 표준편차 1로 변환합니다:

Z-score Standardization
$$z_i = \frac{x_i - \mu}{\sigma}$$
scikit-learn의 StandardScaler가 하는 일이 정확히 이것입니다. 딥러닝의 BatchNorm, LayerNorm도 본질적으로 같은 발상입니다.
CHAPTER 04 · PROBABILITY

확률과 분포

— The vocabulary of uncertainty

통계학의 모든 추론은 확률 분포(probability distribution) 위에서 일어납니다. 분포란 "어떤 값이 얼마나 자주 나올 수 있는지"를 함수로 표현한 것입니다. 데이터를 본다는 것은 결국 그 데이터를 생성한 분포를 추측하는 일입니다.

두 종류의 분포

분포는 크게 두 가지로 나뉩니다.

종류 특징 예시 ML 적용
이산 분포
Discrete		 값이 셀 수 있는 점들. PMF(확률질량함수) 사용. 주사위, 동전, 클릭 여부 분류(classification), 토큰 예측
연속 분포
Continuous		 값이 연속체. PDF(확률밀도함수) 사용. 키, 온도, 픽셀 값 회귀(regression), 생성(generation)
▮ 주의 — PDF는 확률이 아닙니다
연속 분포에서 PDF의 값 자체는 확률이 아닙니다. $f(x) = 2$가 나와도 이상하지 않습니다. 연속 분포에서 확률은 면적으로 표현됩니다: $P(a \le X \le b) = \int_a^b f(x)\,dx$. 한 점에서의 확률은 항상 0입니다.

주요 분포 모음

1. 베르누이 분포 (Bernoulli)

한 번의 동전 던지기처럼 성공/실패 두 가지 결과만 있는 분포. 확률 $p$로 1이 나옵니다.

$$P(X=k) = p^k(1-p)^{1-k}, \quad k \in \{0,1\}$$
ML: 이진 분류 — 로지스틱 회귀의 출력이 베르누이 분포의 $p$입니다.

2. 이항 분포 (Binomial)

베르누이 시행을 $n$번 반복했을 때 성공 횟수. "동전을 10번 던져 앞면이 3번 나올 확률"이 이항 분포입니다.

$$P(X=k) = \binom{n}{k} p^k (1-p)^{n-k}$$

3. 카테고리/다항 분포 (Categorical / Multinomial)

주사위처럼 결과가 $K$개인 분포. 다음 토큰 예측은 본질적으로 카테고리 분포의 추정입니다. LLM의 마지막 softmax 출력이 바로 이 분포입니다.

4. 포아송 분포 (Poisson)

단위 시간/공간 안에 사건이 몇 번 일어나는지. "1시간에 도착하는 고객 수", "GPU 노드의 시간당 에러 발생 수"같은 카운트 데이터.

$$P(X=k) = \frac{\lambda^k e^{-\lambda}}{k!}$$

5. 정규분포 (Normal / Gaussian)

다음 챕터 전체를 차지하는 주인공입니다. 자연과 ML에 가장 자주 등장하는 분포.

6. 균등 분포, 지수 분포, 베타 분포...

이외에도 많지만, 위 다섯 개를 이해하면 80%는 처리 가능합니다.

CHAPTER 05 · GAUSSIAN

정규분포의 위엄

— Why the bell curve rules everything around us

정규분포(Normal distribution), 일명 가우시안(Gaussian)은 통계학과 머신러닝의 절대 군주입니다. 데이터 사이언티스트 사이에서 "의심스러우면 일단 가우시안을 가정하라"는 말이 있을 정도입니다.

정규분포의 확률밀도함수
$$f(x) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp\!\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)$$
$\mu$ = 평균(중심), $\sigma^2$ = 분산(폭). 단 두 개의 파라미터로 종 모양 곡선이 완전히 결정됩니다.

왜 이렇게 중요한가

  • 중심극한정리(CLT) — 어떤 분포에서 표본을 뽑아 평균을 내면, 그 평균의 분포는 가우시안에 수렴합니다. (다음 챕터)
  • 최대 엔트로피 — 평균과 분산만 주어진 상태에서 가장 "무지(uninformative)"한 분포가 가우시안입니다.
  • 해석적으로 다루기 쉬움 — 가우시안끼리의 곱, 합, 조건부 분포가 모두 다시 가우시안입니다.
  • 자연의 노이즈는 보통 가우시안 — 측정 오차, 센서 노이즈, 작은 변동의 합은 가우시안에 가깝습니다.

68-95-99.7 규칙

이 숫자는 외워두면 평생 씁니다.

  • 평균에서 ±1σ 안에 데이터의 약 68%
  • 평균에서 ±2σ 안에 약 95%
  • 평균에서 ±3σ 안에 약 99.7%

물리/공학의 "Six Sigma 품질"이란 표현은 ±6σ 밖의 결함률 (백만 분의 3.4)을 목표로 한다는 뜻입니다.

CHAPTER 06 · CLT

중심극한정리

— The most beautiful theorem in statistics

통계학에 단 하나의 정리만 알아야 한다면, 이것입니다. 중심극한정리(Central Limit Theorem, CLT)는 다음과 같이 말합니다:

"어떤 분포에서 뽑은 표본이라도, 그 평균은 가우시안 분포에 수렴한다."

The Central Limit Theorem

이것이 왜 충격적인가? 원본 분포가 어떤 모양이든 — 균등 분포든, 지수 분포든, 이상한 두 봉우리 분포든 — 표본 평균의 분포는 항상 가우시안이 됩니다. 자연 어디에나 가우시안이 출현하는 이유의 90%는 바로 이 정리 때문입니다.

CLT의 수학적 표현
$$\bar{X}_n \xrightarrow{d} \mathcal{N}\!\left(\mu, \frac{\sigma^2}{n}\right)$$
표본 크기 $n$이 커질수록, 표본 평균은 평균 $\mu$, 분산 $\sigma^2/n$인 가우시안으로 수렴합니다. 표본이 많아질수록 분산이 줄어드는 것 (1/n)이 핵심입니다.

왜 ML 엔지니어가 알아야 하는가

  • A/B 테스트의 근거 — 두 실험군의 평균 차이가 유의미한지 판단하는 모든 통계 검정은 CLT에 기반합니다.
  • 미니배치 학습 — 미니배치 gradient는 전체 gradient의 표본 평균이고, 따라서 가우시안 노이즈를 따릅니다. 이 노이즈가 좋은 일반화를 돕습니다.
  • 모델 평가의 신뢰성 — 평가 데이터가 충분히 많으면 정확도의 분포는 가우시안이고, 신뢰구간을 계산할 수 있습니다.

CHAPTER 07 · INFERENCE

추론 통계와 가설검정

— From samples to claims about the world

추론 통계(inferential statistics)는 표본으로부터 모집단을 추론하는 기술입니다. 우리는 결코 모든 사용자의 데이터를 볼 수 없고, 일부 표본만으로 결론을 내려야 합니다. 이때 "얼마나 확신할 수 있는가"를 정량화하는 것이 추론 통계의 일입니다.

신뢰구간 — Confidence Interval

"이 모델의 정확도는 87%다"라고 말할 때, 실제로는 표본에서 87%일 뿐, 진짜 정확도는 알 수 없습니다. 그래서 우리는 이렇게 말합니다: "95% 신뢰수준에서 정확도는 85% ~ 89% 사이에 있다."

평균에 대한 95% 신뢰구간 (대표본)
$$\bar{x} \pm 1.96 \cdot \frac{s}{\sqrt{n}}$$
표본평균 $\pm$ (1.96 × 표준오차). 1.96은 표준정규분포에서 95%를 포함하는 z-값입니다.
▮ 주의 — 신뢰구간의 올바른 해석
"참 평균이 이 구간에 들어갈 확률이 95%다"는 틀린 해석입니다. 정확한 의미는: "같은 방법으로 신뢰구간을 100번 만든다면, 그중 95개가 참 평균을 포함한다." 미묘하지만 중요한 차이입니다 — 이것이 빈도주의(frequentist) 통계의 관점입니다.

가설검정 — Hypothesis Testing

새로 학습한 모델 B가 기존 모델 A보다 정말 더 좋을까? 단순히 평균 정확도가 1% 높다고 결론낼 수는 없습니다. 그 차이가 우연인지 아닌지를 판단해야 합니다.

가설검정의 절차:

  1. 귀무가설 $H_0$ (null hypothesis): "두 모델은 차이가 없다"
  2. 대립가설 $H_1$ (alternative hypothesis): "두 모델은 다르다"
  3. 데이터로부터 검정 통계량을 계산
  4. $H_0$가 참이라고 가정했을 때 이 통계량이 나올 확률 ($p$-value)을 구함
  5. $p$-value가 작으면 ($\alpha = 0.05$ 미만) $H_0$를 기각
▮ p-value를 오해하지 말 것
$p$-value는 "귀무가설이 참일 확률"이 아닙니다. 정확한 정의: "귀무가설이 참이라고 가정했을 때, 현재 데이터 또는 그보다 더 극단적인 데이터가 관측될 확률." $p=0.03$은 "내가 본 결과가 우연일 가능성이 3%"라는 의미에 가깝습니다.

실무 — A/B 테스트

회사에서 두 가지 LLM 프롬프트 중 어느 것이 더 사용자 만족도가 높은지 테스트한다고 합시다.

# A/B 테스트 예시 (Python, scipy)
from scipy import stats

# 프롬프트 A: 1000명의 만족도 (0~10)
prompt_a = [...]
# 프롬프트 B: 1000명의 만족도
prompt_b = [...]

# 두 집단의 평균이 같다는 귀무가설을 검정
t_stat, p_value = stats.ttest_ind(prompt_a, prompt_b)

if p_value < 0.05:
    print("통계적으로 유의미한 차이가 있음")
else:
    print("차이가 우연일 가능성을 배제할 수 없음")
CHAPTER 08 · BAYESIAN

베이즈 통계 — ML의 사고방식

— Updating beliefs in the face of evidence

지금까지 다룬 것은 모두 빈도주의(frequentist) 관점이었습니다. 모르는 모수(parameter)는 고정된 값이고, 우리는 그것을 추정하려 합니다. 베이즈주의(Bayesian)는 다릅니다. 모수도 확률 변수로 보고, 그것에 대한 믿음(belief)을 데이터로 업데이트합니다.

현대 머신러닝의 많은 부분 — 특히 생성 모델, 강화학습, 불확실성 추정 — 은 베이즈적 사고에 깊이 의존합니다.

베이즈 정리 (Bayes' Theorem) · 통계학에서 가장 중요한 공식
$$P(\theta \mid D) = \frac{P(D \mid \theta) \cdot P(\theta)}{P(D)}$$
사후 분포 = (우도 × 사전 분포) / 증거. 데이터를 본 후 모수에 대한 우리의 믿음을 어떻게 업데이트해야 하는가, 그 답입니다.

네 개의 용어를 익히면 끝

  • 사전 분포 $P(\theta)$ (Prior) — 데이터를 보기 전의 믿음.
  • 우도 $P(D \mid \theta)$ (Likelihood) — 모수 $\theta$가 주어졌을 때 데이터가 관측될 확률.
  • 사후 분포 $P(\theta \mid D)$ (Posterior) — 데이터를 본 후의 업데이트된 믿음.
  • 증거 $P(D)$ (Evidence / Marginal) — 정규화 상수. 보통 계산이 어려워서 우회합니다.

의학 진단으로 보는 베이즈

유명한 예제입니다. 1%의 사람이 걸리는 질병이 있고, 검사는 99% 정확합니다. 양성 판정을 받았다면, 실제로 병에 걸렸을 확률은?

직관: 99%?
베이즈의 답: 약 50% 정도밖에 안 됩니다.

왜? 1000명 중 병 환자는 10명. 그 중 양성 = 9.9명. 건강한 990명 중 거짓 양성 = 9.9명. 양성 판정자 19.8명 중 진짜 환자는 9.9명, 즉 50%. 사전 확률(prior)이 작으면 검사가 정확해도 양성의 의미가 약합니다.

CHAPTER 09 · MLE & FRIENDS

MLE · MAP · Cross-Entropy

— Where statistics becomes machine learning

이제 모든 것이 연결됩니다. 머신러닝의 손실 함수(loss function)들은 하늘에서 떨어진 것이 아니라, 통계학에서 유도됩니다.

최대 우도 추정 — MLE

Maximum Likelihood Estimation: "관측된 데이터가 가장 잘 설명되는 모수 $\theta$를 찾자."

MLE의 정의
$$\hat{\theta}_{MLE} = \arg\max_\theta \prod_{i=1}^{n} P(x_i \mid \theta)$$
로그를 취하면 곱이 합이 되어 다루기 쉬워집니다: $\hat{\theta} = \arg\max_\theta \sum_i \log P(x_i \mid \theta)$. 이를 log-likelihood라 부릅니다.

MLE → 손실 함수의 정체

이 부분이 통계학과 ML을 연결하는 황금 다리입니다.

(1) 선형 회귀 = 가우시안 MLE

"$y = wx + b + \varepsilon$, 여기서 $\varepsilon \sim \mathcal{N}(0, \sigma^2)$" — 노이즈가 가우시안이라고 가정합니다.

이 가정 하에서 MLE를 유도하면 정확히 MSE(Mean Squared Error)가 나옵니다:

$$-\log P(y \mid x, w) \propto \sum_i (y_i - wx_i)^2$$
MSE를 최소화한다 = 가우시안 노이즈 가정에서 우도를 최대화한다. 같은 일입니다.

(2) 분류 = 카테고리 MLE = Cross-Entropy

라벨 $y \in \{1, ..., K\}$의 확률 분포가 $p_y$라면, MLE는:

$$\mathcal{L} = -\sum_i \log p_{y_i}$$
이것이 바로 PyTorch/TensorFlow의 CrossEntropyLoss입니다. LLM이 다음 토큰을 학습할 때 사용하는 그 손실 함수.

MAP — Maximum A Posteriori

MLE에 사전 분포(prior)를 더하면 MAP가 됩니다:

$$\hat{\theta}_{MAP} = \arg\max_\theta \left[ \log P(D \mid \theta) + \log P(\theta) \right]$$
우도 항 + 사전 항. 두 번째 항이 곧 정규화(regularization)입니다.
  • L2 정규화 (weight decay) = 가우시안 사전 분포 가정
  • L1 정규화 (Lasso) = 라플라스 사전 분포 가정

그래서 정규화는 마법이 아니라 "파라미터에 대한 우리의 사전 믿음"의 수학적 표현입니다.

Cross-Entropy & KL Divergence

두 분포 $P$와 $Q$가 얼마나 다른지 측정하는 핵심 개념입니다.

엔트로피 · 크로스 엔트로피 · KL 발산
$$H(P) = -\sum_x P(x)\log P(x)$$
$$H(P, Q) = -\sum_x P(x)\log Q(x)$$
$$D_{KL}(P \| Q) = \sum_x P(x) \log \frac{P(x)}{Q(x)} = H(P,Q) - H(P)$$
$H(P)$ = $P$의 본질적 불확실성. $H(P,Q)$ = $Q$로 $P$를 표현했을 때 드는 평균 비트. $D_{KL}$ = $Q$가 $P$에서 얼마나 떨어졌는가.
▮ LLM 학습의 정체
LLM 학습은 본질적으로 실제 데이터 분포 $P_{\text{data}}$와 모델 분포 $P_\theta$의 KL 발산을 최소화하는 일입니다. 실제 데이터는 라벨로 표현되니까 (one-hot), $H(P_{\text{data}})$는 상수가 되고, 결국 Cross-Entropy를 최소화하는 것과 같아집니다. RLHF의 PPO에서 KL penalty가 등장하는 것도 동일한 맥락 — "원 모델에서 너무 멀어지지 마".

편향-분산 트레이드오프

모델의 일반화 오류를 통계적으로 분해하면:

$$\text{Error} = \text{Bias}^2 + \text{Variance} + \text{Noise}$$
단순한 모델은 편향이 크고(과소적합), 복잡한 모델은 분산이 큽니다(과적합). 둘의 합을 최소화하는 지점이 최적 복잡도.
CHAPTER 10 · IN PRACTICE

실무에서의 통계

— Statistics in the daily life of an ML engineer

마지막 장에서는 지금까지 배운 것을 실무 시나리오에 적용해봅니다.

시나리오 1 — 모델 성능 보고

나쁜 보고: "새 모델의 정확도는 87.3%입니다."

좋은 보고: "새 모델의 정확도는 87.3% (95% CI: [86.1%, 88.5%], n=5000)이며, 베이스라인 84.1%와의 차이는 통계적으로 유의미합니다 (p < 0.001, 2-sample t-test). 다만 P5(worst-case) 시나리오에서 여전히 72%로 개선 여지가 있습니다."

시나리오 2 — 데이터 품질 진단

새로운 학습 데이터셋을 받았다면:

  1. 분포 시각화: 히스토그램, 박스플롯으로 각 피처를 확인.
  2. 중심 경향과 산포: 평균, 중앙값, 표준편차. 평균과 중앙값이 크게 다르면 비대칭 분포.
  3. 이상치 탐지: IQR 기반 또는 Z-score > 3 기준.
  4. 분포 비교: 학습/검증/테스트 분포가 같은가? 분포 이동(distribution shift)이 일어나면 모델이 무력해집니다.
  5. 라벨 균형: 분류 문제라면 클래스 비율 확인. 극단적 불균형은 별도 처리 필요.

시나리오 3 — LLM 평가 (LLMOps 맥락)

LLM의 응답 품질을 평가할 때:

  • 점 추정 대신 분포: 단일 점수가 아니라 점수의 분포 전체를 보고.
  • 다중 평가자(annotator): 평가자 간 일치도(Cohen's kappa, Krippendorff's alpha)를 측정.
  • 샘플 크기 계산: 두 모델의 1%p 차이를 95% 검정력으로 검출하려면 몇 개의 평가가 필요한가? — Power analysis.
  • 온도(temperature)와 분산: 같은 프롬프트에 대해 여러 번 생성하여 응답의 분산도 측정.

시나리오 4 — 프로덕션 모니터링

배포된 모델의 성능 저하를 통계적으로 감지:

  • 예측 분포의 변화: KL divergence, JS divergence로 학습 시점과 비교.
  • 입력 데이터의 변화: 각 피처의 분포가 통계적으로 변했는가? — KS test(Kolmogorov-Smirnov).
  • 모니터링 메트릭의 신뢰구간: 일일 정확도의 변동이 자연스러운 잡음인가, 진짜 저하인가?
▮ 마무리
통계학은 외우는 학문이 아니라 생각하는 방식입니다. 모든 데이터에는 노이즈가 있고, 모든 추정에는 불확실성이 있다는 사실을 기억하는 것 — 이것이 ML 엔지니어가 통계학에서 배워야 할 가장 중요한 자세입니다.
