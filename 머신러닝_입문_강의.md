## 1. 머신러닝이란 무엇인가?

### 🌟 한 줄 정의

> **머신러닝(Machine Learning)이란, 컴퓨터가 데이터로부터 스스로 규칙을 찾아내어 문제를 해결하는 기술입니다.**

### 🍎 일상 속 비유로 이해하기

여러분이 어린 아이에게 "사과"를 가르친다고 상상해보세요.

**방법 1: 규칙을 알려주기 (전통적 방식)**
> "사과는 빨갛고 둥글고 손바닥만 하단다."

그런데 아이가 빨간 토마토를 보면 어떻게 될까요? 사과라고 착각하겠죠. 노란 사과는 또 어떻게 설명할까요? 규칙이 점점 복잡해집니다.

**방법 2: 사진을 많이 보여주기 (머신러닝 방식)**
> 사과 사진 100장을 보여주며 "이게 사과야"라고 알려주고,
> 토마토, 오렌지, 배 사진을 보여주며 "이건 사과가 아니야"라고 알려줍니다.

어느 순간 아이는 처음 보는 사과 사진을 보고도 "사과!"라고 외칠 수 있게 됩니다.

**바로 이것이 머신러닝의 핵심입니다.**
- 명시적으로 규칙을 알려주지 않습니다.
- 대신 **수많은 예시(데이터)**를 보여줍니다.
- 컴퓨터는 그 예시들 사이의 **패턴**을 스스로 발견합니다.
- 발견한 패턴을 새로운 상황에 적용합니다.

### 💡 왜 머신러닝이 필요한가?

세상에는 규칙으로 명확히 정의하기 어려운 문제들이 많습니다.

| 문제 | 규칙으로 풀기 어려운 이유 |
|------|---------------------------|
| 고양이 사진 구별 | 고양이의 모양, 색깔, 자세가 너무 다양함 |
| 음성 인식 | 사람마다 발음, 억양, 속도가 다름 |
| 스팸 메일 분류 | 스팸의 형태가 계속 진화함 |
| 추천 시스템 | 사람마다 취향이 다름 |

이런 문제들은 사람도 명확한 규칙으로 설명하기 어렵습니다. 우리는 그냥 "보면 안다"라고 말하죠. 머신러닝은 바로 이런 영역에서 빛을 발합니다.

## 2. 전통적 프로그래밍 vs 머신러닝

### 🔧 전통적 프로그래밍

```
[입력 데이터] + [규칙(코드)] → [결과]
```

예: 계산기 프로그램
- 입력: 숫자 2와 3
- 규칙: "두 숫자를 더하라"
- 결과: 5

**프로그래머가 모든 규칙을 직접 작성합니다.**

### 🤖 머신러닝

```
[입력 데이터] + [정답(결과)] → [규칙(모델)]
```

예: 사과 인식 프로그램
- 입력: 과일 사진들
- 정답: 각 사진이 사과인지 아닌지
- 결과: "사과를 알아보는 규칙"이 만들어짐

**컴퓨터가 데이터를 보고 규칙을 스스로 만듭니다.**

### 📊 비교표

| 구분 | 전통적 프로그래밍 | 머신러닝 |
|------|-------------------|----------|
| 규칙 작성자 | 사람(프로그래머) | 컴퓨터 |
| 필요한 것 | 명확한 로직 | 많은 데이터 |
| 적합한 문제 | 규칙이 분명한 문제 | 패턴이 복잡한 문제 |
| 예시 | 계산기, 은행 시스템 | 얼굴 인식, 번역 |

## 3. AI, 머신러닝, 딥러닝의 관계

이 세 용어는 자주 혼용되지만 사실은 **포함 관계**입니다.

```
┌─────────────────────────────────────────┐
│  인공지능 (AI)                           │
│  ┌───────────────────────────────────┐  │
│  │  머신러닝 (ML)                    │  │
│  │  ┌─────────────────────────────┐  │  │
│  │  │  딥러닝 (DL)                │  │  │
│  │  └─────────────────────────────┘  │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

### 🧠 인공지능 (AI, Artificial Intelligence)
- **가장 큰 개념**
- 사람처럼 생각하고 행동하는 모든 기술
- 1950년대부터 존재한 오래된 개념
- 예: 체스 프로그램, 챗봇, 자율주행차

### 📊 머신러닝 (ML, Machine Learning)
- **AI를 구현하는 한 가지 방법**
- 데이터로부터 학습하는 방식
- 예: 스팸 필터, 추천 시스템

### 🧬 딥러닝 (DL, Deep Learning)
- **머신러닝의 한 분야**
- "신경망"이라는 특별한 모델을 깊게(deep) 쌓아 만듦
- 사람의 뇌 구조에서 영감을 받음
- 예: ChatGPT, 이미지 생성 AI, 음성 인식

### 🍰 케이크 비유

- **AI**: 케이크 전체
- **머신러닝**: 케이크 안의 크림
- **딥러닝**: 크림 안의 특별한 재료(생크림)

딥러닝은 머신러닝의 한 종류이고, 머신러닝은 AI의 한 종류입니다.

## 4. 머신러닝의 세 가지 종류

머신러닝은 학습 방식에 따라 크게 세 가지로 나뉩니다.

### 4-1. 지도학습 (Supervised Learning) 🎯

> **"정답이 있는 문제집으로 공부하기"**

선생님이 옆에서 정답을 알려주며 가르치는 방식입니다.

**비유**: 학원에서 수학 문제집을 풀 때, 답지가 함께 있는 경우입니다. 문제를 풀고, 답지를 보고, 틀린 부분을 고칩니다.

**예시**:
- **분류(Classification)**: 이메일이 스팸인지 아닌지 구분
  - 입력: 이메일 내용
  - 정답: "스팸" 또는 "정상"
- **회귀(Regression)**: 집값 예측
  - 입력: 집의 크기, 위치, 방 개수
  - 정답: 실제 거래된 가격(숫자)

**실생활 예시**:
- 카드 부정 사용 탐지
- 의료 영상으로 질병 진단
- 영화 별점 예측

### 4-2. 비지도학습 (Unsupervised Learning) 🔍

> **"정답 없이 스스로 패턴 찾기"**

정답을 알려주지 않고, 데이터의 숨은 구조를 스스로 찾아내게 합니다.

**비유**: 옷장에 옷이 잔뜩 쌓여 있을 때, 누가 알려주지 않아도 색깔별로, 계절별로, 종류별로 자연스럽게 분류하는 것과 같습니다.

**예시**:
- **군집화(Clustering)**: 고객을 비슷한 그룹으로 묶기
  - VIP 고객, 일반 고객, 신규 고객 등을 자동으로 분류
- **차원 축소(Dimensionality Reduction)**: 복잡한 데이터를 단순화
  - 100개의 특성을 핵심적인 2~3개로 압축

**실생활 예시**:
- 쇼핑몰의 고객 세분화
- 뉴스 기사 자동 분류
- 이상 거래 탐지 (평소 패턴과 다른 거래 발견)

### 4-3. 강화학습 (Reinforcement Learning) 🎮

> **"시행착오를 통해 보상받으며 배우기"**

좋은 행동에는 보상을, 나쁜 행동에는 벌점을 주며 가르칩니다.

**비유**: 강아지에게 "앉아"를 가르칠 때, 잘하면 간식을 주고 못하면 안 주는 방식입니다. 강아지는 점점 간식을 받기 위한 행동을 학습하게 됩니다.

**예시**:
- 알파고 (바둑 AI)
- 게임 AI (스타크래프트, 도타 2)
- 자율주행차 (안전한 운전 → 보상, 충돌 → 벌점)
- 로봇 제어

### 📋 세 가지 학습 방식 비교

| 종류 | 정답 데이터 | 비유 | 대표 활용 |
|------|------------|------|-----------|
| 지도학습 | ✅ 있음 | 답지 있는 문제집 | 분류, 예측 |
| 비지도학습 | ❌ 없음 | 옷장 정리 | 군집화, 패턴 발견 |
| 강화학습 | 💰 보상 | 강아지 훈련 | 게임, 로봇 |

## 5. 데이터: 머신러닝의 연료

### ⛽ "데이터는 새로운 석유다"

머신러닝에서 **데이터의 양과 질**은 모델의 성능을 좌우합니다. 아무리 좋은 알고리즘이 있어도 데이터가 부족하거나 잘못되었다면 좋은 결과를 얻을 수 없습니다.

### 5-1. 특성(Feature)과 레이블(Label)

데이터를 표로 표현하면 이렇게 생겼습니다:

| 키(cm) | 몸무게(kg) | 나이 | → | 성별 |
|--------|-----------|------|---|------|
| 175 | 70 | 25 | → | 남 |
| 162 | 55 | 30 | → | 여 |
| 180 | 80 | 28 | → | 남 |

- **특성(Feature)**: 입력 정보 (키, 몸무게, 나이)
  - 모델이 보고 판단할 단서들
- **레이블(Label)**: 정답 (성별)
  - 모델이 맞춰야 할 답

### 5-2. 데이터의 분할

머신러닝에서는 데이터를 보통 **3등분**합니다.

```
전체 데이터 (100%)
├── 학습 데이터 (Training Set, 60~80%)
├── 검증 데이터 (Validation Set, 10~20%)
└── 테스트 데이터 (Test Set, 10~20%)
```

**비유**: 시험 준비 과정과 같습니다.

| 데이터 | 비유 | 역할 |
|--------|------|------|
| 학습 데이터 | 교과서 | 모델을 가르침 |
| 검증 데이터 | 모의고사 | 모델 성능 확인 및 개선 |
| 테스트 데이터 | 실제 시험 | 최종 성능 평가 |

⚠️ **중요한 원칙**: 테스트 데이터는 모델 학습에 절대 사용하면 안 됩니다. 학생이 시험 문제를 미리 보고 외우는 것과 같기 때문입니다.

### 5-3. 좋은 데이터의 조건

1. **양이 충분해야 함**: 많을수록 좋습니다
2. **다양해야 함**: 한쪽으로 치우치면 안 됩니다
3. **정확해야 함**: 잘못된 정답은 모델을 망칩니다
4. **최신이어야 함**: 너무 오래된 데이터는 현재와 맞지 않을 수 있습니다

> 💬 **유명한 격언**: "Garbage In, Garbage Out (쓰레기를 넣으면 쓰레기가 나온다)"

## 6. 머신러닝은 어떻게 학습하는가?

### 🎯 학습의 본질: "오차를 줄이는 과정"

머신러닝의 학습 과정은 본질적으로 **'예측한 답'과 '실제 정답'의 차이를 줄여나가는 과정**입니다.

### 🏹 양궁 비유로 이해하기

처음 양궁을 배우는 사람을 상상해보세요.

1. **첫 번째 화살**: 과녁에서 한참 빗나갑니다 (큰 오차)
2. **자세를 조금 수정**: "조금 위로 쏴야겠다"
3. **두 번째 화살**: 조금 더 가까이 맞습니다 (오차 감소)
4. **계속 조정**: 바람, 거리, 자세를 미세하게 조정
5. **결국 명중**: 정확히 과녁 중앙에 맞춥니다 (오차 최소화)

머신러닝도 정확히 이런 과정을 거칩니다.

### 🔑 핵심 3요소

#### 1️⃣ 모델 (Model)
- 예측을 수행하는 함수
- 양궁의 "활과 화살"에 해당
- 내부에 **파라미터(parameter)**라는 조절 가능한 값들이 있음
- 비유: 라디오의 주파수 다이얼처럼, 이 값들을 미세하게 조정해서 정답에 맞춥니다

#### 2️⃣ 손실 함수 (Loss Function)
- 예측과 정답의 차이를 측정하는 도구
- 양궁의 "과녁에서의 거리"에 해당
- 손실(loss)이 작을수록 좋은 모델

**예시**:
- 실제 집값: 5억 원
- 모델 예측: 4억 5천만 원
- 손실: 5천만 원

#### 3️⃣ 최적화 (Optimization)
- 손실을 줄이는 방향으로 모델을 업데이트하는 과정
- 양궁에서 "자세를 조정"하는 것에 해당
- 가장 유명한 방법: **경사 하강법(Gradient Descent)**

### 🏔️ 경사 하강법: 산에서 내려오기

**상황**: 안개 낀 산에 갇혔습니다. 가장 낮은 골짜기로 내려가야 합니다.

**전략**: 발 밑의 경사를 느끼고, 가장 가파르게 내려가는 방향으로 한 걸음씩 이동합니다.

```
        🏔️ (시작 지점, 손실이 큼)
       /
      /  ← 한 걸음씩 내려감
     /
    /
   🏞️ (도착 지점, 손실이 최소)
```

이 과정을 수천, 수만 번 반복하면 모델은 점점 정확한 예측을 하게 됩니다.

### 📈 학습 과정 요약

```
1. 모델이 예측을 한다
        ↓
2. 손실 함수로 오차를 계산한다
        ↓
3. 오차를 줄이는 방향으로 파라미터를 조정한다
        ↓
4. 1~3을 수천 번 반복한다
        ↓
5. 더 이상 개선되지 않으면 학습 완료
```

## 7. 대표적인 알고리즘 살펴보기

머신러닝에는 수많은 알고리즘이 있지만, 가장 기초적이고 중요한 몇 가지만 살펴봅시다.

### 7-1. 선형 회귀 (Linear Regression) 📏

**가장 단순하고 기초적인 알고리즘**

데이터에 가장 잘 맞는 **직선**을 찾는 방법입니다.

**예시**: 공부 시간과 시험 점수의 관계

```
점수
 100│                    ●
  90│                ●
  80│            ●        ← 이 점들에 가장 잘 맞는
  70│        ●              직선을 찾는 것!
  60│    ●
  50│●
    └────────────────────→ 공부 시간
```

직선의 방정식: `점수 = 5 × 공부시간 + 50`

이제 7시간 공부하면 점수는 `5 × 7 + 50 = 85점`으로 예측할 수 있습니다.

**활용**: 집값 예측, 매출 예측, 키와 몸무게 관계 등

### 7-2. 로지스틱 회귀 (Logistic Regression) ⚖️

**이름은 회귀지만 실제로는 분류 알고리즘**

"예/아니오"처럼 두 가지 중 하나를 선택하는 문제에 사용합니다.

**예시**: 시험 합격 여부 예측
- 입력: 공부 시간
- 출력: 합격(1) 또는 불합격(0)

**핵심**: 0과 1 사이의 확률을 출력합니다.
- 0.8 이상 → "합격일 가능성이 높음"
- 0.3 → "불합격일 가능성이 높음"

### 7-3. 의사결정 트리 (Decision Tree) 🌳

**스무고개처럼 질문을 던져 분류하는 방법**

**예시**: 동물 분류

```
                     [날개가 있나?]
                    /              \
                  Yes               No
                   ↓                 ↓
              [부리가 있나?]    [다리가 4개?]
              /        \        /        \
            Yes        No     Yes         No
             ↓          ↓      ↓           ↓
            🦅         🦋    [짖나?]      🐍
                              /    \
                            Yes     No
                             ↓       ↓
                            🐕      🐈
```

**장점**: 사람이 이해하기 쉽습니다 (해석 가능)
**활용**: 대출 승인, 의료 진단, 고객 분류

### 7-4. 랜덤 포레스트 (Random Forest) 🌲🌲🌲

**여러 개의 의사결정 트리를 묶어서 더 좋은 결과를 내는 방법**

**비유**: 한 명의 의사보다 여러 의사의 종합 진단이 더 정확합니다.

각 트리가 독립적으로 예측하고, 그 결과를 **투표**로 결정합니다.

### 7-5. 신경망 (Neural Network) 🧠

**사람의 뇌 구조에서 영감을 받은 모델**

```
입력층          은닉층            출력층
  ●─────────────●
                  \
  ●─────────────● ─────────────● 결과
                  /
  ●─────────────●
```

각 동그라미는 **뉴런(neuron)**이며, 신호를 주고받으며 복잡한 패턴을 학습합니다.

**딥러닝**은 이 신경망을 **매우 깊게(여러 층) 쌓은 것**입니다.

```
입력 → [층1] → [층2] → [층3] → ... → [층100] → 출력
```

### 📊 알고리즘 선택 가이드

| 상황 | 추천 알고리즘 |
|------|---------------|
| 숫자 예측 (간단) | 선형 회귀 |
| 예/아니오 분류 | 로지스틱 회귀 |
| 해석이 중요한 분류 | 의사결정 트리 |
| 정확도가 중요한 분류 | 랜덤 포레스트 |
| 이미지/음성/언어 | 신경망(딥러닝) |

## 8. 과적합과 과소적합

머신러닝에서 가장 흔히 마주치는 두 가지 문제입니다.

### 📚 학생 비유

세 명의 학생이 수학 시험을 준비합니다.

#### 학생 A: 공부를 너무 안 했다 → 과소적합 (Underfitting)
- 문제집을 대충 훑어봤음
- 기본 개념도 모름
- 결과: 모의고사도, 실제 시험도 모두 못 봄

#### 학생 B: 적당히 공부했다 → 좋은 모델 (Good Fit)
- 핵심 개념을 이해했음
- 다양한 유형을 풀어봤음
- 결과: 모의고사도, 실제 시험도 잘 봄

#### 학생 C: 문제집을 통째로 외웠다 → 과적합 (Overfitting)
- 문제집의 모든 답을 외웠음
- 그러나 응용력이 없음
- 결과: 모의고사(=문제집)는 100점, 실제 시험은 50점

### 📈 시각적으로 이해하기

```
과소적합 (너무 단순)         좋은 모델               과적합 (너무 복잡)
                                                 
   ●                          ●                      ●
●     ●                    ●    ●                  ╱●╲
   ●     ●     →         ●        ●     →     ╱●     ●╲
●─────────────       ╱─────────────╲       ╱  ●  ●  ●  ╲
                                            (모든 점을 지나려 함)
```

### 🎯 어떻게 해결할까?

**과소적합 해결법**:
- 더 복잡한 모델 사용
- 더 많은 특성(feature) 추가
- 더 오래 학습

**과적합 해결법**:
- 더 많은 데이터 수집
- 모델을 더 단순하게
- **정규화(Regularization)**: 모델이 너무 복잡해지지 않도록 제한
- **드롭아웃(Dropout)**: 학습 중 일부 뉴런을 무작위로 끔
- **조기 종료(Early Stopping)**: 검증 성능이 나빠지면 학습 멈춤

### ⚖️ 핵심 교훈

> 머신러닝의 목표는 **학습 데이터를 완벽하게 외우는 것이 아니라**, **새로운 데이터에서도 잘 작동하는 것**입니다. 이를 **일반화(Generalization)**라고 합니다.

## 9. 모델은 어떻게 평가하는가?

모델을 만들었으면 "얼마나 잘 작동하는가?"를 측정해야 합니다.

### 9-1. 분류 문제의 평가 지표

#### 혼동 행렬 (Confusion Matrix)

암 진단 모델을 예로 들어봅시다.

|  | 예측: 암 | 예측: 정상 |
|---|---|---|
| **실제: 암** | TP (정답!) | FN (놓침 ⚠️) |
| **실제: 정상** | FP (오진) | TN (정답!) |

- **TP (True Positive)**: 암 환자를 암으로 진단 ✅
- **TN (True Negative)**: 정상인을 정상으로 진단 ✅
- **FP (False Positive)**: 정상인을 암으로 오진 ❌
- **FN (False Negative)**: 암 환자를 정상으로 진단 ❌ (가장 위험!)

#### 주요 평가 지표

**1. 정확도 (Accuracy)**
- 전체 중 맞춘 비율
- `(TP + TN) / 전체`
- 단순하지만 함정이 있음 ⚠️

> **함정 예시**: 1000명 중 암 환자가 10명일 때, 모든 사람을 "정상"이라고 예측하면 정확도는 99%! 하지만 암 환자를 한 명도 못 잡았으니 쓸모없는 모델입니다.

**2. 정밀도 (Precision)**
- "암이라고 예측한 사람 중 진짜 암 환자는?"
- `TP / (TP + FP)`
- 오진을 줄이는 게 중요할 때 사용

**3. 재현율 (Recall)**
- "실제 암 환자 중 암으로 진단한 사람은?"
- `TP / (TP + FN)`
- 놓치면 안 될 때 사용 (의료, 보안)

**4. F1 점수**
- 정밀도와 재현율의 조화 평균
- 두 지표를 균형있게 보고 싶을 때

### 9-2. 회귀 문제의 평가 지표

숫자를 예측하는 문제에서는 다음 지표를 사용합니다.

**1. MAE (Mean Absolute Error, 평균 절대 오차)**
- 예측과 실제의 차이를 절대값으로 평균
- 직관적이고 이해하기 쉬움

**2. MSE (Mean Squared Error, 평균 제곱 오차)**
- 차이를 제곱해서 평균
- 큰 오차에 더 큰 페널티

**3. R² (결정 계수)**
- 0~1 사이의 값 (1에 가까울수록 좋음)
- "모델이 데이터의 변동성을 얼마나 설명하는가?"

### 🎯 어떤 지표를 사용할까?

| 상황 | 중요한 지표 |
|------|-------------|
| 일반적인 분류 | 정확도, F1 |
| 의료 진단 (놓치면 안 됨) | 재현율 |
| 스팸 필터 (오진 곤란) | 정밀도 |
| 가격 예측 | MAE, R² |

## 10. 머신러닝 프로젝트의 전체 흐름

실제 머신러닝 프로젝트는 어떤 단계로 진행될까요?

### 🔄 머신러닝 워크플로우

```
1. 문제 정의
        ↓
2. 데이터 수집
        ↓
3. 데이터 탐색 및 전처리
        ↓
4. 모델 선택
        ↓
5. 모델 학습
        ↓
6. 모델 평가
        ↓
7. 모델 개선 (4~6 반복)
        ↓
8. 배포 (Deployment)
        ↓
9. 모니터링 및 유지보수
```

### 단계별 자세히 보기

#### 1️⃣ 문제 정의
- **무엇을** 예측/분류할 것인가?
- 머신러닝이 정말 필요한가?
- 성공의 기준은 무엇인가?

> 예: "고객 이탈을 예측해서 90% 정확도로 잡아내자"

#### 2️⃣ 데이터 수집
- 어디서 데이터를 가져올 것인가? (DB, API, 웹 크롤링 등)
- 데이터의 양은 충분한가?
- 개인정보 등 법적 문제는 없는가?

#### 3️⃣ 데이터 탐색 및 전처리

이 단계가 **전체 시간의 70~80%**를 차지합니다!

**탐색**:
- 데이터 분포 확인
- 결측치(빈 값) 확인
- 이상치(이상한 값) 확인

**전처리**:
- **결측치 처리**: 빈 값을 평균값으로 채우거나 제거
- **정규화(Normalization)**: 값의 범위를 0~1로 맞춤
- **인코딩(Encoding)**: 텍스트를 숫자로 변환 ("남"→0, "여"→1)
- **특성 엔지니어링**: 새로운 의미 있는 특성 만들기

> 예: "생년월일"에서 "나이" 추출하기

#### 4️⃣ 모델 선택
- 문제 유형에 맞는 알고리즘 선택
- 보통 여러 모델을 시도해보고 비교

#### 5️⃣ 모델 학습
- 학습 데이터로 모델 훈련
- **하이퍼파라미터(hyperparameter)** 조정
  - 학습률, 모델 크기 등 사람이 설정하는 값

#### 6️⃣ 모델 평가
- 검증 데이터로 성능 측정
- 과적합/과소적합 확인

#### 7️⃣ 모델 개선
- 더 많은 데이터
- 더 좋은 특성
- 다른 알고리즘 시도

#### 8️⃣ 배포
- 실제 서비스에 모델 적용
- 사용자가 사용할 수 있게 만들기
- **MLOps**의 영역

#### 9️⃣ 모니터링
- 시간이 지나면서 모델 성능이 떨어질 수 있음 (**모델 드리프트**)
- 새로운 데이터로 정기적으로 재학습 필요

- **친구 추천**: 그래프 분석
- **유해 콘텐츠 필터링**: 텍스트/이미지 분류
- **광고 타겟팅**: 사용자 행동 예측

### 🤖 최신 AI

- **ChatGPT, Claude**: 대규모 언어 모델 (LLM)
- **이미지 생성 (DALL-E, Midjourney)**: 생성형 AI

### 핵심 정리

이 강의에서 배운 핵심 개념을 정리해봅시다.

✅ **머신러닝**은 데이터에서 패턴을 학습하는 기술입니다.
✅ **AI ⊃ 머신러닝 ⊃ 딥러닝**의 포함 관계가 있습니다.
✅ **지도학습, 비지도학습, 강화학습**이 세 가지 주요 학습 방식입니다.
✅ **데이터의 양과 질**이 모델의 성능을 결정합니다.
✅ 학습은 **오차를 최소화하는 과정**입니다.
✅ **과적합**을 피하고 **일반화** 성능을 높여야 합니다.
✅ **올바른 평가 지표** 선택이 중요합니다.
✅ 머신러닝은 이미 우리 일상에 깊이 들어와 있습니다.

## 📖 부록: 핵심 용어 사전

| 용어 | 영어 | 의미 |
|------|------|------|
| 모델 | Model | 데이터로부터 학습한 예측 함수 |
| 특성 | Feature | 모델의 입력값 |
| 레이블 | Label | 정답값 |
| 학습 | Training | 모델이 데이터로부터 배우는 과정 |
| 추론 | Inference | 학습된 모델로 예측하는 과정 |
| 파라미터 | Parameter | 모델 내부에서 학습되는 값 |
| 하이퍼파라미터 | Hyperparameter | 사람이 설정하는 값 |
| 손실 함수 | Loss Function | 오차를 측정하는 함수 |
| 경사 하강법 | Gradient Descent | 손실을 줄이는 최적화 방법 |
| 에폭 | Epoch | 전체 데이터를 한 번 학습하는 단위 |
| 배치 | Batch | 한 번에 학습하는 데이터 묶음 |
| 과적합 | Overfitting | 학습 데이터에만 너무 잘 맞는 현상 |
| 과소적합 | Underfitting | 학습이 부족한 현상 |
| 일반화 | Generalization | 새 데이터에도 잘 작동하는 능력 |
| 정규화 | Regularization | 과적합을 막는 기법 |
