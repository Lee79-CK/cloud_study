I.

왜 모두가 PyTorch를 쓰는 걸까?

2017년 메타(당시 페이스북)에서 처음 공개된 PyTorch는 불과 몇 년 만에 연구실 노트북에서 거대 기업의 추론 서버까지, AI가 돌아가는 거의 모든 곳을 차지했습니다. 무엇이 이 라이브러리를 특별하게 만들었을까요?

1.1PyTorch는 한마디로

PyTorch는 "파이썬으로 다루는 GPU 기반 수치 계산 + 자동 미분 + 신경망" 라이브러리입니다. NumPy처럼 배열을 다루지만, 그 배열을 GPU로 옮겨 수십 배 빠르게 계산하고, 모든 연산의 그래디언트(미분값)를 알아서 추적합니다. 바로 이 두 가지가 딥러닝의 두 기둥입니다.

π

Pythonic

파이썬 그 자체처럼 동작합니다. 디버거를 걸어도, 중간에 print()를 찍어도 그대로 보입니다.

∂

Autograd

역전파(backpropagation)에 필요한 미분을 라이브러리가 자동으로 계산합니다. 사용자는 forward만 정의합니다.

⚡

GPU 가속

.to('cuda') 한 줄로 CPU 코드를 GPU 코드로 바꿉니다. CUDA 학습 없이도 가능합니다.

∞

거대한 생태계

HuggingFace, Lightning, TorchServe, vLLM 등 수많은 도구가 PyTorch 위에서 동작합니다.

1.2"Define-by-Run"이 뭐길래

PyTorch의 가장 큰 철학은 Define-by-Run (또는 동적 계산 그래프, Dynamic Computational Graph) 방식입니다. 코드를 실행하는 그 순간에 계산 그래프가 만들어지고, 다음 줄에서 다른 그래프를 만들 수도 있습니다. 반대로 옛날 TensorFlow 1.x는 Define-and-Run이었습니다 — 먼저 그래프를 "선언"한 다음에야 실행할 수 있었죠.

왜 중요한가

Define-by-Run 방식에서는 if, for 같은 평범한 파이썬 제어문이 모델 안에서 그냥 작동합니다. 예를 들어 입력에 따라 신경망의 깊이가 달라지는 모델, RNN처럼 시퀀스 길이가 매번 달라지는 모델을 자연스럽게 짤 수 있습니다. 이게 바로 연구자들이 PyTorch에 열광한 이유입니다.

II.

PyTorch vs. 다른 라이브러리들

ML/DL 라이브러리는 PyTorch 하나만 있는 게 아닙니다. TensorFlow, JAX, Keras, scikit-learn — 각각이 다른 시기에 다른 목적으로 만들어졌습니다. 어떤 도구를 언제 써야 하는지, 포지셔닝부터 명확히 정리해봅시다.

2.1같은 일을 4가지 방식으로

"두 텐서를 더해서 GPU에서 미분한다" — 이 단순한 작업을 각 라이브러리가 어떻게 다르게 표현하는지 비교해보면, 각자의 철학이 한눈에 들어옵니다.

파이썬으로 그냥 코드를 쓰듯이 자연스럽게 흘러갑니다.

pytorch_example.py
import torch

# 1. 텐서 생성 (requires_grad=True 로 미분 추적)
x = torch.tensor([2.0, 3.0], requires_grad=True)
y = torch.tensor([1.0, 4.0], requires_grad=True)

# 2. 평범한 연산 — 그래프는 자동으로 만들어진다
z = (x + y).sum()

# 3. 미분: backward() 한 줄로 끝
z.backward()
print(x.grad)   # tensor([1., 1.])

미분을 추적하려면 tf.GradientTape 컨텍스트가 필요합니다. 약간의 "껍데기"가 더 붙죠.

tensorflow_example.py
import tensorflow as tf

x = tf.Variable([2.0, 3.0])
y = tf.Variable([1.0, 4.0])

# GradientTape 컨텍스트 안의 연산만 기록된다
with tf.GradientTape() as tape:
    z = tf.reduce_sum(x + y)

dx = tape.gradient(z, x)
print(dx)   # tf.Tensor([1., 1.], ...)

JAX는 함수형입니다. 함수에 grad를 씌워서 "미분 함수"를 새로 만든다는 발상이죠.

jax_example.py
import jax
import jax.numpy as jnp

def f(x, y):
    return jnp.sum(x + y)

# grad(f) 자체가 "f를 x로 미분한 함수"
grad_fn = jax.grad(f, argnums=0)

x = jnp.array([2.0, 3.0])
y = jnp.array([1.0, 4.0])
print(grad_fn(x, y))   # [1., 1.]

scikit-learn은 딥러닝이나 자동미분을 지원하지 않습니다. 대신 고전적 ML 알고리즘에 강점이 있죠.

sklearn_example.py
from sklearn.linear_model import LinearRegression
import numpy as np

X = np.array([[1], [2], [3], [4]])
y = np.array([2, 4, 6, 8])

model = LinearRegression()
model.fit(X, y)
print(model.coef_)   # [2.]  — 알고리즘이 알아서 풀어준다
언제 무엇을

표 형식 데이터(CSV처럼 행/열로 구성된 데이터)에 회귀·분류 알고리즘을 빠르게 시도하고 싶다면 scikit-learn이 압도적으로 편합니다. 반면 이미지·자연어·음성처럼 텐서 단위로 처리해야 하는 데이터는 PyTorch가 정답입니다.

결론

오늘날 새로운 AI 연구의 약 80~90%가 PyTorch로 출판됩니다 (Papers with Code 기준). LLM 시대의 사실상 표준 도구이며, HuggingFace Transformers, vLLM, Lightning, Diffusers 같은 거의 모든 메이저 도구가 PyTorch 위에 만들어집니다. 처음 배우는 사람에게도, 산업 현장의 엔지니어에게도 PyTorch가 가장 합리적인 출발점입니다.

III.

텐서 — 모든 것은 숫자의 박스에서 시작한다

PyTorch의 모든 데이터는 텐서(Tensor)라는 그릇에 담깁니다. 텐서는 어렵게 말하면 "n차원 배열", 쉽게 말하면 "여러 차원으로 쌓인 숫자들"입니다. 이미지도, 문장도, 음성도 결국엔 텐서로 표현됩니다.

3.1차원(rank)으로 이해하기

텐서의 차원(rank)에 따른 모양과 실제 사용 예
  • 스칼라(0-rank): 숫자 하나. 손실(loss) 값이 대표적인 예입니다.
  • 벡터(1-rank): 1차원 배열. 한 단어의 임베딩이 보통 768~4096차원짜리 벡터입니다.
  • 행렬(2-rank): 표 같은 모양. 흑백 이미지 한 장은 (높이, 너비)의 2D 행렬입니다.
  • 3차원 이상: 컬러 이미지(RGB 3채널)는 (3, H, W). 배치까지 포함하면 (배치크기, 3, H, W)의 4차원이 됩니다.

3.2텐서 만들고 다루기

IV.

Autograd — 미분을 자동으로

신경망을 학습한다는 것은 결국 "현재 모델이 얼마나 틀렸는지(손실)"를 정의하고, 그 손실을 줄이는 방향으로 가중치를 조금씩 움직이는 일입니다. "어떤 방향으로 움직여야 하는지"를 알려면 미분(gradient)이 필요한데, PyTorch는 이걸 자동으로 해줍니다. 그 마법의 이름이 Autograd입니다.

4.1계산 그래프(Computational Graph)

requires_grad=True로 만든 텐서에 어떤 연산을 가하면, PyTorch는 뒤에서 조용히 그래프를 그립니다 — 어떤 텐서가 어떤 연산을 거쳐 어떤 텐서가 되었는지를 모두 기억해두는 거죠.

Forward(검정 화살표): 입력 x, w, b로부터 손실 L을 계산. Backward(주황 점선): L에서 시작해 거꾸로 거슬러 올라가며 각 변수의 미분값을 구함.

4.2실제로 어떻게 보이나

autograd_example.py
import torch

# 미분이 필요한 가중치 — requires_grad=True가 핵심
w = torch.tensor(3.0, requires_grad=True)
b = torch.tensor(1.0, requires_grad=True)
x = torch.tensor(2.0)   # 입력 데이터는 보통 미분 대상이 아님

# Forward — 그래프가 자동으로 만들어진다
y = w * x + b           # y = 3*2 + 1 = 7
loss = (y - 5) ** 2      # 정답이 5라고 가정 → loss = 4

# Backward — 단 한 줄로 모든 미분이 계산된다
loss.backward()

print(w.grad)   # dL/dw = 2*(y-5)*x = 2*2*2 = 8
print(b.grad)   # dL/db = 2*(y-5)*1 = 2*2*1 = 4

이게 전부입니다. 우리는 forward 계산만 정의했을 뿐인데, PyTorch가 알아서 연쇄 법칙(chain rule)을 따라 모든 변수에 대한 미분을 채워넣었습니다. 신경망이 가중치를 100만 개 가지고 있어도 똑같이 작동합니다.

자주 하는 실수

grad는 누적됩니다. backward()를 두 번 호출하면 그래디언트가 더해집니다. 그래서 학습 루프마다 optimizer.zero_grad()로 초기화해야 합니다 (뒤에서 다룹니다).

V.

신경망을 짜는 단 하나의 패턴

PyTorch에서 신경망은 torch.nn.Module이라는 클래스를 상속해서 만듭니다. 여기엔 단 두 가지 규칙만 있어요 — (1) 생성자에서 레이어를 정의하고, (2) forward에서 데이터가 어떻게 흐르는지를 쓴다. 그게 전부입니다.

5.1가장 작은 신경망

simple_net.py
import torch
import torch.nn as nn

class SimpleNet(nn.Module):
    def __init__(self):
        super().__init__()
        # (1) 레이어 선언 — 학습되는 가중치들이 여기 들어있다
        self.fc1 = nn.Linear(in_features=784, out_features=128)
        self.fc2 = nn.Linear(128, 10)
        self.relu = nn.ReLU()

    def forward(self, x):
        # (2) 데이터가 어떻게 흐르는지만 적으면 끝
        x = self.fc1(x)        # (배치, 784) → (배치, 128)
        x = self.relu(x)
        x = self.fc2(x)        # (배치, 128) → (배치, 10)
        return x

model = SimpleNet()
x = torch.randn(32, 784)   # 28x28 이미지 32장을 한 줄로 편 것
out = model(x)             # model.forward(x)를 호출
print(out.shape)         # torch.Size([32, 10]) — 10개 클래스 점수

이 코드는 손글씨 숫자(MNIST) 분류를 위한 가장 단순한 신경망입니다. 28×28 픽셀(=784개)을 받아 128차원으로 압축한 뒤, 0~9 10개 클래스의 점수를 내놓습니다. 한 가지 신기한 점은 model(x)로 호출했지 model.forward(x)로 호출하지 않았다는 것 — 그래도 결과는 같습니다. nn.Module의 __call__이 자동으로 forward를 부르되, 훅(hook)과 그래프 추적 같은 부가 처리를 함께 해주거든요.

5.2주요 레이어 한 눈에

L

nn.Linear

완전연결층(Fully Connected). y = xW + b. 가장 기본이 되는 레이어.

C

nn.Conv2d

2D 합성곱. 이미지의 지역적 특징을 추출. CNN의 핵심 빌딩 블록.

R

nn.LSTM / GRU

순환 신경망. 시퀀스(시계열, 텍스트)를 처리. 요즘은 Transformer에 자리를 내줌.

T

nn.Transformer
nn.MultiheadAttention

현대 LLM의 심장. GPT, BERT, Llama 모두 이 위에서 만들어진다.

N

nn.BatchNorm
nn.LayerNorm

정규화 레이어. 학습 안정성과 속도를 크게 개선해준다.

D

nn.Dropout

학습 중에 일부 뉴런을 무작위로 끄는 정규화 기법. 과적합 방지의 고전.

5.3활성화 함수와 손실 함수

신경망은 그냥 행렬 곱셈만으로는 비선형 패턴을 학습할 수 없습니다. 그래서 레이어 사이에 활성화 함수를 끼워 넣습니다. 그리고 학습을 위해 "얼마나 틀렸는지"를 재는 손실 함수가 필요합니다.

자주 쓰는 활성화 함수

  • nn.ReLU — 음수는 0, 양수는 그대로. 가장 흔함.
  • nn.GELU — Transformer 계열의 표준.
  • nn.SiLU (Swish) — LLaMA 등 최근 모델에서 사용.
  • nn.Sigmoid — 0~1 출력. 이진 분류 마지막 단.
  • nn.Softmax — 다중 분류에서 확률로 변환.

자주 쓰는 손실 함수

  • nn.MSELoss — 회귀(연속값 예측)의 표준.
  • nn.CrossEntropyLoss — 다중 분류의 표준.
  • nn.BCEWithLogitsLoss — 이진 분류 (수치적으로 안정).
  • nn.L1Loss — 절댓값 오차. 이상치에 강함.
  • nn.CTCLoss — 음성 인식 등 시퀀스 정렬 문제용.
VI.

학습 루프 — 반복되는 5단계

PyTorch로 어떤 모델을 학습하든 결국 똑같은 5단계가 반복됩니다. 이 패턴을 손에 익히는 게 PyTorch 학습의 90%입니다.

1
Forward

모델에 입력을 통과시켜 예측값을 얻는다

→
2
Loss

예측과 정답의 차이(손실)를 계산

→
3
Zero Grad

이전 그래디언트를 초기화

→
4
Backward

모든 파라미터의 그래디언트 계산

→
5
Step

옵티마이저가 가중치를 갱신

6.1코드로 보면

training_loop.py
import torch
import torch.nn as nn

model = SimpleNet()
loss_fn = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(num_epochs):
    for x, y in train_loader:        # DataLoader가 미니배치 공급
        x, y = x.to(device), y.to(device)

        # ① Forward
        logits = model(x)

        # ② Loss
        loss = loss_fn(logits, y)

        # ③ Zero Grad — 이전 스텝의 grad 누적을 제거
        optimizer.zero_grad()

        # ④ Backward — 모든 파라미터의 .grad가 채워진다
        loss.backward()

        # ⑤ Step — w ← w - lr * grad (Adam의 경우 좀 더 똑똑하게)
        optimizer.step()

    print(f"Epoch {epoch}  loss={loss.item()}")

6.2옵티마이저(Optimizer)란?

옵티마이저는 그래디언트를 받아서 가중치를 어떻게 갱신할지 결정합니다. 가장 단순한 SGD부터 시작해, 학습률을 적응적으로 조절하는 Adam, AdamW, RMSprop 등 다양한 선택지가 있습니다. 처음이라면 Adam(lr=1e-3)이 사실상 안전한 기본값입니다.

6.3Dataset과 DataLoader

대규모 데이터를 한 번에 메모리에 올릴 수는 없습니다. PyTorch는 데이터를 효율적으로 공급하기 위해 두 가지 추상화를 제공합니다 — Dataset은 "데이터 한 개를 어떻게 꺼낼지", DataLoader는 "배치 단위로, 셔플하면서, 여러 워커로 병렬로" 꺼낼지를 담당합니다.

dataset_and_loader.py
from torch.utils.data import Dataset, DataLoader

class MyDataset(Dataset):
    def __init__(self, images, labels):
        self.images, self.labels = images, labels

    def __len__(self):
        return len(self.images)          # 전체 샘플 수

    def __getitem__(self, idx):
        return self.images[idx], self.labels[idx]

train_set = MyDataset(train_imgs, train_lbls)
train_loader = DataLoader(
    train_set,
    batch_size=64,        # 한 번에 64개씩
    shuffle=True,         # 에폭마다 순서를 섞는다
    num_workers=4,        # 4개 프로세스로 병렬 로딩
    pin_memory=True,      # GPU 전송 속도 ↑
)
실전 팁

이미지·텍스트 같은 흔한 데이터셋은 직접 만들 필요가 없습니다. torchvision.datasets, torchaudio.datasets, HuggingFace의 datasets 라이브러리가 미리 준비된 Dataset 객체를 제공합니다.

VII.

전부 합쳐보기 — MNIST 분류기

지금까지 배운 모든 조각을 모아서, 손글씨 숫자(MNIST)를 분류하는 완전한 프로그램을 만들어봅시다. 데이터 로딩 · 모델 정의 · 학습 루프 · 평가까지 약 40줄 안에 끝납니다. 이게 PyTorch의 진짜 매력입니다.

mnist_classifier.py
import torch
import torch.nn as nn
from torch.utils.data import DataLoader
from torchvision import datasets, transforms

device = "cuda" if torch.cuda.is_available() else "cpu"

# ─── 1. 데이터 준비 ───────────────────────────────────
tfm = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.1307,), (0.3081,)),   # MNIST 평균/표준편차
])
train_ds = datasets.MNIST("./data", train=True, download=True, transform=tfm)
test_ds  = datasets.MNIST("./data", train=False, transform=tfm)
train_dl = DataLoader(train_ds, batch_size=128, shuffle=True)
test_dl  = DataLoader(test_ds,  batch_size=256)

# ─── 2. 모델 정의 (간단한 CNN) ────────────────────────
class CNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(1, 32, 3, padding=1), nn.ReLU(), nn.MaxPool2d(2),
            nn.Conv2d(32, 64, 3, padding=1), nn.ReLU(), nn.MaxPool2d(2),
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(64 * 7 * 7, 128), nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(128, 10),
        )
    def forward(self, x):
        return self.classifier(self.features(x))

model = CNN().to(device)
loss_fn = nn.CrossEntropyLoss()
opt = torch.optim.Adam(model.parameters(), lr=1e-3)

# ─── 3. 학습 루프 ──────────────────────────────────────
for epoch in range(3):
    model.train()
    for x, y in train_dl:
        x, y = x.to(device), y.to(device)
        opt.zero_grad()
        loss = loss_fn(model(x), y)
        loss.backward()
        opt.step()

    # ─── 4. 평가 ─────────────────────────────────────────
    model.eval()
    correct = total = 0
    with torch.no_grad():           # 추론에서는 grad 추적 끄기 (메모리·속도 ↑)
        for x, y in test_dl:
            x, y = x.to(device), y.to(device)
            pred = model(x).argmax(dim=1)
            correct += (pred == y).sum().item()
            total   += y.size(0)
    print(f"Epoch {epoch}  acc={100*correct/total:.2f}%")

약 3 에폭만 돌려도 보통 98% 이상의 정확도가 나옵니다. 위 코드는 PyTorch로 짠 거의 모든 학습 코드의 뼈대와 동일한 구조를 가지고 있습니다. LLaMA를 파인튜닝하든, Diffusion 모델을 학습하든, 본질은 똑같습니다.

두 가지 모드

model.train()과 model.eval()의 차이는 Dropout과 BatchNorm 같은 레이어의 동작에 영향을 줍니다. 학습 중에는 Dropout이 일부 뉴런을 꺼야 하지만, 평가에서는 꺼선 안 되거든요. 잊지 말고 토글하세요.

스스로 점검해보기 — Quiz

지금까지 배운 내용을 확인해보세요. 정답을 누르면 곧바로 해설이 나타납니다. 틀려도 괜찮아요 — 헷갈리는 부분이 어디인지 알아내는 게 더 중요합니다.

Q1.다음 중 PyTorch의 동작 방식을 가장 잘 설명한 것은?
정답: 2번. PyTorch는 Define-by-Run 방식이라 일반 파이썬 흐름(if, for)을 모델 안에서 그대로 쓸 수 있습니다. 이 점이 연구자들이 PyTorch를 선호하는 핵심 이유입니다.
Q2.학습 루프에서 optimizer.zero_grad()를 부르지 않으면 어떤 일이 생기나요?
정답: 3번. PyTorch는 backward() 시 그래디언트를 누적합니다(RNN 같은 곳에서 유용한 동작). 그래서 일반적인 학습에서는 매 스텝마다 명시적으로 초기화해야 합니다.
Q3.다음 중 PyTorch보다 scikit-learn이 더 적합한 작업은?
정답: 1번. scikit-learn은 표 형식 데이터의 고전적 ML 알고리즘에 최적화되어 있습니다. 딥러닝이나 GPU 가속이 필요한 작업은 PyTorch가 정답입니다.
Q4.torch.tensor(2.0, requires_grad=True)에서 requires_grad=True가 의미하는 것은?
정답: 2번. requires_grad=True인 텐서는 autograd가 연산을 추적하므로, 이후 backward()를 호출하면 미분값이 .grad에 채워집니다.
Q5.nn.Module을 상속해 신경망을 만들 때 반드시 정의해야 하는 메서드는?
정답: 4번. __init__에서 레이어를 선언하고, forward에서 데이터 흐름을 정의합니다. backward는 autograd가 자동으로 만들어줍니다.
