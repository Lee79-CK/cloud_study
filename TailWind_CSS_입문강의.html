Chapter 01

TailWind CSS,
도대체 뭐길래?

TailWind CSS는 미리 만들어진 작은 CSS 클래스들(유틸리티 클래스)을 HTML에 직접 조합해서 사용하는 유틸리티 우선(Utility-First) CSS 프레임워크입니다.

기존에는 .my-button 같은 클래스를 만들고 CSS 파일에 스타일을 작성했죠. TailWind는 다릅니다. bg-blue-500, p-4, rounded-lg 같은 의미가 정해진 작은 클래스들을 HTML에서 바로 조합합니다.

🎨

디자인 시스템

색상, 간격, 폰트 크기가 미리 정의돼 일관된 UI를 만들 수 있어요.

⚡

빠른 개발

CSS 파일을 오갈 필요 없이 HTML에서 바로 스타일링이 가능합니다.

🪶

가벼운 결과물

실제 사용된 클래스만 빌드되어 최종 CSS 용량이 매우 작아요.

🧠

이름 고민 끝

.card__title--big 같은 클래스명 짓기에서 해방됩니다.

💡 핵심 비유: 일반 CSS가 "가구를 직접 만드는 목공"이라면, TailWind는 "레고 블록 조립"입니다. 정해진 블록(클래스)을 조합해서 원하는 모양을 빠르게 만드는 방식이죠.

Chapter 02

일반 CSS vs TailWind CSS,
같은 결과, 다른 길.

똑같은 디자인을 두 가지 방식으로 만들어 봅시다. 아래 탭을 눌러서 코드를 비교해보세요.

예제 1. 기본 버튼 만들기

파란 배경, 흰 글자, 둥근 모서리, 패딩이 있는 버튼.

일반 CSS 방식
CSS + HTML 분리 
/* style.css */
.my-button {
  background-color: #3b82f6;
  color: white;
  padding: 12px 24px;
  border-radius: 8px;
  border: none;
  font-weight: 600;
}
.my-button:hover {
  background-color: #2563eb;
}

<!-- index.html -->
<button class="my-button">
  눌러보세요
</button>
TailWind CSS 방식
HTML 한 곳에서 끝! 
<!-- index.html  (CSS 파일 불필요) -->
<button class="bg-blue-500
               hover:bg-blue-600
               text-white
               font-semibold
               py-3 px-6
               rounded-lg">
  눌러보세요
</button>
📄 파일 수
CSS: 2개 (HTML + CSS) ↔ TW: 1개
🧠 클래스 이름
CSS: 직접 작명 필요 ↔ TW: 정해진 이름
🔁 재사용
CSS: 클래스 재사용 ↔ TW: 컴포넌트 단위 재사용

예제 2. 카드 컴포넌트

이미지가 있는 정보 카드를 만들어볼까요?

일반 CSS — 길어집니다. 
.card {
  background: white;
  border-radius: 12px;
  box-shadow: 0 4px 6px rgba(0,0,0,.1);
  overflow: hidden;
  max-width: 320px;
}
.card__body {
  padding: 20px;
}
.card__title {
  font-size: 20px;
  font-weight: 700;
  margin-bottom: 8px;
}
.card__text {
  color: #6b7280;
  line-height: 1.6;
}
TailWind — HTML 한 줄로 끝! 
<div class="bg-white rounded-xl
            shadow-md overflow-hidden
            max-w-xs">
  <div class="p-5">
    <h3 class="text-xl font-bold
                mb-2">
      카드 제목
    </h3>
    <p class="text-gray-500
                leading-relaxed">
      간결한 본문 텍스트입니다.
    </p>
  </div>
</div>

카드 제목

간결한 본문 텍스트입니다.

↑ 두 방식 모두 위와 같은 결과를 만듭니다.

🤔 "HTML이 길어지지 않나요?" 처음에는 그렇게 느껴집니다. 하지만 ① 파일을 오갈 필요가 없고, ② 이 요소가 어떻게 보일지 한눈에 파악되며, ③ React/Vue 같은 컴포넌트 단위에서 재사용하면 단점이 거의 사라집니다.

Chapter 03

핵심 문법 6가지.
이것만 알면 90%는 끝.

TailWind 클래스는 일정한 패턴을 따릅니다. 카테고리별로 익혀봅시다.

1. 색상 (Color)

패턴: 속성-색이름-진하기

bg-red-500→ 빨강 배경
bg-blue-300→ 연한 파랑 배경
bg-green-700→ 진한 초록 배경
Atext-pink-500→ 분홍 글자
진하기는 50, 100, 200 ... 900, 950까지. 숫자가 클수록 진해집니다.
⇔

2. 간격 (Spacing)

패턴: 속성-방향-크기

p-4  = 사방 패딩 (1rem)
px-6 = 좌우 패딩
py-2 = 상하 패딩
pt-8 = 위쪽만
m-4  = 사방 margin
mx-auto = 좌우 자동 (가운데 정렬)
크기 단위: 0, 0.5, 1, 2, 3, 4, 6, 8, 12, 16, 24, 32... (1 = 0.25rem = 4px)
↔

3. 크기 (Sizing)

패턴: w-크기 / h-크기

w-32 = 너비 8rem (128px)
w-full = 너비 100%
w-1/2 = 너비 50%
h-screen = 높이 화면 전체
min-h-0, max-w-md
w-1/3, w-2/3, w-1/4, w-3/5 같은 분수 표기도 가능합니다.
Aa

4. 글꼴 (Typography)

text-xs = 매우 작게 (0.75rem)
text-base = 기본 (1rem)
text-2xl, text-5xl = 점점 크게
font-bold, font-semibold = 굵기
text-center = 가운데 정렬
leading-relaxed = 줄간격 넓게
italic, underline = 기울임/밑줄

5. 레이아웃 (Flex)

flex = display:flex
items-center = 세로 가운데
justify-center = 가로 가운데
justify-between = 양끝 정렬
gap-4 = 자식 간 간격
flex-col = 세로 방향
flex-wrap = 줄바꿈 허용

6. 테두리·그림자

rounded = 약간 둥글게
rounded-lg / rounded-full
border = 1px 회색 테두리
border-2 border-red-500
shadow-sm / shadow-md / shadow-2xl
opacity-50 = 반투명

Chapter 04

반응형 디자인,
접두어 하나로 끝.

화면 크기에 따라 다른 스타일을 적용하려면 sm:, md:, lg: 같은 접두어(prefix)를 붙이면 됩니다.

접두어 최소 너비 대상 기기 예시
(없음) 0px ~ 모바일 (기본) text-sm
sm: 640px ~ 큰 모바일 / 작은 태블릿 sm:text-base
md: 768px ~ 태블릿 md:text-lg
lg: 1024px ~ 노트북 lg:text-xl
xl: 1280px ~ 데스크탑 xl:text-2xl
2xl: 1536px ~ 큰 모니터 2xl:text-3xl

📱 모바일 우선 (Mobile-First)

접두어 없는 클래스가 기본값(모바일)이고, md: 같은 접두어는 "그 크기 이상에서 덮어쓴다"는 의미입니다. 

<p class="text-sm      /* 모바일: 작게  */
           md:text-base /* 태블릿~: 보통 */
           lg:text-xl   /* 노트북~: 크게 */
           xl:text-3xl"> /* 데스크탑: 매우 크게 */
  화면이 커질수록 저는 커집니다.
</p>
↓ 브라우저 창 크기를 바꿔보세요 ↓

화면이 커질수록 저는 커집니다.

현재 화면: 계산 중...

<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4"> ... </div>

Chapter 05

상태(State)에 따라 스타일 바꾸기

마우스를 올렸을 때, 클릭했을 때, 포커스가 되었을 때 스타일을 바꾸려면 어떻게 할까요? 또 다른 접두어를 쓰면 됩니다.

hover:

마우스 올렸을 때

hover:bg-twcyan-600
focus:

포커스 됐을 때

focus:border-twcyan-500
active:

클릭하는 순간

active:scale-95
disabled:

비활성 상태

disabled:bg-ink-300

💡 조합 규칙: 여러 접두어를 함께 쓸 수 있습니다. md:hover:bg-red-500은 "태블릿 이상에서, 호버 시, 빨간 배경"이라는 뜻이에요.

Chapter 08

시작하기 — 세 가지 방법.

가장 쉬운 방법부터 차근차근 알려드릴게요.

1

CDN (가장 쉬움 · 초보자 추천)

HTML 파일에 한 줄만 추가하면 바로 사용 가능합니다. 학습용·간단한 프로토타입에 최적이에요. (지금 보고 있는 이 강의 페이지도 이 방법을 씁니다!) 

<!DOCTYPE html>
<html>
<head>
  <script src="https://cdn.tailwindcss.com"></script>
</head>
<body>
  <h1 class="text-3xl font-bold text-blue-600">
    바로 TailWind 작동!
  </h1>
</body>
</html>

⚠️ 주의: 실제 서비스에는 권장되지 않습니다. 빌드된 파일을 사용하세요.

2

npm 설치 (실무 표준)

Node.js 환경에서 빌드 도구와 함께 사용. 사용하지 않는 클래스는 자동으로 제거되어 최종 CSS 크기가 매우 작아집니다. 

# 1. 설치
npm install -D tailwindcss
npx tailwindcss init

# 2. tailwind.config.js 수정
content: ["./src/**/*.{html,js}"]

# 3. input.css 작성
@tailwind base;
@tailwind components;
@tailwind utilities;

# 4. 빌드
npx tailwindcss -i ./input.css -o ./output.css --watch
3

프레임워크 통합 (React, Vue, Next.js 등)

대부분의 모던 프레임워크에서 공식 가이드를 제공합니다. create-next-app처럼 초기 설정 시 TailWind 옵션을 선택할 수 있어요.

Next.js
create-next-app
Vite
create-vite + 플러그인
Nuxt
@nuxtjs/tailwindcss
Chapter 09 · Wrap Up

이제 당신은 TailWind를
읽을 줄 압니다.

핵심 5가지

  1. 1. 클래스를 HTML에 직접 적는다.
  2. 2. 패턴: 속성-값-진하기
  3. 3. 접두어로 반응형: md:
  4. 4. 접두어로 상태: hover:
  5. 5. 모르는 클래스는 공식 문서로!

다음 학습 단계

  1. → tailwind.config.js 커스터마이징
  2. → 다크모드 (dark: prefix)
  3. → @apply 디렉티브
  4. → 임의 값 [w-[173px]]
  5. → 플러그인 (Forms, Typography 등)
