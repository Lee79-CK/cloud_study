## 1. 인공신경망이 도대체 뭔가요?

### 🍎 일상의 예시부터 시작해 봅시다

여러분이 친구의 얼굴을 알아보는 과정을 한번 생각해 볼까요?

> "어, 저기 멀리 걸어오는 사람... 키가 비슷하고, 머리 스타일이 비슷하고, 걷는 자세도 익숙하네. **아, 영희구나!**"

여러분은 이 판단을 **0.5초** 만에 합니다. 그런데 컴퓨터에게 "친구를 알아봐"라고 시키면 어떨까요?

### 🤖 전통적인 프로그래밍의 한계

옛날 방식의 프로그래머라면 이렇게 코드를 짤 겁니다:

```
IF 키가 165cm이고
   AND 머리가 갈색이고
   AND 안경을 쓰고 있으면
THEN 영희다
```

문제점이 보이시나요?
- 영희가 안경을 벗으면? → 못 알아봄 ❌
- 영희가 머리를 염색하면? → 못 알아봄 ❌
- 어두운 곳에 있으면? → 못 알아봄 ❌

**규칙을 일일이 다 적는 건 불가능**합니다.

### 💡 새로운 접근: "직접 가르치지 말고, 스스로 배우게 하자!"

인공신경망은 이렇게 작동합니다:

> "여기 영희 사진 1,000장이 있어. 그리고 다른 사람들 사진도 1,000장이 있어. 자, 직접 봐 봐. 어떤 게 영희인지 **너 스스로 패턴을 찾아내**."

**이게 바로 인공신경망입니다.** 사람이 규칙을 정해주는 게 아니라, 데이터를 보고 스스로 패턴을 찾아내는 시스템이죠.

### 🎯 한 줄 요약

> **인공신경망(Artificial Neural Network, ANN)이란?**  
> 사람의 뇌가 작동하는 방식에서 영감을 받아 만든, **데이터로부터 스스로 패턴을 학습하는 컴퓨터 시스템**

## 2. 우리 뇌에서 영감을 얻다 - 뉴런 이야기

### 🧠 사람의 뇌는 어떻게 작동할까?

우리 뇌에는 약 **1,000억 개**의 **뉴런(Neuron, 신경세포)**이 있습니다. 그리고 이 뉴런들은 서로 연결되어 있어요.

뉴런 하나의 모습을 그려보면:

```
        가지돌기 (입력 받는 부분)
         \   |   /
          \  |  /
           \ | /
            \|/
        ┌────●────┐ ← 세포체 (판단하는 부분)
        │   뉴런   │
        └────┬────┘
             │
             ↓
          축삭돌기 (다른 뉴런에게 신호 보내는 부분)
             │
             ↓
            ◯◯◯ ← 다음 뉴런들로 연결
```

### 🔌 뉴런이 작동하는 방식

1. **신호 받기**: 가지돌기로 다른 뉴런들에게서 전기 신호를 받습니다.
2. **신호 합치기**: 받은 신호들을 모두 더합니다.
3. **판단하기**: 합쳐진 신호가 일정 수준(임계값)을 넘으면 → "발화(fire)"한다!
4. **신호 보내기**: 발화한 신호를 축삭돌기를 통해 다음 뉴런에게 전달합니다.

> 💡 **비유**: 뉴런은 마치 **회사 직원**과 같습니다.  
> 여러 동료(다른 뉴런)에게서 정보를 받아서, 종합한 다음, 중요하다고 판단되면 상사(다음 뉴런)에게 보고하는 거죠.

### 🤝 뉴런 간 연결의 강도

여기서 중요한 점! 뉴런 간 연결은 **모두 같은 강도가 아닙니다.**

- 친한 친구 뉴런: 신호를 강하게 전달 (강한 연결)
- 별로 안 친한 뉴런: 신호를 약하게 전달 (약한 연결)

이 **연결 강도**가 바로 인공신경망에서 말하는 **가중치(Weight)**입니다. (뒤에서 자세히 다룰 거예요!)

## 3. 인공 뉴런의 구조 - 퍼셉트론

### 🎭 인공 뉴런 = 사람 뉴런의 단순화 버전

과학자들이 이런 생각을 했어요.

> "사람 뉴런을 컴퓨터로 흉내 낼 수 있을까? 아주 단순하게라도?"

그래서 1957년, **프랭크 로젠블랫(Frank Rosenblatt)**이라는 학자가 **퍼셉트론(Perceptron)**이라는 가장 단순한 형태의 인공 뉴런을 만들었습니다.

### 🧮 퍼셉트론의 구조

```
입력 1 (x₁) ──[가중치 w₁]──┐
                           │
입력 2 (x₂) ──[가중치 w₂]──┼──▶ [모두 더하기 + 편향] ──▶ [활성화 함수] ──▶ 출력
                           │
입력 3 (x₃) ──[가중치 w₃]──┘
```

수식으로 표현하면:

```
출력 = 활성화함수( x₁×w₁ + x₂×w₂ + x₃×w₃ + 편향 )
```

겁먹지 마세요! 한 단계씩 풀어볼게요.

### 🍕 피자 주문으로 이해하는 퍼셉트론

오늘 저녁에 피자를 시켜 먹을지 결정한다고 가정해봅시다.

**고려할 요소(입력값)**:
- x₁ = 배가 고픈가? (1: 매우 고픔, 0: 안 고픔)
- x₂ = 돈이 있는가? (1: 있음, 0: 없음)
- x₃ = 시간이 늦었는가? (1: 늦음, 0: 안 늦음)

**각 요소의 중요도(가중치)**:
- w₁ = 0.5 (배고픔이 매우 중요)
- w₂ = 0.8 (돈이 가장 중요)
- w₃ = -0.3 (늦으면 살짝 꺼려짐)

**오늘의 상황**:
- 배고픔 = 1, 돈 있음 = 1, 늦었음 = 1

**계산**:
```
점수 = (1 × 0.5) + (1 × 0.8) + (1 × -0.3) = 0.5 + 0.8 - 0.3 = 1.0
```

**결정 (활성화 함수)**: 점수가 0.5 이상이면 → 피자 주문! ✅

이게 바로 퍼셉트론이 하는 일입니다. **여러 정보를 받아서, 중요도를 곱한 뒤, 합쳐서, 결정을 내리는 것!**

## 4. 가중치(Weight)와 편향(Bias)

### ⚖️ 가중치란?

**가중치(Weight)**는 한마디로 **"이 정보가 얼마나 중요한가?"**를 나타내는 숫자입니다.

#### 🍔 예시: 햄버거 맛집 평가

여러분이 햄버거집을 평가한다고 해봅시다. 무엇을 볼까요?

| 평가 항목 | 점수(0~10) | 중요도(가중치) |
|-----------|------------|----------------|
| 맛        | 9          | 0.7 (매우 중요) |
| 가격      | 5          | 0.2 (보통)     |
| 분위기    | 7          | 0.1 (덜 중요)  |

**총점 계산**:
```
9 × 0.7 + 5 × 0.2 + 7 × 0.1 = 6.3 + 1.0 + 0.7 = 8.0
```

이때 **0.7, 0.2, 0.1이 바로 가중치**입니다. 미식가에게는 맛이 가장 중요하니 0.7이라는 큰 가중치를 주는 거죠.

만약 가난한 학생이라면 가격의 가중치가 더 클 수 있겠죠?

### ➕ 편향(Bias)이란?

**편향(Bias)**은 일종의 **"기본 점수"** 또는 **"기준선 조정"**입니다.

#### 🎢 예시: 놀이공원 키 제한

놀이공원 롤러코스터를 탈 수 있는 기준이 **"키 140cm 이상"**이라고 합시다.

여기서 **140cm가 바로 편향(Bias)**입니다. 키가 140 미만이면 못 타고, 140 이상이면 탈 수 있죠.

수식으로:
```
탈 수 있는가? = 키 - 140 ≥ 0?
```

만약 어린이날 특별 행사라서 기준을 130cm로 낮추면? 편향이 130으로 바뀐 거죠. 그러면 더 많은 사람이 탈 수 있게 됩니다.

> 💡 **편향은 "이 뉴런이 얼마나 쉽게 활성화되는가"를 조절하는 다이얼**이라고 생각하시면 됩니다.

## 5. 활성화 함수 - 신경망에 생명을 불어넣다

### 🚦 활성화 함수가 왜 필요할까?

지금까지 우리는 입력값과 가중치를 곱한 후 더했습니다. 그런데 그 결과를 **그대로 사용하지 않고**, 한 번 더 **변환**을 합니다. 이때 사용하는 게 **활성화 함수(Activation Function)**입니다.

#### 🤔 왜 그래야 할까?

만약 활성화 함수가 없다면, 신경망이 아무리 깊어져도 결국은 **"단순한 직선"** 밖에 못 그립니다. 즉, 복잡한 패턴을 못 배워요!

활성화 함수는 신경망에 **"비선형성(non-linearity)"**을 주는 역할을 합니다. 어려운 말 같지만, **"꺾인 선이나 곡선도 그릴 수 있게 해준다"**는 뜻입니다.

### 📊 대표적인 활성화 함수 3가지

#### ① 시그모이드 (Sigmoid) - "0과 1 사이로 압축"

```
        1 ┤            ___─────
          │         _─
        0.5┤    ─_─
          │ _─
        0 ┤────
          └────────────────────
              -∞    0    +∞
```

- 어떤 숫자가 들어와도 **0과 1 사이의 값**으로 변환
- "이 뉴런이 발화할 확률은?" 처럼 **확률**을 표현할 때 자주 사용
- 단점: 깊은 신경망에서는 학습이 잘 안 될 수 있음

#### ② ReLU (Rectified Linear Unit) - "음수는 0, 양수는 그대로"

```
          │      /
          │     /
          │    /
          │   /
          │  /
        0 ┤────────
          └────────────────────
              -∞    0    +∞
```

- 음수가 들어오면 → 0
- 양수가 들어오면 → 그대로
- **현재 가장 인기 있는 활성화 함수**! 계산이 빠르고 학습이 잘 됨
- 딥러닝 모델에서 표준처럼 사용됨

#### ③ Softmax - "여러 개 중 하나 선택"

여러 카테고리 중 하나를 고를 때 사용합니다.

예시: 사진 한 장이 있을 때, 그게 고양이일 확률, 강아지일 확률, 토끼일 확률을 각각 계산:

```
고양이일 확률: 0.7  ← 가장 높음! "고양이"로 결정
강아지일 확률: 0.2
토끼일 확률:   0.1
─────────────────
합계:          1.0  ← 항상 합이 1이 되도록 만듦
```

> 💡 **요약**: 활성화 함수는 뉴런의 **"성격"**을 결정합니다. 어떤 함수를 쓰느냐에 따라 신경망이 풀 수 있는 문제의 종류가 달라져요.

## 6. 신경망의 전체 구조

### 🏗️ 뉴런 하나로는 부족하다

뉴런(퍼셉트론) 하나로는 단순한 문제만 풀 수 있어요. 복잡한 문제를 풀려면 **뉴런을 여러 개 모아야** 합니다. 이게 바로 **인공신경망**입니다.

### 🏛️ 신경망의 3층 구조

신경망은 보통 **3가지 층(Layer)**으로 구성됩니다.

```
[입력층]      [은닉층]       [출력층]
                                    
입력1 ◯ ─────▶ ◯ ─────▶ ◯
        \   /     \   /
         \ /       \ /
          X         X      ─────▶ 출력
         / \       / \
        /   \     /   \
입력2 ◯ ─────▶ ◯ ─────▶ ◯
        \   /     \   /
         \ /       \ /
          X         X      ─────▶ 출력
         / \       / \
        /   \     /   \
입력3 ◯ ─────▶ ◯ ─────▶ ◯
```

#### 1️⃣ 입력층 (Input Layer)

- 데이터가 처음 들어오는 곳
- 예: 손글씨 숫자 이미지(28×28 픽셀)를 인식한다면 → 입력 뉴런 784개 (28×28=784)

#### 2️⃣ 은닉층 (Hidden Layer)

- 입력층과 출력층 사이의 모든 층
- **실제로 패턴을 학습하는 핵심 부분**
- 은닉층이 1개면 → 신경망 (NN)
- 은닉층이 여러 개면 → **딥러닝(Deep Learning)** ← 그래서 "깊다(deep)"

#### 3️⃣ 출력층 (Output Layer)

- 최종 결과를 내보내는 곳
- 예: "이 이미지는 숫자 7이다" → 출력 뉴런 10개(0~9 각 숫자에 해당)

### 🔍 실제 예시: 손글씨 숫자 인식

```
[입력층: 784개]    [은닉층: 128개]    [은닉층: 64개]    [출력층: 10개]

  픽셀 1 ◯  ──┐                                          ┌──▶ "0일 확률"
  픽셀 2 ◯  ──┤                                          ├──▶ "1일 확률"
  픽셀 3 ◯  ──┤    ◯ ◯ ◯ ◯ ...        ◯ ◯ ◯ ...       ├──▶ "2일 확률"
   ...        ├──▶  (128개 뉴런)  ──▶  (64개 뉴런) ──▶ ├──▶  ...
  픽셀 784◯ ──┘                                          └──▶ "9일 확률"
```

이미지의 픽셀 정보가 → 은닉층들을 거치며 → 점점 더 추상적인 패턴(선, 곡선, 모양)을 인식 → 최종적으로 "이 숫자는 무엇인가" 판단

### 🌟 깊은 신경망(Deep Neural Network)의 신비

은닉층이 깊어질수록 더 복잡한 패턴을 학습할 수 있습니다.

**얼굴 인식 예시**:
- **첫 번째 은닉층**: 단순한 선과 점 인식
- **두 번째 은닉층**: 눈, 코, 입의 일부분 인식
- **세 번째 은닉층**: 눈, 코, 입 자체 인식
- **네 번째 은닉층**: 얼굴 전체 인식
- **출력층**: "이 사람은 김철수"라고 판단

층이 깊어질수록 단순한 것에서 복잡한 것으로! 마치 **레고 블록을 쌓아 올리는 것**과 같습니다.

## 7. '학습'이란 무엇인가?

### 🎓 신경망이 '학습한다'는 게 무슨 뜻일까?

인공신경망이 학습한다는 것은 결국:

> **"정답을 맞히기 위한 가장 좋은 가중치(Weight)와 편향(Bias)을 찾아가는 과정"**

입니다. 처음에는 가중치가 무작위로 설정되어 있어요. 학습을 통해 이 숫자들을 조금씩 조절해서 정답에 가까워지는 거죠.

### 🎯 학습의 3가지 핵심 요소

학습을 위해서는 다음 3가지가 필요합니다:

#### 1. 데이터 (Data)
- 문제와 정답의 쌍 (예: 고양이 사진 + "고양이"라는 정답)
- **많을수록 좋다!** (수천~수백만 장)

#### 2. 손실 함수 (Loss Function)
- "지금 신경망이 얼마나 틀렸는지" 측정하는 도구
- 손실이 클수록 → 많이 틀림
- 손실이 작을수록 → 잘 맞춤

#### 3. 최적화 알고리즘 (Optimizer)
- 손실을 줄이는 방향으로 가중치를 조절하는 방법
- 가장 유명한 것: **경사 하강법(Gradient Descent)**

### 🏔️ 경사 하강법 - 산을 내려가는 등산가

경사 하강법을 가장 쉽게 이해하는 방법은 **"안개 낀 산을 내려가는 등산가"** 비유입니다.

```
        ⛰️
       /  \
      /    \
     /      \      ← 등산가가 여기 있음 (높은 손실)
    /        \
   /          \
  /            \
 /     골짜기   \   ← 여기로 내려가야 함 (낮은 손실)
        ▼
```

**상황**:
- 산속에 있는 등산가는 안개 때문에 멀리 못 봅니다.
- **목표**: 가장 낮은 골짜기까지 내려가기 (손실을 최소화)

**전략**:
1. 발 밑의 경사를 확인
2. 가장 가파르게 내려가는 방향으로 한 걸음 내딛기
3. 다시 경사 확인
4. 또 한 걸음 ... 반복!

이게 바로 경사 하강법입니다. **현재 위치에서 손실이 가장 빨리 줄어드는 방향으로 가중치를 조금씩 수정**하는 거죠.

### 🚶 학습률 (Learning Rate)

위 비유에서 **"한 걸음의 크기"**가 바로 **학습률(Learning Rate)**입니다.

| 학습률 | 결과 |
|--------|------|
| 너무 큼 | 한 번에 너무 멀리 가서 골짜기를 지나쳐 버림 (학습 실패) |
| 너무 작음 | 너무 천천히 가서 학습이 영원히 안 끝남 |
| 적당함 | 효율적으로 골짜기에 도달 ✅ |

학습률을 잘 정하는 것은 신경망 학습의 **중요한 비법** 중 하나입니다.

## 8. 순전파와 역전파 - 신경망이 똑똑해지는 과정

### 🔄 학습의 두 단계

신경망 학습은 크게 **두 단계**가 반복됩니다:

```
1. 순전파 (Forward Propagation): 데이터를 넣어서 결과 예측
                ⬇️
2. 손실 계산: 예측이 정답과 얼마나 다른지 측정
                ⬇️
3. 역전파 (Backward Propagation): 틀린 정도를 거꾸로 전파해서 가중치 수정
                ⬇️
            (다시 1번으로!)
```

### ➡️ 순전파(Forward Propagation)

데이터가 입력층 → 은닉층 → 출력층으로 **앞으로 흘러가면서** 계산되는 과정입니다.

#### 🍳 비유: 요리 만들기

레시피대로 요리를 만든다고 생각해보세요.

```
재료 준비 → 손질 → 조리 → 플레이팅 → 완성!
(입력층)    (은닉층 1) (은닉층 2) (은닉층 3) (출력층)
```

각 단계에서 정해진 방법(가중치)대로 처리하는 거죠.

### ⬅️ 역전파(Backpropagation)

**여기가 인공신경망의 진짜 마법이 일어나는 부분입니다!**

순전파로 예측한 결과가 정답과 다르다면? 이 **"틀린 정도(오차)"**를 **출력층에서 거꾸로 입력층 방향으로** 전파시켜서, 각 가중치가 **얼마나, 어느 방향으로** 수정되어야 하는지 계산합니다.

#### 🧑‍🍳 요리 비유 계속...

요리를 했는데 너무 짜다고 가정해봅시다. 그러면:

```
완성 (너무 짠 요리)
    ⬇ "왜 짤까?" 거꾸로 추적
플레이팅 단계 → 문제없음
    ⬇
조리 단계 → "아! 소금 너무 많이 넣었구나!" ← 이 부분 수정!
    ⬇
손질 단계 → 문제없음
    ⬇
재료 준비 → 문제없음
```

다음번에는 소금을 덜 넣게 됩니다. **이게 학습입니다!**

#### 🎯 수학적으로 일어나는 일

역전파에서는 **미분(편미분)**이라는 수학을 사용해서, 각 가중치가 손실(오차)에 얼마나 기여했는지 계산합니다.

너무 어려운 건 신경 쓰지 마세요. **핵심만 기억하면 됩니다**:

> "오차를 줄이려면 각 가중치를 어떻게 바꿔야 하는지 자동으로 계산해주는 마법 같은 알고리즘이 역전파다."

### 🔁 에포크(Epoch)와 배치(Batch)

#### 에포크 (Epoch)
- 전체 데이터를 **한 번 다 학습**하는 것 = 1 에포크
- 보통 10~수백 에포크 동안 학습합니다.
- **비유**: 시험공부할 때 교재를 처음부터 끝까지 한 번 본 것 = 1 에포크

#### 배치 (Batch)
- 데이터를 작은 묶음으로 나눠서 학습하는 단위
- 예: 데이터 10,000개를 100개씩 묶음 → 배치 사이즈 100, 100개 배치
- **비유**: 책 한 권을 한 번에 다 안 보고, 챕터별로 나눠서 보는 것

```
데이터 10,000개
  │
  ├── 배치 1 (100개) → 학습 → 가중치 수정
  ├── 배치 2 (100개) → 학습 → 가중치 수정
  ├── 배치 3 (100개) → 학습 → 가중치 수정
  │   ...
  └── 배치 100 (100개) → 학습 → 가중치 수정
                    ↑
              여기까지가 1 에포크
```

## 9. 실제 예제로 따라가는 학습 과정

### 🌸 예제: 꽃 종류 분류하기

붓꽃(Iris)이라는 꽃의 종류를 분류해 봅시다. 다음 정보로 어떤 종류인지 맞추는 거예요:

**입력 (특징 4개)**:
- 꽃잎 길이
- 꽃잎 너비
- 꽃받침 길이
- 꽃받침 너비

**출력 (3가지 종류 중 하나)**:
- Setosa (세토사)
- Versicolor (버시컬러)
- Virginica (버지니카)

### 🏗️ 신경망 구조

```
[입력층 4개]   [은닉층 8개]   [출력층 3개]
   ◯  ─┐
       ├─▶  ◯                   ┌──▶ Setosa 확률
   ◯  ─┤    ◯                   │
       ├─▶  ◯  ──────▶  ◯ ─────▶┼──▶ Versicolor 확률
   ◯  ─┤    ◯           ◯       │
       ├─▶  ◯           ◯ ─────▶┴──▶ Virginica 확률
   ◯  ─┘    ◯
            ◯
            ◯
```

### 📚 학습 과정 따라가기

#### 1단계: 초기 상태
- 가중치들이 모두 무작위로 설정되어 있음
- 신경망이 아직 아무것도 모르는 상태

#### 2단계: 첫 번째 데이터 학습

**데이터**: 꽃잎 길이 1.4, 꽃잎 너비 0.2, 꽃받침 길이 5.1, 꽃받침 너비 3.5  
**정답**: Setosa

**순전파 결과 (처음에는 완전히 틀림!)**:
- Setosa 확률: 30%
- Versicolor 확률: 35%
- Virginica 확률: 35%

신경망이 "음... Versicolor인 것 같은데?"라고 잘못 예측 ❌

#### 3단계: 손실 계산

정답은 Setosa인데 30%밖에 안 나왔으니 → 손실(오차)이 큼!

#### 4단계: 역전파로 가중치 수정

- "Setosa 확률을 더 높이려면 어떤 가중치를 조정해야 할까?"
- 자동으로 계산해서 가중치들을 살짝 수정

#### 5단계: 같은 데이터로 다시 시도 (또는 다음 데이터로)

이제 같은 꽃을 다시 보면:
- Setosa 확률: 35% ← 살짝 올라감!
- Versicolor 확률: 33%
- Virginica 확률: 32%

#### 6단계: 수천, 수만 번 반복

수많은 데이터로 위 과정을 반복하면, 결국:
- Setosa 확률: 95% ✅
- Versicolor 확률: 3%
- Virginica 확률: 2%

이렇게 정확하게 맞히게 됩니다!

### 📊 학습 곡선

학습이 진행될수록 손실이 줄어드는 모습:

```
손실
 │
 │●
 │ ●
 │  ●
 │   ●●
 │     ●●●
 │        ●●●●
 │            ●●●●●●●●●●●●●●● ← 학습이 잘 되면 손실이 점점 작아짐
 └──────────────────────────────▶ 에포크
```

## 11. 한계와 주의할 점

### ⚠️ 인공신경망이 만능은 아닙니다

#### ❌ 한계 1: 데이터가 많이 필요하다

- 좋은 모델을 만들려면 **수천~수백만 개의 데이터**가 필요
- 데이터가 부족하면 → 학습이 잘 안 됨
- **개인이나 작은 회사**가 처음부터 만들기 어려운 이유

> 💡 **해결책**: **전이학습(Transfer Learning)**  
> 이미 큰 회사들이 만든 모델을 가져와서, 우리 데이터로 살짝만 추가 학습시키는 기법

#### ❌ 한계 2: 블랙박스 문제

- 신경망이 "왜 그렇게 판단했는지" 설명하기 어려움
- 의료, 법률 등 **설명이 중요한 분야**에서는 큰 문제
- 예: AI가 "당신은 대출 거부"라고 했는데, 이유를 모름

> 💡 **연구 분야**: **설명 가능한 AI (XAI, Explainable AI)**가 활발히 연구 중

#### ❌ 한계 3: 편향된 데이터 = 편향된 결과

만약 학습 데이터가 편향되어 있으면, 신경망도 편향됩니다.

**실제 사례**:
- 한 회사의 채용 AI가 남성 지원자만 선호한 사건
- 이유: 과거 채용 데이터에 남성이 더 많았기 때문!

> 🚨 **교훈**: **다양하고 공정한 데이터**가 매우 중요합니다.

#### ❌ 한계 4: 계산 비용이 크다

- 큰 신경망 학습에는 고성능 GPU가 필요
- ChatGPT 같은 모델 학습에는 **수백억 원**의 비용
- 환경 문제(전력 소비)도 점점 심각해짐

#### ❌ 한계 5: 적대적 공격(Adversarial Attack)

- 사람 눈에는 안 보이는 작은 노이즈를 사진에 추가하면 → AI가 완전히 잘못 인식
- 예: 정지 표지판에 작은 스티커를 붙이면 자율주행차가 "속도 제한"으로 잘못 인식

#### ❌ 한계 6: 일반화 능력 부족

- 학습 데이터와 비슷한 환경에서는 잘 작동
- **새로운 상황**에서는 갑자기 실패할 수 있음
- 사람의 "상식"이 부족함

### 🛡️ 윤리적 고려사항

### 💻 첫 코드 맛보기 (선택사항)

Python으로 간단한 신경망을 만드는 코드는 이렇게 짧습니다:

```python
# Keras를 사용한 손글씨 숫자 인식 (전체 코드!)
from tensorflow.keras.datasets import mnist
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense, Flatten

# 1. 데이터 불러오기
(x_train, y_train), (x_test, y_test) = mnist.load_data()
x_train, x_test = x_train / 255.0, x_test / 255.0

# 2. 신경망 만들기 (3줄!)
model = Sequential([
    Flatten(input_shape=(28, 28)),       # 입력층
    Dense(128, activation='relu'),        # 은닉층
    Dense(10, activation='softmax')       # 출력층
])

# 3. 학습 준비
model.compile(optimizer='adam',
              loss='sparse_categorical_crossentropy',
              metrics=['accuracy'])

# 4. 학습 시작!
model.fit(x_train, y_train, epochs=5)

# 5. 평가
model.evaluate(x_test, y_test)
# 결과: 약 97% 정확도!
```

**놀랍지 않나요?** 단 20줄도 안 되는 코드로 손글씨를 97% 정확도로 인식하는 AI를 만들 수 있습니다!

## 13. 용어 사전

| 용어 | 영어 | 쉬운 설명 |
|------|------|----------|
| 인공신경망 | Artificial Neural Network (ANN) | 뇌를 흉내 낸 컴퓨터 시스템 |
| 딥러닝 | Deep Learning | 은닉층이 여러 개인 깊은 신경망 |
| 뉴런 | Neuron | 신경망의 기본 단위 (뇌세포 같은 것) |
| 퍼셉트론 | Perceptron | 가장 단순한 형태의 인공 뉴런 |
| 가중치 | Weight | 입력의 중요도를 나타내는 숫자 |
| 편향 | Bias | 뉴런의 활성화 기준선 |
| 활성화 함수 | Activation Function | 뉴런 출력을 변환하는 함수 |
| ReLU | Rectified Linear Unit | 가장 많이 쓰이는 활성화 함수 |
| 시그모이드 | Sigmoid | 0~1 사이로 변환하는 활성화 함수 |
| 소프트맥스 | Softmax | 여러 분류 중 확률 계산 |
| 입력층 | Input Layer | 데이터가 들어오는 첫 층 |
| 은닉층 | Hidden Layer | 가운데에 있는 층들 |
| 출력층 | Output Layer | 결과가 나오는 마지막 층 |
| 순전파 | Forward Propagation | 입력 → 출력 방향 계산 |
| 역전파 | Backpropagation | 오차를 거꾸로 전파해 가중치 수정 |
| 손실 함수 | Loss Function | 얼마나 틀렸는지 측정하는 함수 |
| 경사 하강법 | Gradient Descent | 손실을 줄이는 가중치 수정 방법 |
| 학습률 | Learning Rate | 한 번에 얼마나 가중치를 수정할지 |
| 에포크 | Epoch | 전체 데이터를 한 번 학습한 단위 |
| 배치 | Batch | 데이터를 나눈 작은 묶음 |
| 과적합 | Overfitting | 학습 데이터만 잘 맞히고 새 데이터엔 약함 |
| 과소적합 | Underfitting | 학습이 부족해서 데이터를 못 맞춤 |
| 전이학습 | Transfer Learning | 이미 학습된 모델을 가져와 활용 |
| 데이터셋 | Dataset | 학습에 사용하는 데이터 모음 |
