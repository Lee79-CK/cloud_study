01
Chapter One

정규식이란 무엇인가

정규식(Regular Expression, 줄여서 regex 또는 regexp)은 "문자열의 패턴을 표현하는 작은 언어"입니다. 한국어, 영어, 파이썬, 자바스크립트와는 다른, 오로지 "텍스트의 모양"을 묘사하기 위해 태어난 언어이지요.

예를 들어 누군가 여러분에게 "문자열 안에서 휴대전화 번호를 모두 찾아 줘" 라고 요청했다고 합시다. 여러분은 머릿속에서 이렇게 생각할 겁니다.

  • "숫자가 3개 나오고, 하이픈, 숫자 4개, 하이픈, 숫자 4개구나"
  • "또는 하이픈 없이 11자리 숫자가 연속될 수도 있겠지"

이 문장을 그대로 정규식으로 옮기면 \d{3}-?\d{4}-?\d{4} 가 됩니다. 정규식은 결국 "머릿속 규칙을 기호로 옮긴 것" 입니다.

한 줄 비유

"문자열을 찾는 검색창이 단어 매칭이라면,
정규식은 규칙 매칭이다."

Ctrl+F 검색이 김철수 같은 정확한 문자열을 찾는 도구라면, 정규식은 "한글로 시작하는 두세 글자 이름"처럼 조건을 검색합니다.

검색 치환 검증 추출 분리

정규식이 잘하는 5가지 일은 위와 같습니다. 이 강의를 마칠 즈음에는 각각을 자유롭게 다룰 수 있게 됩니다.

왜 배우는가 로그 분석, 데이터 클리닝, 입력값 검증, 코드 검색, 파일 일괄 변경, 텍스트 추출 — 엔지니어가 매일 만나는 문제의 절반은 "텍스트에서 규칙을 찾는 문제"입니다. 정규식은 그 문제들을 한 줄로 해결해 주는 만능 열쇠 중 하나입니다.
02
Before / After

정규식을 쓰지 않으면 어떻게 될까

같은 문제를 정규식 없이 풀어 보고, 정규식으로 풀어 보면 차이가 확실하게 보입니다. 세 가지 흔한 예시를 비교해 봅시다.

예시 ① 본문에서 이메일 주소 추출하기

● 정규식 없이

# 공백으로 자르고, '@' 들어있는 토큰만 모은 뒤
# 앞뒤 문장부호 제거 ... 점점 조건이 늘어남
def find_emails(text):
    results = []
    for token in text.split():
        token = token.strip(".,;!?()[]<>")
        if "@" in token and "." in token:
            local, _, domain = token.partition("@")
            if local and "." in domain:
                results.append(token)
    return results
# ↑ 한글 메일, 따옴표, IP 도메인 ... 예외가 끝없이 늘어남
vs

● 정규식과 함께

import re

pattern = r"[\w.+-]+@[\w-]+\.[\w.-]+"

emails = re.findall(pattern, text)
# 끝.

한 줄로 끝났습니다. 게다가 "이메일의 일반적 형태"라는 의도가 코드에 그대로 드러나 있죠. 규칙이 바뀌면 패턴만 고치면 됩니다.

예시 ② 로그에서 IPv4 주소 찾기

● 정규식 없이

def find_ips(line):
    parts = line.split()
    ips = []
    for p in parts:
        chunks = p.split(".")
        if len(chunks) == 4 and \
           all(c.isdigit() and 0 <= int(c) <= 255
               for c in chunks):
            ips.append(p)
    return ips
vs

● 정규식과 함께

ips = re.findall(
    r"\b(?:\d{1,3}\.){3}\d{1,3}\b",
    line
)

한 줄에 "1–3자리 숫자가 점으로 4번 묶여 있다"는 규칙이 직접 드러납니다. (255 초과 검사 같은 의미 단위 검증은 후처리로 분리하면 깔끔합니다 — 정규식과 코드의 역할 분담.)

예시 ③ 연속된 공백/탭/줄바꿈을 공백 하나로 정리

● 정규식 없이

while "  " in text:
    text = text.replace("  ", " ")
text = text.replace("\t", " ")
text = text.replace("\n", " ")
while "  " in text:
    text = text.replace("  ", " ")
vs

● 정규식과 함께

text = re.sub(r"\s+", " ", text)

\s 는 "공백 종류 전부(스페이스, 탭, 줄바꿈, ...)"를, + 는 "1개 이상 연속"을 의미합니다. 이 한 줄로 모든 종류의 연속 공백이 단일 공백으로 정리됩니다.

정리 정규식은 "조건 분기"를 "패턴"으로 압축해 줍니다. 결과적으로 코드가 짧아지고, 의도가 드러나고, 수정도 쉬워집니다. 단, 패턴을 처음 읽을 때는 익숙해질 시간이 필요합니다. 이 강의의 목적이 바로 그 익숙함을 빠르게 얻는 것입니다.
03
The Grammar

정규식 문법, 단계별로 쌓아 올리기

정규식의 모든 문법은 "무엇을 매치할 것인가"를 표현하기 위한 도구입니다. 여기서는 가장 작은 단위(글자 하나)에서 출발해서, 점점 표현력을 키워 나갑니다. 각 문법마다 작은 예시 박스를 두었고, 섹션 끝의 플레이그라운드에서 직접 시험해 볼 수 있습니다.

3.1 리터럴(literal) — 그냥 글자 그대로

가장 기본은 그냥 "찾고 싶은 글자를 그대로 적는 것"입니다. 정규식 apple 은 문자열에서 "apple"이라는 5글자가 연속으로 나오는 부분을 찾습니다. 대부분의 알파벳, 숫자, 한글, 공백은 그냥 자기 자신을 의미합니다.

정규식: apple
텍스트: I ate an apple and an apple pie.

3.2 메타문자(metacharacter) — 특별한 의미를 가진 기호

어떤 문자는 "자기 자신"이 아니라 특별한 의미를 가집니다. 이것들을 메타문자라고 합니다. 외워야 할 메타문자는 다음 12개뿐입니다:

. ^ $ * + ? | ( ) [ ] { } \

만약 이런 글자 자체를 매치하고 싶다면 앞에 \(백슬래시)를 붙여 이스케이프(escape) 합니다. 예를 들어 진짜 점 . 을 찾고 싶으면 \. 으로 씁니다.

.

임의의 한 문자 (줄바꿈 제외)

"무엇이든 한 글자"를 의미합니다. 알파벳, 숫자, 한글, 기호 모두 가능. 단, 기본적으로 줄바꿈은 매치하지 않습니다.

b.t → bat, but, b9t, b!t 매치 ⭕
\.

이스케이프: 진짜 그 글자

메타문자 앞에 백슬래시를 붙이면 "그 글자 자체"가 됩니다. 도메인이나 IP의 점을 찾을 때 자주 씁니다.

3\.14 → 파이는 3.14이다 ⭕   314는 매치 ❌

3.3 문자 클래스 [ ] — "이 중 하나"

대괄호 [ ] 안에 글자들을 넣으면 "그중 아무거나 한 글자"를 의미합니다. 범위를 표시할 때는 -(하이픈)을 씁니다.

[abc]

a 또는 b 또는 c

대괄호 안 글자 중 하나와 매치.

[aeiou] → hello
[a-z]

범위 지정

하이픈으로 연속된 범위. 여러 범위도 가능: [a-zA-Z0-9].

[0-9] → room 301
[^abc]

부정: 이것들 빼고

대괄호 안 첫 글자가 ^ 면 "그 외 아무거나 한 글자".

[^0-9] → P123x

3.4 자주 쓰는 단축 표기 (미리 정의된 문자 클래스)

아래는 매번 [0-9] 같이 쓰기 귀찮으니까 표준으로 약속된 단축어입니다. 정규식의 8할을 이 단축어가 담당한다고 해도 과언이 아닙니다.

기호의미같은 표현예시
\d숫자 한 글자[0-9]\d\d\d → "123"
\D숫자가 아닌 한 글자[^0-9]대문자 D는 보통 "Not"
\w"단어 글자"
(알파벳/숫자/_)		[A-Za-z0-9_]변수명에 자주 사용
\W단어 글자가 아닌 것[^A-Za-z0-9_]구두점, 공백 등
\s공백 종류 한 글자
(스페이스, 탭, 줄바꿈)		[ \t\n\r\f\v]\s+ → 연속 공백
\S공백이 아닌 것[^\s]"빈 칸 아닌 글자들"
.줄바꿈 제외 모든 글자—가장 흔하게 남용되는 메타문자
한글과 \w 엔진/언어에 따라 다릅니다. Python re는 기본적으로 \w에 한글도 포함합니다(유니코드 모드). 하지만 JavaScript의 \w는 ASCII만 — 한글은 빠집니다. JavaScript에서 한글을 잡으려면 [\uAC00-\uD7A3] 또는 \p{L}(u 플래그) 같은 표현을 씁니다.

3.5 수량자(quantifier) — 몇 번 나타나야 하는가

이제 "몇 번 반복"을 표현해 봅시다. 수량자는 바로 앞 요소에 적용됩니다.

*

0번 이상

없어도 되고, 있어도 되고, 여러 번이어도 됩니다.

go*l → gl, gol, goooool
+

1번 이상

최소 한 번은 있어야 함. 가장 자주 쓰는 수량자.

go+l → gol, goool   gl ❌
?

0번 또는 1번

"있어도 되고 없어도 되는" 선택적 부분.

colou?r → color, colour 둘 다
{n}

정확히 n번

고정 횟수. 우편번호, 카드번호처럼 자릿수가 정해진 경우.

\d{5} → 우편번호 34025
{n,}

n번 이상

최소 n번, 최대 무제한.

\d{3,} → 123, 12345   12 ❌
{n,m}

n번 이상 m번 이하

최소·최대 모두 지정.

\d{2,4} → 연도 2026, 번호 38

3.6 앵커(anchor) — "위치"를 지정한다

앵커는 글자를 매치하는 게 아니라 위치를 매치합니다. 줄의 시작, 줄의 끝, 단어의 경계 같은 "보이지 않는 지점"이지요.

^

줄의 시작

문자열(또는 줄)의 맨 앞에서만 매치. "이 줄이 ~로 시작하는가?"

^Error → Error: file not found ⭕
[Warn] Error logged ❌
$

줄의 끝

문자열(또는 줄)의 맨 끝. "이 줄이 ~로 끝나는가?"

\.log$ → app.log ⭕   .log.bak ❌
\b

단어 경계 (word boundary)

단어 글자(\w)와 그 외의 경계. "단어 단위"로 매치하고 싶을 때.

\bcat\b → a cat sat ⭕   catalog ❌
\B

단어 경계가 아닌 곳

단어 안쪽. \b 의 반대.

\Bcat → educate ⭕   a cat ❌

3.7 그룹 ( ) — 묶기, 캡처하기, 재사용하기

괄호의 역할은 세 가지입니다. 모두 같은 기호를 쓰지만 용도가 다릅니다.

  1. 묶기 — 수량자나 대안의 적용 범위를 정해줍니다. 예) (ab)+ 는 "ab"의 반복(ab, abab, ababab...), ab+ 와 다릅니다.
  2. 캡처(capture) — 매치된 부분 중 괄호 안 부분을 "기억"해서 나중에 꺼내 쓸 수 있게 합니다.
  3. 역참조(backreference) — 캡처한 부분을 패턴 안에서 다시 참조합니다.
(ab)+

그룹 묶음 + 반복

"ab"라는 덩어리 자체가 1번 이상 반복.

(ab)+ → abababcd
(\d+)-(\d+)

캡처해서 꺼내기

매치되면 그룹 1, 그룹 2가 따로 저장됨. 치환식에서 $1, $2 또는 \1 로 참조.

(\d+)-(\d+) → 2026-05
$1 = 2026, $2 = 05
(?:...)

비캡처 그룹

"묶기만 하고 저장은 안 함". 성능과 가독성에 좋습니다. 그룹 번호도 차지하지 않습니다.

(?:ab)+c → ababc
(?P<year>\d{4})

명명된 그룹 (Python 문법)

번호 대신 이름으로 캡처. 코드 가독성↑. JS는 (?<year>...).

(?P<y>\d{4}) → match['y'] = "2026"

3.8 대안(alternation) | — A 또는 B

파이프 기호 | 는 OR입니다. 그룹과 함께 쓰면 효과적입니다.

(cat|dog|bird) → a cat and a dog and a bird

https?:// → http:// 와 https:// 둘 다 매치
// 's'가 0회 또는 1회

3.9 욕심쟁이(greedy) vs 게으른(lazy) — 정규식이 가장 자주 헷갈리는 지점

기본적으로 정규식 수량자는 욕심쟁이입니다. 즉 "최대한 길게 매치"하려고 합니다. 뒤에 ? 를 붙이면 게으른(또는 비탐욕적, lazy) 매칭이 되어 "최소한만 매치"합니다.

● 욕심쟁이 (기본)

<.+>
텍스트: "<b>hello</b> <i>world</i>"
↓
매치: <b>hello</b> <i>world</i>
# 처음 < 부터 마지막 > 까지!
vs

● 게으른 (?+)

<.+?>
텍스트: "<b>hello</b> <i>world</i>"
↓
매치1: <b>
매치2: </b>
매치3: <i>
매치4: </i>

기억할 한 줄: "걸어가다 잡히면 멈춰라" = 게으름. "일단 끝까지 가본다" = 욕심. *?, +?, ??, {n,m}? 모두 게으른 버전이 있습니다.

3.10 플래그(flag) — 매칭 방식 옵션

패턴 자체와는 별개로, 매칭 방식을 조정하는 플래그가 있습니다. 가장 흔한 4개만 기억합시다.

플래그의미예시 효과
i대소문자 무시 (ignorecase)/apple/i 는 "Apple", "APPLE"도 매치
g전역 검색 (global, JS에서 모든 매치)모든 매치를 찾음. Python은 findall로 대체
m여러 줄 (multiline)^, $ 가 각 줄의 시작/끝에서도 동작
s한 줄 모드 (dotall). 이 줄바꿈도 포함하게 함
u유니코드 (JS)\p{L} 같은 유니코드 속성 사용 가능

05
Field-tested Patterns

현장에서 자주 쓰는 7가지 패턴

실제 업무에서 만나기 쉬운 7가지 시나리오를 모았습니다. 각 카드의 테스트 텍스트를 직접 수정하면서 패턴이 어떻게 동작하는지 확인해 보세요.

① 이메일 주소 추출

Extract

로그, 가입 명단, 게시글 본문에서 이메일 주소를 모두 꺼내고 싶을 때. 100% 표준 RFC 준수는 아니지만 실무에 충분한 "느슨한 이메일" 패턴입니다.

② 한국 휴대전화 번호

Validate

010으로 시작하는 11자리. 하이픈은 있어도 되고 없어도 됩니다. ^...$ 로 전체 문자열이 번호인지 검증하는 형태로도 자주 씁니다.

③ 날짜(YYYY-MM-DD) 추출 + 캡처

Capture Groups

캡처 그룹을 쓰면 매치 결과를 "년/월/일"로 따로 받을 수 있습니다. 아래 결과 패널의 그룹 1, 2, 3을 확인해 보세요.

④ IPv4 주소

Log Parsing

"1~3자리 숫자 + 점"이 3번, 그리고 마지막 "1~3자리 숫자". \b 로 단어 경계를 잡아 숫자 더미에 들러붙지 않게 합니다.

⑤ http(s) URL 링크 추출

Extract

https? 의 ? 가 "s가 있어도 되고 없어도 된다"를 표현. 뒤는 "공백이 아닌 글자"의 1번 이상.

⑥ 연속 공백/탭/개행을 공백 하나로

Replace

아래 테스터에서는 매치되는 부분만 보여주지만, 실제 코드에서는 re.sub(r"\s+", " ", text) 처럼 치환에 씁니다. 매치된 모든 공백 더미가 한 글자로 압축된다고 상상해 보세요.

⑦ Kubernetes Pod 이름 (RFC 1123)

Validate

소문자/숫자로 시작하고 끝나는, 소문자/숫자/하이픈으로 이뤄진 이름. K8s 리소스 이름의 표준입니다.

🔧 텍스트를 한 줄씩 바꿔 보세요. my-pod, web-1, -bad, UPPER, my_pod ... 무엇이 통과하고 무엇이 막히는지!

06
Across Languages

언어별 정규식 사용법 — Python · JS · Shell

정규식의 문법은 거의 공통이지만, 그것을 호출하는 API와 일부 세부 동작은 언어마다 다릅니다. 여기서는 가장 흔히 쓰는 세 환경의 사용법을 비교해 봅니다.

Python — re 모듈

Python

import re

text = "가입자: alice@a.io, bob@b.kr"
pattern = r"[\w.+-]+@[\w-]+\.[\w.-]+"
# r"..." raw 문자열 — \ 이스케이프 안 해도 됨

# ① 모두 찾기
re.findall(pattern, text)
# ['alice@a.io', 'bob@b.kr']

# ② 첫 매치 객체
m = re.search(pattern, text)
m.group()       # 'alice@a.io'
m.span()        # (5, 16)

# ③ 치환
re.sub(pattern, "[email]", text)
# '가입자: [email], [email]'

# ④ 자주 쓰는 패턴은 컴파일하면 빠름
EMAIL = re.compile(pattern)
EMAIL.findall(text)

# ⑤ 플래그
re.findall(r"^error", log,
           flags=re.IGNORECASE | re.MULTILINE)
JavaScript — 내장 RegExp

JavaScript

const text = "가입자: alice@a.io, bob@b.kr";

// 슬래시 리터럴 (가장 흔함)
const re = /[\w.+-]+@[\w-]+\.[\w.-]+/g;

// ① 모두 찾기 (matchAll, g 플래그 필수)
[...text.matchAll(re)].map(m => m[0]);
// ['alice@a.io', 'bob@b.kr']

// ② 단순 검사
re.test(text);   // true

// ③ 치환
text.replace(re, "[email]");

// ④ 캡처 그룹 — 명명도 가능
const dateRe = /(?<y>\d{4})-(?<m>\d{2})/;
"2026-05".match(dateRe).groups;
// { y: '2026', m: '05' }

// ⑤ 동적 생성 (외부 입력은 escape 필요!)
new RegExp(userInput, "g");
Shell — grep / sed / awk

커맨드라인 — 로그 다루는 사람의 필수기

# grep — 기본은 BRE(기본 정규식), -E로 ERE(확장)
grep -E "^ERROR" app.log
grep -Eo "\b[0-9]{1,3}(\.[0-9]{1,3}){3}\b" access.log # IP만 추출
grep -Ei "timeout|connection refused" app.log    # OR + 대소문자 무시

# sed — 치환
sed -E "s/[0-9]+/N/g" file.txt        # 모든 숫자를 N으로
sed -E "s/(\w+)@(\w+\.\w+)/***@\2/g" users.txt  # 로컬 마스킹

# awk — 정규식으로 줄 필터링
awk "/ERROR/ {print}"  app.log
awk "$1 ~ /^2026-05/ {sum+=$NF} END {print sum}" metric.log
주의: 같은 패턴이 안 통할 수도 있다 엔진(POSIX BRE, ERE, PCRE, JS, Python, Go RE2 등)에 따라 지원하는 문법이 조금씩 다릅니다. 특히 역참조(\1)와 룩어라운드는 Go의 RE2 엔진처럼 일부 엔진에서 지원되지 않습니다. 복잡한 패턴을 옮길 때는 항상 대상 엔진을 의식해야 합니다.
07
Pitfalls & Tips

초보자가 자주 빠지는 함정 7가지

1. . 은 "아무 글자"가 아니다 (사실은)

. 은 줄바꿈을 제외한 한 글자입니다. 여러 줄을 가로질러 매치하고 싶다면 s 플래그(dotall)를 켜야 합니다.

/<a>.*<\/a>/   # 한 줄 안에서만
/<a>.*<\/a>/s  # 줄바꿈 가로질러도 OK

2. 욕심쟁이의 함정

HTML/XML이나 따옴표 안 추출에서 가장 흔한 실수. ".*" 는 첫 따옴표부터 마지막 따옴표까지 다 잡습니다.

# 보통은:
/"(.*?)"/g     # lazy
/"([^"]*)"/g   # 더 안전

3. 이스케이프 빼먹기

., ?, +, (, ) 같은 메타문자를 글자 그대로 찾고 싶으면 \로 감싸야 합니다.

/3.14/   # "3X14"도 매치 ❌
/3\.14/  # 진짜 "3.14"만 ⭕

4. 앵커 없이 검증하기

"전체 문자열이 형식에 맞는지" 검증할 땐 ^ 와 $ 로 양끝을 묶어야 합니다.

/\d{4}/      # "abc1234xyz"도 매치 ❌
/^\d{4}$/    # 정확히 4자리 숫자만 ⭕

5. 한글 / 유니코드 처리

JavaScript의 \w는 ASCII 한정 — 한글이 빠집니다. 한글까지 잡으려면 범위 또는 \p{L}(u 플래그) 사용.

# JS — 한글 단어 매치
/[\uAC00-\uD7A3]+/
/\p{L}+/u   # 모든 언어의 글자

6. 입력값으로 패턴 만들기 (보안)

사용자 입력을 그대로 RegExp() 에 넣으면 정규식 인젝션이나 ReDoS(catastrophic backtracking)가 생길 수 있습니다.

// JS - escape helper
const esc = s =>
  s.replace(/[.*+?^${}()|[\]\\]/g, "\\$&");
new RegExp(esc(userInput), "g");

7. 정규식으로 풀지 말아야 할 문제

정규식은 만능이 아닙니다. 다음과 같은 경우엔 다른 도구가 더 적합합니다.

  • 중첩 구조 파싱(HTML, JSON, 프로그래밍 언어): 본격 파서를 쓰세요. 정규식은 임시방편으로만.
  • 의미 단위 검증(IP의 옥텟이 0–255인가, 윤년의 2월 29일인가): 정규식은 "모양" 검증, 의미 검증은 일반 코드로 분리하면 깔끔합니다.
  • 거대한 텍스트 + 복잡한 백트래킹 패턴: 성능이 폭발할 수 있습니다(ReDoS). 패턴을 단순화하거나 RE2 같은 선형 시간 엔진을 검토.
실전 팁 — 좋은 정규식의 조건 ① 의도가 드러나는 변수명으로 분리한다. EMAIL = re.compile(...)
② 자주 쓰는 패턴은 주석으로 의도를 적어둔다. Python의 re.VERBOSE 플래그를 쓰면 패턴 안에 주석도 가능.
③ regex101.com 같은 시각화 도구로 디버깅한다. 패턴이 어디서 멈추는지 한눈에 보인다.
④ "정규식 하나로 다 풀기"보다 "정규식 + 일반 코드"의 분업을 즐겁게 받아들인다.
