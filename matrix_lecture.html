Chapter 01

행렬이란 무엇인가 — 표(table) 한 장에서 시작합니다

"행렬은 그저 숫자를 직사각형 모양으로 배열한 것이다." — 이 단순한 문장이 출발점입니다.

학교 시험 성적표를 떠올려 봅시다. 가로축에는 과목이, 세로축에는 학생 이름이 있고, 그 교차점에 점수가 들어 있습니다. 이미 우리는 평생 행렬을 사용해 왔습니다 — 단지 그것을 "행렬(matrix)"이라고 부르지 않았을 뿐입니다.

위 SCORES는 3×3 행렬입니다. 가로줄을 행(row), 세로줄을 열(column)이라고 부릅니다. "3행 3열의 행렬"이라고 말하면, 학생 3명의 과목 3개 점수가 정리된 격자 한 장을 의미합니다.

핵심 직관
행렬은 "데이터의 형태"이자 "변환의 도구"입니다. AI에서 우리는 두 가지 의미를 자유롭게 오갑니다 — (1) 데이터를 담는 그릇으로서의 행렬과, (2) 입력을 출력으로 바꾸는 함수로서의 행렬. 이 두 얼굴을 동시에 이해하는 것이 이 강의의 목표입니다.

왜 AI 엔지니어가 행렬을 알아야 하는가

딥러닝 모델 한 줄, 단 한 줄을 본 적이 있다면 — 그 모든 줄은 사실 행렬 연산입니다.

# PyTorch에서 가장 간단한 신경망 레이어
output = torch.matmul(input, weight) + bias
#               ↑          ↑        ↑
#             행렬       행렬     벡터

여기서 matmul은 "matrix multiplication"의 줄임말입니다. GPU가 빠른 이유, GPT 같은 거대 모델이 가능한 이유, 이미지를 합성곱(convolution)으로 처리하는 이유 — 그 모든 것이 결국 큰 행렬을 빠르게 곱하는 능력에 달려 있습니다.

Chapter 02

스칼라·벡터·텐서 — 차원의 사다리

행렬을 외따로 이해하려 하지 마세요. 그것은 "숫자를 담는 차원의 사다리"의 한 단계일 뿐입니다.

수학에서 다루는 숫자 묶음은 차원(dimension)이라는 개념으로 정리할 수 있습니다. 같은 숫자라도 어떻게 "쌓느냐"에 따라 이름이 달라집니다.

엄밀히 말하면 모든 스칼라/벡터/행렬은 텐서의 특수한 경우입니다. 그래서 PyTorch는 모든 데이터를 Tensor라는 하나의 자료형으로 다룹니다. 0차원 텐서 = 스칼라, 1차원 텐서 = 벡터, 2차원 텐서 = 행렬, 3차원 이상 = (일반적인 의미의) 텐서.

다른 수학 개념과의 비교

처음 행렬을 배울 때 가장 헷갈리는 부분은 "이게 함수랑 뭐가 다른가?"입니다. 한 줄로 정리하면:

개념 표현 예시 역할 행렬과의 관계
일차함수 y = ax + b 한 입력 → 한 출력 행렬은 이 식을 n개 입력 → m개 출력으로 확장한 것
연립방정식 2x + 3y = 5
x - y = 1		 여러 변수의 관계 표현 계수만 뽑으면 그대로 행렬: Ax = b 형태
벡터의 내적 a·b = a₁b₁ + a₂b₂ 두 벡터의 유사도 행렬 곱셈은 본질적으로 내적의 반복
좌표 변환 회전, 확대, 반사 점의 위치를 바꾸는 규칙 모든 선형 변환은 행렬 하나로 표현 가능 (Ch.6에서 시각화)

요약하자면, 행렬은 "여러 개의 일차식을 한꺼번에 다루는 도구"입니다. 이 한 문장이 향후 모든 개념의 뿌리가 됩니다.

Chapter 03

기본 용어와 표기법

새로운 도구를 다루려면, 먼저 그 도구를 부르는 이름을 알아야 합니다.

크기(shape)와 원소(element)

행렬 A가 m행 n열이라면 그 크기를 m × n으로 적습니다 ("엠 바이 엔"으로 읽음). 행렬 안의 각 숫자를 원소(element)라 부르고, i행 j열의 원소는 aij로 표기합니다.

주의 — 1-indexed vs 0-indexed
수학에서는 a11(1행 1열)부터 시작하지만, NumPy/PyTorch에서는 A[0][0](0행 0열)부터 시작합니다. 논문을 코드로 옮길 때 이 한 칸 차이로 디버깅에 한 시간을 쓰는 일이 흔합니다.

자주 쓰는 표기 약속

  • ℝm×n — 실수(real number)로 이루어진 m×n 행렬의 집합. AI 논문의 "W ∈ ℝ768×768"는 "W는 768행 768열의 실수 행렬"이라는 뜻.
  • 대문자(A, B, W)는 행렬, 소문자 굵게/화살표(v, →v)는 벡터, 일반 소문자(a, b, λ)는 스칼라.
  • AT — 전치(transpose). 행과 열을 뒤집은 행렬.

3) 곱셈의 중요한 성질

성질일반 수의 곱셈행렬 곱셈
교환법칙 ab = ba ✓ AB ≠ BA (일반적으로 성립하지 않음!) ✗
결합법칙 (ab)c = a(bc) ✓ (AB)C = A(BC) ✓

"AB ≠ BA"는 행렬을 처음 만나는 사람을 가장 당황시키는 사실입니다. 신경망에서 W₁ × W₂와 W₂ × W₁은 완전히 다른 결과를 냅니다. 곧 Ch.6의 시각화에서 이 사실을 직관적으로 느낄 수 있습니다.

Chapter 06

선형 변환 — 공간을 바꾸는 마법

행렬은 단지 숫자판이 아니다. 그것은 공간 전체를 늘이고, 회전시키고, 뒤집는 함수다.

2×2 행렬 하나는 "2차원 평면 전체를 변형하는 규칙"으로 볼 수 있습니다. 벡터 (x, y)에 행렬을 곱하면, 그 벡터의 위치가 새로운 곳으로 이동합니다. 이것이 선형 변환(linear transformation)입니다.

행렬식(determinant)이 0이 되면 평면이 직선으로 찌부러집니다. 이런 행렬은 "역행렬이 없다"고 말하며, 정보를 잃어버리는 변환입니다. 신경망에서 가중치 행렬이 이 상태에 가까워지면 학습이 망가집니다 — 이래서 정규화(regularization)가 필요합니다.

왜 이게 AI에서 중요한가

Linear layer 한 줄(nn.Linear(in, out))이 하는 일은 정확히 이것입니다 — 입력 벡터를 받아, 행렬을 곱해, 새로운 공간으로 옮기는 것. 신경망의 각 층은 데이터를 점점 더 "분류하기 쉬운 모양"으로 변형해 갑니다. 그 변형의 도구가 바로 행렬입니다.

Chapter 07

AI와 ML에서의 활용

이제 다리를 건널 시간입니다. 이 격자가 실제로 어디서 모습을 드러내는지 봅시다.
01 · Neural Network
완전연결층 (Linear Layer)

가장 기본적인 신경망 연산: y = Wx + b

입력 벡터 x를 가중치 행렬 W에 곱하고 편향 b를 더해 출력 y를 만듭니다. GPT의 모든 토큰 처리, ResNet의 분류기 — 핵심은 이 한 줄입니다.

02 · Computer Vision
이미지 = 행렬

흑백 이미지는 픽셀 밝기로 이루어진 2D 행렬입니다. 컬러는 R, G, B 채널 3개를 쌓은 3차원 텐서입니다.

합성곱(convolution)은 작은 필터 행렬을 이미지 전체에 슬라이딩하며 곱하는 연산 — 결국 행렬 곱셈의 응용입니다.

03 · NLP / LLM
임베딩 행렬 & Attention

모든 단어(토큰)는 벡터로 변환되며, 어휘 전체는 거대한 임베딩 행렬 E ∈ ℝV×d로 저장됩니다.

Transformer의 핵심인 Attention은 softmax(QKT/√d)V — Q, K, V 모두 행렬이고, 전치와 곱셈의 향연입니다.

04 · Recommendation
행렬 분해 (Matrix Factorization)

Netflix·YouTube가 쓰는 협업 필터링: 사용자×아이템 평점 행렬을 두 개의 작은 행렬 곱으로 근사합니다.

R ≈ U × VT — 빠진 평점도 이 분해로 예측 가능. 추천의 수학적 뼈대입니다.

05 · Computer Graphics
3D 변환 행렬

게임·3D 렌더링·로봇 공학에서 모델·뷰·투영(MVP) 변환은 모두 4×4 행렬의 곱셈입니다.

NVIDIA GPU가 처음 만들어진 이유 — 4×4 행렬을 초당 수십억 번 곱하기 위해서. AI 가속도 그 부산물입니다.

06 · Data Science
PCA & 차원 축소

고차원 데이터를 압축할 때 쓰는 PCA(주성분 분석)는 공분산 행렬의 고유값 분해입니다.

768차원 BERT 임베딩을 2D 평면에 시각화하는 t-SNE/UMAP도 결국 행렬 연산의 변주입니다.

왜 GPU인가 — 행렬과 병렬성의 만남

행렬 곱셈에서 결과 행렬의 각 원소는 다른 원소와 독립적으로 계산할 수 있습니다. 즉 1000×1000 결과 행렬을 만들 때, 1,000,000개 원소를 동시에 계산해도 됩니다. CPU는 한 번에 8~16개 정도밖에 못 하지만, GPU는 수천~수만 개를 동시에 처리합니다. "AI = 큰 행렬 곱하기"이고, "GPU = 행렬 곱셈 전용 가속기"인 셈입니다.

Chapter 08

실전 — NumPy와 PyTorch로 다뤄보기

개념을 알았다면, 이제 코드로 손에 익혀야 합니다.

NumPy로 행렬 다루기

import numpy as np

# 행렬 만들기
A = np.array([[1, 2, 3],
              [4, 5, 6]])      # 2 × 3
B = np.array([[7, 8],
              [9, 10],
              [11, 12]])      # 3 × 2

# 크기(shape) 확인
print(A.shape)             # (2, 3)

# 덧셈 — 같은 크기끼리만
A + A                       # element-wise 덧셈

# 행렬 곱셈 (세 가지 표기법 모두 동일)
C = np.matmul(A, B)         # 가장 명시적
C = A @ B                   # Python 3.5+ 의 행렬곱 연산자
C = A.dot(B)                # 옛 스타일

# 전치
A.T                         # 3 × 2 가 됨

# 특수 행렬
np.eye(3)                   # 3 × 3 단위행렬
np.zeros((2, 3))             # 영행렬
np.random.randn(2, 3)         # 정규분포 랜덤 행렬

PyTorch로 — GPU 가속의 시작

import torch

# GPU로 보낼 수 있는 텐서로 만들기
A = torch.tensor([[1., 2.], [3., 4.]]).to('cuda')
B = torch.randn(2, 2, device='cuda')

# 행렬 곱 — NumPy와 동일한 인터페이스
C = A @ B                          # GPU에서 즉시 계산
C = torch.matmul(A, B)            # 동일

# Linear layer = 행렬곱 + 편향
linear = torch.nn.Linear(768, 256)   # 768 → 256 차원 변환
# 내부적으로: output = input @ W.T + b
#         W ∈ ℝ^(256 × 768),  b ∈ ℝ^256

# 배치 행렬곱 — 딥러닝의 일상
batch = torch.randn(32, 10, 20)    # 32개 (10×20) 행렬
weights = torch.randn(32, 20, 5)
result = torch.bmm(batch, weights)      # (32, 10, 5)
실무 팁 — 가장 흔한 에러
RuntimeError: mat1 and mat2 shapes cannot be multiplied (32x768 and 256x10)

이 에러는 100% "가운데 차원이 안 맞아서" 납니다. (32×768) × (256×10) → 768 ≠ 256. 해결책: 둘 중 하나를 전치(.T)하거나, Linear layer의 입출력 차원을 확인하세요.
