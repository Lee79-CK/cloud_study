TensorFlow는 구글이 만든 머신러닝·딥러닝 프레임워크입니다. 이름 그대로 데이터(텐서)가 계산 그래프를 따라 흐르며(flow) 모델이 학습합니다. 수식 한 줄 몰라도 괜찮아요. 직접 만지며 원리를 이해해 봅시다.

01 — Overview

01TensorFlow란 무엇인가?

한 문장으로: 숫자 덩어리(텐서)를 빠르게 계산하고, 그 계산 과정을 자동으로 미분해서 "학습"시켜 주는 도구상자입니다. 2015년 구글 브레인 팀이 오픈소스로 공개했고, 지금은 전 세계에서 가장 널리 쓰이는 딥러닝 프레임워크 중 하나입니다.

🍳
비유로 이해하기. TensorFlow를 "고성능 주방"이라고 생각해 보세요. 여러분은 요리사(개발자)이고, 재료(데이터)를 준비해서 레시피(신경망 구조)를 짜면, 주방은 알아서 가장 빠른 화구(CPU·GPU·TPU)를 골라 요리(학습)를 해 줍니다. 불 조절(미분 계산)도 자동입니다.

왜 이런 게 필요할까?

딥러닝 모델은 수백만~수십억 개의 숫자(가중치)를 조금씩 바꿔가며 정답에 가까워지도록 "학습"합니다. 이 과정에는 엄청난 양의 행렬 곱셈과 미분(기울기) 계산이 필요합니다. 이걸 매번 직접 코딩하면 평생 걸립니다. TensorFlow는 이 모든 걸 대신 해 줍니다.

🧮

자동 미분

모델이 얼마나 틀렸는지 계산하면, 가중치를 어느 방향으로 바꿔야 할지(기울기)를 TensorFlow가 자동으로 구해 줍니다. (autograd)

⚡

하드웨어 가속

같은 코드를 CPU, GPU, 구글 전용 칩 TPU에서 그대로 실행. GPU를 쓰면 학습이 수십~수백 배 빨라집니다.

🚀

배포까지 한 번에

학습한 모델을 서버(TF Serving), 모바일(TF Lite), 웹브라우저(TF.js)로 곧장 내보낼 수 있는 생태계를 갖췄습니다.

알아두기"텐서(Tensor)"는 여러 차원으로 늘어선 숫자 배열이고, "플로우(Flow)"는 그 숫자들이 계산 단계를 거쳐 흘러가는 것을 뜻합니다. 두 단어를 합친 게 바로 TensorFlow. 다음 섹션에서 텐서를 직접 만져 봅니다.
02 — Core Concept

02핵심 개념 ① — 텐서(Tensor)

텐서는 어렵지 않습니다. 숫자를 담는 "차원이 있는 상자"일 뿐입니다. 차원(축) 개수에 따라 이름이 달라질 뿐, 본질은 모두 같은 숫자 배열입니다. 아래 버튼을 눌러 직접 모양을 바꿔 보세요.

03 — Core Concept

03핵심 개념 ② — 계산 그래프 & 자동 미분

텐서가 "재료"라면, 계산 그래프는 "레시피 순서도"입니다. 어떤 텐서에 어떤 연산을 거쳐 결과가 나오는지를 노드(node)와 화살표로 표현한 지도죠. TensorFlow는 이 지도를 보고 자동으로 미분까지 해냅니다.

🔁 순전파 (Forward)

입력 데이터가 그래프를 따라 앞으로 흘러가며 예측값을 만듭니다. 예: y = w·x + b → 예측 완료.

⬅️ 역전파 (Backward)

예측이 정답과 얼마나 틀렸는지(손실)를 기준으로, 그래프를 거꾸로 따라가며 "각 가중치를 어떻게 바꿔야 손실이 줄지"를 계산합니다. 이게 바로 자동 미분.

Eager 모드 vs Graph 모드 — 두 가지 실행 방식

초보자가 자주 헷갈리는 부분입니다. 쉽게 정리하면:

구분Eager 모드 (즉시 실행)Graph 모드 (@tf.function)
동작 방식한 줄 한 줄 바로 실행 (파이썬처럼)그래프로 먼저 컴파일 후 실행
장점디버깅 쉬움, 직관적, 배우기 좋음훨씬 빠름, 배포에 최적화
언제?개발·실험·학습할 때 (TF2의 기본값)속도가 중요한 학습 루프·서빙
핵심TensorFlow 2.x부터는 Eager 모드가 기본이라 처음 배우기 훨씬 쉬워졌습니다. 속도가 필요하면 함수 위에 @tf.function 데코레이터 한 줄만 붙이면 자동으로 그래프 모드로 바뀝니다.

05 — Ecosystem

05TensorFlow 생태계 한눈에

TensorFlow의 진짜 강점은 단일 라이브러리가 아니라 "학습부터 배포까지" 이어지는 거대한 생태계라는 점입니다. (MLOps 관점에서 특히 강력합니다.)

Keras

TF의 공식 고수준 API. 레고 블록처럼 신경망을 쌓습니다. 초보자는 여기서 시작하세요.

tf.data

대용량 데이터를 효율적으로 불러오고 전처리하는 파이프라인 도구.

TF Serving

학습한 모델을 REST/gRPC API 서버로 배포. Kubernetes 위에서 운영하기 좋습니다.

TF Lite

모델을 경량화해 모바일·IoT 기기에서 실행.

TensorFlow.js

웹브라우저·Node.js에서 모델을 학습·실행.

TFX

프로덕션 ML 파이프라인 전체를 묶는 엔드투엔드 플랫폼.

TensorBoard

학습 과정(손실·정확도 등)을 그래프로 시각화하는 대시보드.

TF Hub

미리 학습된 모델을 가져다 바로 쓰는 모델 저장소.

🏗️
MLOps 한 줄 메모. Keras로 학습 → SavedModel로 저장 → TF Serving 컨테이너로 패키징 → Kubernetes에 배포 → TensorBoard로 모니터링 이라는 흐름이 표준처럼 자리 잡혀 있어, 모델을 실서비스로 올리기까지의 경로가 잘 닦여 있습니다.
06 — Hello, World

06첫 코드 따라하기 — 손글씨 숫자 인식

개념을 봤으니 실제 코드를 봅시다. 아래는 0~9 손글씨 숫자(MNIST)를 알아맞히는 신경망의 전체 코드입니다. 놀랍게도 단 20여 줄입니다. 각 줄 위에 마우스를 올려 보세요.

mnist_classifier.py 
import tensorflow as tf

# 1) 데이터 불러오기 — 손글씨 숫자 7만 장 (TF에 내장)
(x_train, y_train), (x_test, y_test) = tf.keras.datasets.mnist.load_data()
x_train, x_test = x_train / 255.0, x_test / 255.0  # 0~1 범위로 정규화

# 2) 모델 설계 — 레고처럼 층(layer)을 쌓는다
model = tf.keras.Sequential([
    tf.keras.layers.Flatten(input_shape=(28, 28)),   # 28x28 이미지를 한 줄로 펼침
    tf.keras.layers.Dense(128, activation='relu'),  # 은닉층: 뉴런 128개
    tf.keras.layers.Dense(10, activation='softmax') # 출력층: 0~9 확률
])

# 3) 학습 방법 설정 — 옵티마이저·손실함수·평가지표
model.compile(optimizer='adam',
              loss='sparse_categorical_crossentropy',
              metrics=['accuracy'])

# 4) 학습 — 데이터를 5번 반복 학습 (경사하강법 자동 수행!)
model.fit(x_train, y_train, epochs=5)

# 5) 평가 — 본 적 없는 데이터로 정확도 확인
model.evaluate(x_test, y_test)  # 보통 97% 이상 정확도!

📖 읽는 법

Sequential은 "층을 차곡차곡 쌓는다"는 뜻, Dense는 "모든 뉴런이 연결된 층", relu/softmax는 뉴런을 활성화하는 함수입니다. 이름이 낯설어도, 결국 "레고 블록 조립"이라고 생각하면 됩니다.

✨ 마법은 fit() 한 줄

앞서 데모에서 본 경사하강법·자동미분·가중치 업데이트가 전부 model.fit() 안에서 자동으로 일어납니다. 우리가 직접 미분을 코딩할 필요가 전혀 없죠.

손에 익히려면위 코드는 Google Colab(무료, 브라우저에서 실행)에 그대로 붙여넣으면 바로 돌아갑니다. 설치도 필요 없습니다. "직접 한 번 돌려보기"가 가장 빠른 학습법입니다.

라이브러리배우기 쉬움한 줄 설명

TensorFlow vs PyTorch — 가장 흔한 질문

🔶 TensorFlow

강점: 배포·운영(TF Serving, TF Lite, TF.js), 모바일/웹/엣지까지 이어지는 생태계, 구글·기업 환경에서의 안정성.

예전 이미지: "어렵다"였지만 TF2 + Keras 이후 매우 쉬워졌습니다.

🔥 PyTorch

강점: 파이썬스럽고 직관적인 코드, 디버깅 편의성, 연구·논문 커뮤니티의 사실상 표준. 최신 모델 구현체가 대부분 PyTorch.

약점이었던: 배포는 약했으나 TorchServe·ONNX 등으로 빠르게 따라잡는 중.

결론부터둘 다 훌륭하고 개념(텐서·자동미분·경사하강)은 거의 동일합니다. 하나를 익히면 다른 하나도 금방 배웁니다. 연구·논문 따라하기가 목표면 PyTorch, 모바일/웹/대규모 배포·기업 운영이 목표면 TensorFlow가 무난한 출발점입니다.

그래서, 나는 무엇을 써야 할까?

→ TensorFlow + Keras

  • 딥러닝 불필요한 표 형태 데이터
  • 가볍고 배우기 쉬움

🚀 대규모·고성능 연구

→ JAX
  • 구글의 차세대 고성능 라이브러리
  • TPU 대규모 학습에 강함
  • 난이도는 높은 편 (중상급용)
